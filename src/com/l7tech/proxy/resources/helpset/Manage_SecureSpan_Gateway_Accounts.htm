Manage SecureSpan™ Gateway Accounts

NOTE: The term "identity" includes both users and groups, the term "user" can represent an individual human or machine, the term “service” includes both Web services and XML applications, and the term "SecureSpan™ Gateway" includes all of the SecureSpan Gateway products.

The SecureSpan™ Bridge acts as a service proxy by establishing a PKI-based trust relationship with one or more SecureSpan™ Gateways. Known as a “Gateway Account,” the proxy relationship in the SecureSpan Bridge is essentially a setting that combines a SecureSpan Gateway with a single set of credentials that are used to process service requests.

 

A single SecureSpan Gateway can be accessed by multiple SecureSpan Bridges. Furthermore, a single SecureSpan Bridge can access more than one Gateway Account — each with different credentials — for a single SecureSpan Gateway. In a configuration where the SecureSpan Bridge is servicing requests from a JSP page hosted on IIS, for example, the JSP page could be programmed to send requests in three ways. Requests from a certain IP address could be sent as “guest” users, requests with a user name and password could be sent as “member” users, and requests from inside the corporate firewall could be sent as “employee” users and routed to a separate internal SecureSpan Gateway. In this scenario, each way of sending requests requires a different set of credentials and a separate Gateway Account in the SecureSpan Bridge.

 

Two different types of Gateway Accounts are available in the SecureSpan Bridge: 1) A Trusted Gateway Account for a SecureSpan Gateway on which you have an established identity (see Create Trusted Gateway Accounts) and 2) A Federated Gateway Account for a SecureSpan Gateway on which you do not have an established identity (see Create Federated Gateway Accounts). Each play an important role in an identity-bridging configuration. See Identity Bridging for more information.

 

For information on changing a Gateway Account password, editing and deleting a Gateway Account, or assigning and deleting the default status applied to a Gateway Account, see Manage Existing Gateway Accounts.
