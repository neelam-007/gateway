Import/Export Policies

In the SecureSpan™ Bridge, XML-based policies can be imported into and exported from the Gateway Account Properties form.

Import a Policy

An XML-based policy can be manually imported into the SecureSpan™ Bridge. The policy must contain references to recognizable SecureSpan™ Gateway assertions and contain a meaningful assertion tree structure in order to be recognized and applied to service request messages by the SecureSpan Bridge. The policy could be an exported SecureSpan™ Manager policy or a policy that was created in another program.

 

When a policy is imported into the SecureSpan Bridge, the Policy Attachment Key for the service that it applies to must be manually configured in the Configure Policy Attachment Key form. Ensure that you have the service namespace, SOAPAction header, and proxy URI qualifier for the target service before you import a new policy. An imported policy will only contain the assertions applicable to the type of message decoration that will be performed by the SecureSpan Bridge. See About Stored Policies for more information.

 

  1. Select the target Gateway Account from the Account List.

  2. Click the [Properties] button on the Main Tool Bar or select [File] > Properties from the Main Menu.

  3. The Gateway Account Properties form appears. Click the [Service Policies] tab.

  4. Click [Import]. The Open form appears. Navigate to and select the target policy. Click [Open].

 NOTE: Imported policies must be in XML format with a document element of either <wsp:Policy> or <exp:Export>, as saved from the SecureSpan™ Bridge or exported from the SecureSpan™ Manager. An “Unable to parse policy” error message will appear if the selected file is not a properly formatted XML file.

  1. The Configure Policy Attachment Key form appears. Enter the Policy Attachment Key details and select the service options as outlined in Manage Stored Policies.  

  2. Click [OK]. The Policy Attachment Key is added to the Services with Cached Policies table. See About Stored Policies for information on viewing the new policy.

 

Export a Policy

A copy of the policy stored in the SecureSpan™ Bridge can be exported as an XML file. For information about the policy assertions used by the SecureSpan Bridge, see About Stored Policies.

 

  1. Select the target Gateway Account from the Account List.

  2. Click the [Properties] button on the Main Tool Bar or select [File] > Properties from the Main Menu.

  3. The Gateway Account Properties form appears. Click the [Service Policies] tab.

  4. Click [Export]. The Export form appears. Navigate to the target directory. Click [Save].
