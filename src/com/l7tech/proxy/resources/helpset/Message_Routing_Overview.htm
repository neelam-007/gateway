Message Routing Overview

 NOTE: The term "identity" includes both users and groups, the term "user" can represent an individual human or machine, the term “service” includes both Web services and XML applications, and the term "SecureSpan™ Gateway" includes all of the SecureSpan Gateway products.

The SecureSpan™ Bridge performs three main tasks that enable it to act as an effective proxy between a client application and a SecureSpan™ Gateway-protected service:

  1. The SecureSpan Bridge obtains and saves the security requirements for a service as a policy in the applicable Gateway Account. A policy is downloaded from the SecureSpan Gateway during the first consumption of a service, or imported from another location into the SecureSpan Bridge. See Policies Overview for more information.

  2. When the SecureSpan Bridge receives a service request message from a client application, it decorates the message according to the security requirements required by the service as defined in the saved policy before routing the request to the target SecureSpan Gateway for processing.

  3. When the SecureSpan Bridge receives a response message from the SecureSpan Gateway, it un-decorates the message according to the security requirements that were in effect for the corresponding request. The SecureSpan Bridge then returns the response message to the client application to complete the service transaction process.

 

See About Message Routing for more information about the request and response message routing process. For information about configuring a client application to send service request messages to the SecureSpan Bridge, see Configure Client Applications.
