Manage Stored Policies

 NOTE: For more information about the [Service Policies] tab, see About Stored Policies.

In the SecureSpan™ Bridge, stored policies are viewed in the Gateway Account Properties form for the Gateway Account representing the SecureSpan™ Gateway that is protecting the associated services. Polices are listed in the Services with Cached Policies table in the [Service Policies] tab according to the Policy Attachment Key gleaned from the policy that is automatically downloaded from the SecureSpan Gateway or manually configured during the manual importation of a policy. Policy details for a service selected in the Services with Cached Policies table are viewed in the Associated Policy window. For more information about the Services with Cached Policies table and the Associated Policy window, see About Stored Policies.

 

Individual stored policies in the SecureSpan Bridge can be locked, unlocked, configured, and deleted, and all cached policies stored for a Gateway Account can be purged. For information about importing and exporting policies to and from the SecureSpan Bridge, see Import/Export Policies.

Lock or Unlock a Stored Policy

 TIP: You can also lock or unlock a policy in the Configure Policy Attachment Key form. See Edit the Policy Attachment Key for more information.

The Lock column in the Services with Cached Policies table contains a check box for each row in the table. When checked, the policy is considered locked and cannot be edited, deleted, or over-written by a policy that is automatically downloaded from the SecureSpan Gateway. By default, all manually configured and imported policies are locked, whereas automatically downloaded policies are unlocked.

 

To lock an unlocked policy in the Services with Cached Policies table, check the check box in the Lock column that corresponds to the target service. To unlock a locked service policy, uncheck the check box in the corresponding Lock column. The locked or unlocked status of a policy can also be set in the Configure Policy Attachment Key form. See Edit the Policy Attachment Key for more information.

 

Set the Policy Match Type

 TIP: You set the match type for a policy in the Match Type column in the Services with Cached Policies table and in the Configure Policy Attachment Key form.

In the SecureSpan™ Bridge, the match type setting for a Policy Attachment Key in the Services with Cached Policies table determines how an incoming request message is matched to a policy for the target service. Two match type options are available: “equals” and “begins with.”

Set an “Equal” Policy Match Type

An “equals” match requires that the incoming request message exactly match the Policy Attachment Key in order to be processed under its associated policy. To set the equal match type:

  • Select “equal” from the Match Type column for the target service in the Services with Cached Policies table, or

  • Select the target service from the Services with Cached Policies table. Click [Edit]. The Configure Policy Attachment Key form appears. In the “Apply policy to requests…” statement, select “equals” from the drop-down list.

 

If an incoming request message does not exactly match the Policy Attachment Key that belongs to the target policy and service, then the SecureSpan™ Bridge will not apply any security decorations and will process the message using SSL, with a client certificate if one exists. When the request message is routed to the SecureSpan™ Gateway for processing, the request may be denied for not matching the security requirements configured for the protected service.

Set a “Begins With” Policy Match Type

When selected in the Match Type column, the “begins with” match type permits the processing of an incoming request message that matches the partial values in the Policy Attachment Key. To configure an expected setting in the Policy Attachment Key for a “begins with” matching scenario:

 NOTE: Ensure that you configure unique values for the Policy Attachment Keys that have a “begins with” setting.

  • If the Policy Attachment Key already contains the partial values, select “begins with” from the Match Type column for the target service in the Services with Cached Policies table, or

  • If the Policy Attachment Key must be changed, then:

    1. Select the target service from the Services with Cached Policies table. Click [Edit]. The Configure Policy Attachment Key form appears.

    2. Edit the Policy Attachment Key fields as outlined in Edit the Policy Attachment Key. Do one of the following:

      • If configuring a Policy Attachment Key for a SOAP Web service, then enter at least one character for each field value

      • If configuring a Policy Attachment Key for an XML application or non-SOAP service, then:

 NOTE: For more information about SOAP and XML application or non-SOAP messages and Policy Attachment Keys, see About Message Routing.

        1. Ensure that the Body Namespace and SOAPAction fields are blank.

        2. Enter at least one character in the Proxy URI field.

    1. In the “Apply policy to requests…” statement, select “begin with” from the drop-down list.

    2. Click [OK]. The Policy Attachment Key is immediately changed in the Services with Cached Policies table.

 

 NOTE: “Begins with” in the Services with Cached Policies table and “begin with” in the Configure Policy Attachment Key form are same value.

 

When the SecureSpan™ Bridge receives a service request message, it first compares the message’s Policy Attachment Key with all of the stored Policy Attachment Keys that have an “equals” setting. If an exact match is not found, then the SecureSpan Bridge will compare the message’s Policy Attachment Key with the stored Policy Attachment Keys that have a “begins with” setting. When a single match is found, the SecureSpan Bridge will proceed to decorate and route the request message as discussed in About Message Routing. If the Policy Attachment Key matches more than one Policy Attachment Key with a “begins with” setting, then the SecureSpan Bridge will choose one of the matches. As a result, the request message may be processed under the incorrect policy, causing the SecureSpan™ Gateway to deny the request. To prevent this scenario, it is recommended that you configure unique partial values for the “begins with” Policy Attachment Keys.

 

Edit the Policy Attachment Key

 NOTE: You can also set the match type for the Policy Attachment Key in the Configure Policy Attachment Key form. For detailed information about the match type options, see Set the Policy Match Type.

The Policy Attachment Key is the combination of the Body Namespace value, SOAPAction header, and Proxy URI from a policy. A service request must either exactly match a Policy Attachment Key or match the beginning values of a Policy Attachment Key in order to be processed under the target policy. If an incoming request does not match or partially match the Policy Attachment Key for the target service, then no security decorations will be applied to the request and it will be processed using SSL, with a client certificate if one exists, by the SecureSpan™ Bridge. When the request message is sent to the SecureSpan™ Gateway for processing, the request may be denied for not matching the security requirements configured for the protected service. See About Message Routing for an example of a request message Policy Attachment Key.

 

 NOTE: When configuring a Policy Attachment Key for an XML application or non-SOAP service, leave the Body Namespace and SOAPAction fields blank in the Configure Policy Attachment Key form. For more information, see About Message Routing.

 

The values that make up the Policy Attachment Key for an unlocked service can be edited as follows:

 

 NOTE: The following Policy Attachment Key example assumes an “equal” policy match. For information on configuring the Policy Attachment Key for the “begins with” match type, see Set the Policy Match Type. For detailed information about the Services with Cached Policies table, see About Stored Policies.

 

  1. Select the target Gateway Account from the Account List.

  2. Click the [Properties] button on the Main Tool Bar or select [File] > Properties from the Main Menu.

  3. The Gateway Account Properties form appears. Click the [Service Policies] tab.

 NOTE: The service in the Services with Cached Policies table must be unlocked in order to edit its Policy Attachment Key. If necessary, see Lock or Unlock a Stored Policy above for information on unlocking the policy.

  1. Select the target service from the Services with Cached Policies table.

  2. Click [Edit]. The Configure Policy Attachment Key form appears.

  3. In the Body Namespace field, enter a new service namespace.

 NOTE: If the Policy Attachment Key and associated policy will be used for an XML application or non-SOAP service, then leave the Body Namespace and SOAPAction fields blank. For more information, see About Message Routing.

  1. In the SOAPAction field, enter a new SOAPAction header value.

  2. In the Proxy URI field, enter a new unique qualifier for the SecureSpan Gateway. For example, if a request is made to the SecureSpan Gateway identified by the proxy URL “http://localhost:7700/gateway17/accounting/expenses,” then the Proxy URI for the request would be the string “/gateway17/accounting/expenses.”

  3. Set the match type for the Policy Attachment Key as outlined in Set the Policy Match Type.

  4. To lock the Policy Attachment Key, check the Lock this policy check box.

  5. Click [OK]. The Policy Attachment Key is immediately changed in the Services with Cached Policies table.

 

Delete a Policy

 NOTE: You can only delete unlocked policies. For information on unlocking a locked policy, see Lock or Unlock a Stored Policy. See Delete All Cached Policies for information on deleting all cached policies from the Gateway Account.

To delete a single unlocked Policy Attachment Key and its associated policy from the SecureSpan™ Bridge:

 

  1. Select the target Gateway Account from the Account List.

  2. Click the [Properties] button on the Main Tool Bar or select [File] > Properties from the Main Menu.

  3. The Gateway Account Properties form appears. Click the [Service Policies] tab.

  4. Select the target Policy Attachment Key from the Service Policies with Cached Policies table. Click [Delete].

  5. A confirmation message appears. Click [OK]. The policy is removed from the [Service Policies] tab and SecureSpan Bridge.

 

Delete All Cached Policies

 NOTE: You can only delete unlocked policies. For information on unlocking a locked policy, see Lock or Unlock a Stored Policy. For information on deleting a single policy, see Delete a Policy.

To delete all of the Policy Attachment Keys and associated policies attached to a Gateway Account from the SecureSpan™ Bridge:

 

  1. Select the target Gateway Account from the Account List.

  2. Click the [Properties] button on the Main Tool Bar or select [File] > Properties from the Main Menu.

  3. The Gateway Account Properties form appears. Click the [Service Policies] tab.

  4. Click [Clear Policy Cache]. A confirmation message appears. Click [OK]. All policies are removed from the [Service Policies] tab and SecureSpan Bridge.
