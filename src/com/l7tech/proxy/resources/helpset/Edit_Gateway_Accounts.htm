Manage Gateway Accounts

In the SecureSpan™ Bridge, configured SecureSpan™ Gateway Accounts appear in the Account List. Each column in the Account List corresponds with a property of the associated Gateway Account:

  • The URL in the Gateway column is the host name of the SecureSpan Gateway that is protecting the target service

  • The name in the Proxy column is the unique proxy URI prefix assigned to the Gateway Account by the SecureSpan Bridge. The numbered portion of the identifier, “gateway<sequential number>,” is assigned based on the sequential number of Gateway Accounts configured in the SecureSpan Bridge

  • The Type column identifies the type of Gateway Account — trusted or federated — configured between the SecureSpan Bridge and the SecureSpan Gateway

  • The User Name column contains the user name portion of the credentials, if any, configured for the Gateway Account.

 

 NOTE: See SecureSpan™ Bridge Interface for more information about the SecureSpan™ Bridge interface, and About Stored Policies and About Message Routing for more information about stored service policies and the message routing process.

 

A Gateway Account can be edited or deleted, the password for the Gateway Account and SecureSpan Gateway can be changed, a Gateway Account can be set as the default in the Account List, and the default status in the SecureSpan Bridge can be removed at any time.

Edit a Gateway Account

To edit the configuration settings of a Gateway Account:

 

  1. Select the target Gateway Account from the Account List.

  2. Click the [Properties] button on the Main Tool Bar or select [File] > Properties from the Main Menu.

  3. The Gateway Account Properties form appears. See Create Trusted Gateway Accounts or Create Federated Gateway Accounts for field-level information and instructions.

 

Delete a Gateway Account

 NOTE: Deleting the default Gateway Account from the Account List will not automatically assign default status to a different Gateway Account. You must manually specify a new default Gateway Account for the SecureSpan™ Bridge as outlined in Change the Default Gateway Account. Deleting a default or non-default Gateway Account will not result in the re-numbering of the remaining Gateway Accounts in the Account List.

To delete a Gateway Account from the SecureSpan™ Bridge:

 

  1. Select the target Gateway Account from the Account List.

  2. Click the [Remove] button on the Main Tool Bar or select [File] > Remove from the Main Menu.

  3. A confirmation message appears. Click [Remove].

 NOTE: If the Gateway Account is attached to a client certificate private key that will also be deleted, then a warning message will appear. Click [OK] to proceed to delete the Gateway Account. If the removed private key is required by another Gateway Account in the SecureSpan™ Bridge, then do one of the following before proceeding to delete the Gateway Account: 1) Import the client certificate and private key into the other Gateway Account as outlined in the Import the Client Certificate section in Create Trusted Gateway Accounts, or 2) Make a backup copy of the keystore file “ ~/.l7tech/keysNNN.p12,” delete the first Gateway Account, then restore the keystore file to the appropriate directory for use by the other Gateway Account.

  1. The Gateway Account is removed from the Account List.

 

Change the SecureSpan™ Gateway Password

 NOTE: You can only change a password for a user name that is provided by the Internal Identity Provider in the SecureSpan™ Gateway. Credentials that are provided by an LDAP Identity Provider or non-LDAP identity provider should be changed with their management program. For information about identity providers and the optional add-on assertions that permit connections to external programs, or non-LDAP identity providers, see the  “SecureSpan™ Manager User Manual” or program-based “SecureSpan™ Manager Help System.”

The SecureSpan™ Gateway password that is used in the Gateway Account in the SecureSpan™ Bridge and that is used to login to the SecureSpan Gateway and SecureSpan™ Manager can be modified as follows:

 

 NOTE: The password associated with a Gateway Account in the SecureSpan™ Bridge is also used to login to the SecureSpan™ Gateway from the SecureSpan™ Manager. Therefore, changing the password from the SecureSpan Bridge changes the SecureSpan Gateway password everywhere. If the credentials for the Gateway Account are tied to a client certificate, then changing the password will automatically revoke the client certificate that is saved in the SecureSpan Bridge and SecureSpan Gateway. For information on the use of client certificates in a Gateway Account, see About SSL/TLS Security Sessions.

 

  1. Select the target Gateway Account from the Account List.

  2. Select [File] > Change Account Password from the Main Menu.

  3. A confirmation message appears. Click [Change Password].

  4. The Log on Gateway form appears. Enter the user name into the User Name field.

  5. Enter the current password for the user name into the Password field. Click [OK].

  6. The Change Password form appears. Enter the new password into the Password field. Repeat the new password in the Verify Password field.

  7. Click [OK]. The new password is automatically applied to the SecureSpan Gateway and Gateway Account in the SecureSpan Bridge. If the Gateway Account included a client certificate, then the client certificate is revoked and the new password is automatically applied to the SecureSpan Gateway and Gateway Account in the SecureSpan Bridge. During the next consumption of the service, the SecureSpan Bridge will use a Certificate Signing Request (CSR) to request a new private key from the SecureSpan Gateway. Repeat the client certificate generation process outlined in About SSL/TLS Security Sessions.

 

Change the Default Gateway Account

 NOTE: If you do not want to have a default Gateway Account in the SecureSpan™ Bridge, then follow the instructions in Remove the Default Status.

The default Gateway Account is the default handler for service requests sent to “localhost” without a specified SecureSpan™ Gateway. Default status is automatically assigned to the first Gateway Account configured in the SecureSpan Bridge. Default status can be changed to another Gateway Account at any time as follows:

 

  1. Select the target Gateway Account from the Account List.

  2. Click the [Set Default] button on the Main Tool Bar or select [File] > Set Default from the Main Menu.

  3. The new default Gateway Account in the Account List is bolded.

 

Remove the Default Status

For security purposes, you may not want to assign default status to a Gateway Account in the Account List. To remove the default status from the Account List and SecureSpan™ Bridge:

 

  1. Create a dummy Gateway Account as follows:

    1. Open the Create Gateway Account form by one of two methods:

      • Click [New] on the Main Tool Bar or,

      • Click [File] > New from the Main Menu.

    1. In the Create Gateway Account form, click [Create]. The Gateway Account Properties form appears.

    2. Enter any two characters into the Gateway Host Name field. Click [OK].

    3. The dummy Gateway Account appears in the Account List. The two characters that were entered in the Gateway Host Name field appear in the Gateway column of the Account List. Use them to identify the dummy Gateway Account.

  1. Make the dummy Gateway Account the default Gateway Account as outlined in Change the Default Gateway Account.

  2. Delete the Gateway Account as outlined in Delete a Gateway Account. The bolded default dummy Gateway Account is removed from the Account List. The SecureSpan Bridge does not automatically assign default status to another Gateway Account, so the Account List no longer contains a default Gateway Account.

 

 NOTE: To re-assign default status to a Gateway Account in the Account List, select the target Gateway Account and click [Set Default] on the Main Tool Bar.
