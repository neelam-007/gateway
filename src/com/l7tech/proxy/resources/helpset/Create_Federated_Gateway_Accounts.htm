Create Federated Gateway Accounts

 NOTE: See Manage Gateway Accounts for information on changing a Gateway Account password, editing and deleting a Gateway Account, or assigning and deleting the default status applied to a Gateway Account. For information about the role of Gateway Accounts in an identity bridging-configuration, see Gateway Accounts and Identity Bridging.

A new Federated SecureSpan™ Gateway Account is configured with the SecureSpan™ Bridge’s Gateway Account Properties form. Accessed from the Create Gateway Account form when configuring a new Gateway Account, the Gateway Account Properties form configures the required and optional settings for the proxy relationship.

 

Two types of Federated Gateway Accounts can be configured in the SecureSpan Bridge:

  • The source of the SAML security tokens used to authenticate requests can come from a third-party WS-Trust server, or

  • The source of the SAML security tokens used to authenticate requests can come from an already-configured Trusted Gateway.

 

For more information about Federated Gateway Accounts, see SecureSpan™ Gateway Accounts Overview.

Add a New Federated Gateway Account

Use the SecureSpan™ Bridge’s Create Gateway Account form to establish a new Federated Gateway Account for a SecureSpan™ Gateway as follows:

 

  1. Open the Create Gateway Account form by one of two methods:

    • Click the [New] button on the Main Tool Bar or,

    • Click [File] > New from the Main Menu.

  1. The Create Gateway Account form appears. Select the Federated Gateway option.

  2. Select one of the following security options:

    • To configure the Gateway Account to obtain SAML security tokens from a Security Token Service, proceed to Use a Security Token Service

    • To configure the Gateway Account to obtain SAML security tokens from a previously-configured Trusted Gateway, proceed to Use a Trusted Gateway.

Use a Security Token Service

    1. Select the Obtain Security Tokens from a WS-Trust Security Token Service option to configure the SecureSpan™ Bridge to use the specified third-party WS-Trust token server to obtain SAML security tokens whenever a service policy from this Federated Gateway specifies that a request must include a SAML security token.

    2. Click [Create]. The Gateway Account Properties form appears.

    3. In the [General] tab of the Gateway Account Properties form, enter the URL of the SecureSpan Gateway into the Gateway Host Name field in the following format: machinename.domain.com.

 NOTE: The host name must exactly match the name of the SecureSpan™ Gateway machine assigned during the creation of the SecureSpan Gateway server certificate. To check the host name in the server certificate after it has been discovered, click the [View Server Certificate] button. Verify that the SecureSpan Gateway DNS information or SecureSpan™ Bridge hosts file information for connecting to the SecureSpan Gateway server is correct.

    1. Click the [Identity] tab. Enter the URL of the security token service into the WS-Trust URL field.

    2. In the Default Applies To field, enter the “WS-Trust Applies To” field information required by your token service. The WS-Trust request must include this information before it can issue you a security token for the Federated Gateway.

 TIP: Some token servers require the Applies To field to be set to the URL of the target service — such as “http://gateway.hostname/” — where “gateway.hostname” is the host name of the Federated Gateway.  

    1.  In the User Name field, enter the user name assigned to you by the security token service.

    2. In the Password field, enter the password assigned to you by the security token service.

    3. Click [Test Token Service]. One of the following will occur:

      • A pop-up error message notifies you that the SecureSpan Bridge could not connect to the token service. Do the following:

        1. Note the errors.

        2. Click [OK] to close the dialog box.

        3. Verify the security token service information in Steps #4–7. Make changes if required.

        4. Click [Test Token Service]. If a “Connection Established” message appears, proceed to Step #9 below. If the error message appears, contact the security token service to verify the connection information. If necessary, see Technical Support and Contact Information to contact a Technical Support Representative.

      • A “Connection Established” message appears. Proceed to Step #9 below.

 NOTE: The SecureSpan™ Gateway server certificate and, in most cases, the token service certificate, will not be discovered until they are required by the SecureSpan™ Bridge. Once discovered, the certificates will appear when the respective [View Server Certificate] and [View Token Service Certificate] buttons are clicked.

    1. Click the [Network] tab. In the Outgoing Requests to the Gateway section:

      • Optionally change the default port settings (HTTP: 8080 and HTTPS: 8443; the Gateway Uses Standards Ports option is selected by default) of the SecureSpan Gateway to custom port numbers as required by your network. To change the default ports:

        1. Select the Gateway Requires Custom Ports option.

        2. Enter the applicable port numbers into the HTTP Port and HTTPS Port fields.

      • Optionally specify one or more SecureSpan Gateway IPs (the Look Up IP Addresses in DNS option is selected by default) that the SecureSpan Bridge can use for routing service messages and accepting policies and required certificates:

 TIP: Multiple hard-coded IPs allow the SecureSpan™ Bridge to bypass the DNS when a host name does not resolve to an appropriate SecureSpan™ Gateway IP address. Manual IP configuration can also be used for failover protection, to connect to a SecureSpan Gateway whose certificate does not contain a domain name, or to connect to a SecureSpan Gateway whose certificate contains an incorrect domain name. In most cases, however, it is best to get IP information from the DNS.

        1. Select the Use the Following IP Addresses option.

        2. Click [Add]. The Add IP Address form appears.

        3. Enter the SecureSpan Gateway IP address into the IP Address field. Click [OK].

        4. The IP address appears in the field. If required, repeat Steps #2-3 to add another SecureSpan Gateway IP address.

        5. When all IP addresses have been added, select one of the following processing options from the Failover Strategy drop-down list:

          • When the Ordered Sticky with Failover processing option is selected, the SecureSpan Bridge will send service messages to the first IP in the list, and continue to do so until the IP does not respond. The SecureSpan Bridge will then proceed to the next IP in the list

          • When the Random Sticky with Failover processing option is selected, the SecureSpan Bridge will choose an IP randomly at the beginning of each session and use it for the duration of the session. If the chosen IP address fails, then the SecureSpan Bridge will select another IP at random

          • When the Pure Round Robin processing option is selected, the SecureSpan Bridge will rotate through the IP list on a request-by-request basis (round-robin) from the first IP, to the second IP, and so on. When the end of the IP list is reached, the SecureSpan Bridge will cycle back to the top and start again.

    1. Click the [Bridge Policy] tab. By default, the Use SSL by Default check box is checked. Optionally uncheck the check box to turn off the SSL download of policies to the SecureSpan Bridge.

 NOTE: If you uncheck the Use SSL by Default check box, then the initial service request will not be secured. If the target service is secure, then you may be prompted for a password to download the policy for the target service, but you may have unwittingly transmitted unsecured sensitive data as part of the initial request.  

    1. Click the [Service Policies] tab. The Services with Cached Policies table is the repository for the policies, listed by Policy Attachment Key, being applied to service messages by the SecureSpan Bridge. The Associated Policy window contains assertion-level information about a policy selected in the Services with Cached Policies table. For more information, see About Stored Policies. For information on configuring, editing, and deleting policies, see Manage Stored Policies. For information on importing or exporting policies, see Import/Export Policies.

 NOTE: If automatic policy discovery will be used in the Gateway Account, then no additional configuration in the [Service Policies] tab is required. When automated, the SecureSpan™ Gateway will automatically populate the [Services Policies] tab with any policies required by its services.

    1. Click [OK]. The SecureSpan Bridge automatically adds the new Gateway Account to the Account List with a unique identifier. The format of the identifier in the Proxy column of the Account List will be “gateway<sequential number>.” The “sequential number” part of the identifier is assigned based on the number of Gateway Accounts configured in the SecureSpan Bridge. The first configured Gateway Account, for example, is assigned a unique identifier of “gateway1” and considered the default Gateway Account. Subsequent Gateway Accounts will contain a number reflecting their configuration order and will not be assigned a default status. To change the default Gateway Account in the SecureSpan Bridge, or to remove the default status from the Account List, see Manage Gateway Accounts.

Use a Trusted Gateway

 NOTE: The relationship for the target trusted SecureSpan™ Gateway must already exist in the SecureSpan™ Bridge. See Create Trusted Gateway Accounts for information on configuring a Trusted Gateway Account.

    1. In the Create Gateway Account form, select the Obtain Security Tokens from a Previously-Created Trusted Gateway option to configure the SecureSpan™ Bridge to use a previously configured and tested Trusted Gateway to obtain SAML security tokens whenever a service policy from this Federated Gateway specifies that a request must include a SAML security token. From the drop-down list, select the target Trusted Gateway Account that will provide the security tokens.

    2. Click [Create]. The Gateway Account Properties form appears.

    3. In the [General] tab of the Gateway Account Properties form, enter the URL of the SecureSpan Gateway into the Gateway Host Name field in the following format: machinename.domain.com.

 NOTE: The host name must exactly match the name of the SecureSpan™ Gateway machine assigned during the creation of the SecureSpan Gateway server certificate.
