Manage Security Sessions

NOTE: The term "identity" includes both users and groups, the term "user" can represent an individual human or machine, the term “service” includes both Web services and XML applications, and the term "SecureSpan™ Gateway" includes all of the SecureSpan Gateway products.

As part of the security requirements for a service, the SecureSpan™ Bridge may have to establish a security session with the target SecureSpan™ Gateway. The policies that define the type of security session required between the SecureSpan Bridge and SecureSpan Gateway, if any, are developed in the SecureSpan™ Manager and saved in the target SecureSpan Gateway. See the SecureSpan™ Manager documentation for more information. The policies are automatically downloaded to the SecureSpan Bridge from the SecureSpan Gateway during the initial consumption of a service or manually imported from another location. For more information about policies in the SecureSpan Bridge, see Manage Policies.

 

An example security session is discussed in About SSL/TLS Security Sessions. Other types of transient security sessions, such as those discussed in Lifecycle of Security Sessions, are possible depending on the security requirements for a service.
