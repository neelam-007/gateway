Add and Edit Federated Identity Providers

 NOTE: The generic "Trusted Gateway A" and "Federated Gateway B" references are used in the identity bridging examples, workflows, and instructions. The Trusted Gateway A is the certificate authority (CA) that issues and manages security credentials and that is responsible for authentication. The Federated Gateway B is the Web service provider that is responsible for authorization.

The Federated Identity Provider (FIP) is an essential element when bridging disparate security domains. See Federated Identity Provider Overview for more information. The certificates that are used as the credential source in an FIP are defined and imported into the trust store using the Add Certificate Wizard (see Add and Edit Certificates) according to the source type-specific instructions provided in Identity Bridging Workflow Using an X.509 Certificate or Identity Bridging Workflow Using SAML.

 

The SecureSpan™ Manager's Create Federated Identity Provider Wizard is used to add an FIP to the Federated Gateway B. In the Wizard, you can define the type(s) of credential source(s) that the FIP will support, and when applicable, add trust store certificates. You can edit or delete an FIP from the [Identity Providers] tab at any time. For information on adding federated users, groups, and/or virtual groups to the FIP, see Add/Edit Federated Identity Provider Users and Groups.

 

 NOTE: For the SAML credential source, SAML constraints are defined in the SAML Assertion that is included in the Web service policy. For information about the SAML Assertion, see Configure and Edit Access Control Assertions in the “SecureSpan™ Manager User Manual.” Ensure that all required certificates are added to the trust store prior to creating the Federated Identity Provider. For credential source-type instructions, see Identity Bridging Workflow Using an X.509 Certificate or Identity Bridging Workflow Using SAML. For detailed certificate addition instructions, see Add and Edit Certificates.

Add a New Federated Identity Provider

 TIP: Follow the credential source-specific workflow instructions outlined in Identity Bridging Workflow Using an X.509 Certificate or Identity Bridging Workflow Using SAML, along with the following instructions, when adding a new Federated Identity Provider.

A new Federated Identity Provider (FIP) is added to the Federated Gateway B with the SecureSpan™ Manager's Create Federated Identity Provider Wizard. The Wizard contains two screens corresponding to the two main tasks in the configuration process:

  1. The Enter Provider Information screen defines the FIP name and supported credential source types.

  2. The Select the Trusted Certificates screen adds and manages the imported trust store certificates for the FIP.

 

You can view screen-specific information and instructions in the Tips Box located near the bottom of the Create Federated Identity Provider Wizard.

 

  1. Access the Create Federated Identity Provider Wizard by one of three methods:

    • Click the "Create Federated Identity Provider" link on the Home Page

    • Click [Tasks] > Create Identity Provider > Create Federated Identity Provider from the Main Menu

    • Right-click the "Identity Providers" title at the top of the [Identity Providers] tab. A drop-down menu appears. Select Create Federated Identity Provider.

  1. The Enter Provider Information screen of the Create Federated Identity Provider Wizard appears. Enter a name for the FIP into the Provider Name field.

 NOTE: The name entered in the Provider Name field will appear in the [Identity Providers] tab. The provider name must be between three and twenty-four characters long.

  1. In the Credential Source Type Allowed section, select an option corresponding to the authorization source used in the identity bridging configuration:

 NOTE: If you intend to add one or more virtual groups to the Federated Identity Provider, then you must check the X.509 Certificate check box even if you are using the SAML credential source type. See Identity Bridging Workflow Using SAML for more information.

X.509 Credential Source

If using X.509 certificates for credential authorization:

    • Check the X.509 Certificate check box

    • Click [Next]. The Select the Trusted Certificates screen appears. Proceed to Step #5 below.

 NOTE: The Configure SAML Credential Source screen only appears when you select the SAML Token check box.

SAML Credential Source

If using the SAML Security assertion for credential authorization:

    1. Check the SAML Token check box

    2. Click [Next]. The Select the Trusted Certificates screen of the Create Federated Identity Provider Wizard appears. Proceed to Step #5 below.

 

  1. In the Select the Trusted Certificates screen, click [Add]. The Search Trusted Certificates form appears. If necessary, specify any search criteria using the Subject DN or Issuer Name drop-down lists and fields.

 TIP: To view all certificates in the trust store, leave the Subject DN and Issuer Name fields blank.

  1. Click [Search]. Search results appear in the Search Results window. Select one or more certificates to add to the FIP as follows:

Add a Single Certificate

To add a certificate, select the certificate's name and click [Select]. The Search Trusted Certificates form closes and the certificate is added to the Trusted Certificates window in the Select the Trusted Certificates screen.

Add Multiple Certificates

    • Add multiple certificates by one of two methods:

      • To select a consecutive range of certificates, press [Shift] on your keyboard, left-click the first certificate in the range, and then left-click the last certificate in the range

      • To add certificates placed throughout the list, press [Ctrl] on your keyboard as you click each certificate name that you want to add.

    • Click [Select]. The Search Trusted Certificates form closes and the certificate is added to the Trusted Certificates window in the Select the Trusted Certificates screen.

 TIP: To view certificate details, select a certificate in the Trusted Certificates window and click [Properties]. The Certificate Properties form appears. For information on editing a certificate, see Add and Edit Certificates.

 

  1. In the Select the Trusted Certificates screen of the Create Federated Identity Provider Wizard, click [Finish]. The new FIP is immediately added to the [Identity Providers] tab.

 

Edit a Federated Identity Provider

A Federated Identity Provider (FIP) is edited from the Edit Federated Identity Provider Wizard accessed from the [Identity Providers] tab.

 

  • In the [Identity Providers] tab, access the Edit Federated Identity Provider Wizard for the target FIP by one of two methods:

    • Double-click the FIP name

    • Right-click the FIP name. A drop-down menu appears. Select Properties.

  • The Edit Federated Identity Provider Wizard appears. Modify as required. For screen and field-level information, see Add a New Federated Identity Provider.

 

Delete a Federated Identity Provider

 NOTE: An error message will appear in the Policy Validation Messages window if you deleted a Federated Identity Provider (FIP) whose users and groups are attached to the policy. To delete the FIP, you must first delete the FIP users and groups from all policies. See Configure and Edit Policies in the “SecureSpan™ Manager User Manual” for more information.

 

  1. In the [Identity Providers] tab, right-click the Federated Identity Provider (FIP) name that you want to delete.

  2. A drop-down menu appears. Select Delete.

  3. A confirmation message appears. Click [Yes]. The FIP is removed from the [Identity Providers] tab.
