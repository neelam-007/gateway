Add and Edit Certificates

 NOTE: The generic "Trusted Gateway A" and "Federated Gateway B" references are used in the identity bridging examples, workflows, and instructions. The Trusted Gateway A is the certificate authority (CA) that issues and manages security credentials and that is responsible for authentication. The Federated Gateway B is the Web service provider that is responsible for authorization.

The SecureSpan™ Manager allows you to import policies into the Federated Gateway B trust store. The trust store is the repository for four types of policies that may be required by the Federated Identity Provider in an identity bridging configuration:

  • CA policies used for signing client policies

  • SSL server policies

  • CA policies used for signing SSL server policies

  • Certificates used for signing SAML assertions.

 

 NOTE: Individual client policies can also be imported into the trust store for specific federated users from the Create Federated User form. See Add and Edit Federated Identity Provider Users and Groups for more information.

 

Using the Add Certificate Wizard, you can add, view, and assign one or more usage options to each certificate imported into the trust store. You can use the Manage Certificates form to view, edit, or delete trust store policies at any time.

 

 NOTE: For information on Federated Identity Providers, see Add and Edit Federated Identity Provider Users and Groups. For information on the policies required in each security domain in an identity bridging configuration, see Identity Bridging Requirements.

Add a New Certificate

 TIP: Follow the credential source-specific workflow instructions outlined in Identity Bridging Workflow Using an X.509 Certificate or Identity Bridging Workflow Using SAML, along with the following instructions, when adding a new trust store certificate.

A new certificate is added to the Federated Gateway B trust store with the SecureSpan™ Manager's Add Certificate Wizard. The Wizard contains three screens corresponding to the three main tasks in the configuration process:

  1. The Enter Certificate Info screen allows you to import policies from an HTTPS connection, file, or by copying and pasting them in PEM format.

  2. The View Certificate Details screen allows you to name trust store entries and display certificate details.

  3. The Specify Certificate Options screen allows you to define one or more usage options for the certificate.  

 

You can view screen-specific information and instructions in the Tips Box located near the bottom of the Add Certificate Wizard.

 

  1. In the SecureSpan Manager, select Tasks > Manage Certificates from the Main Menu. The Manage Certificates form appears.

  2. Click [Add]. The Enter Certificate Info screen of the Add Certificate Wizard appears. Do one of the following:

 NOTE: An error message will appear if you attempt to add an expired or duplicate certificate into the trust store.

Load the Certificate from a URL

The Retrieve via SSL Connection (HTTPS URL) option is selected by default. Do the following:

    • Enter the URL into the field

    • Click [Next]. Proceed to Step #3 below.

Import the Certificate from a File

To import the certificate from a local file:

    • Select the Import from a File option, then either:

      • Enter the file path into the field, or

      • Click [Browse] to navigate to the file location. The Open form appears. Navigate to the location of the certificate. Select the file, and then click [Open].

    • Click [Next]. Proceed to Step #3 below.

Copy and Paste the Certificate

 NOTE: You can only cut and paste a certificate that is in Base 64 PEM format. In the Code window, the SecureSpan™ Manager will only add a pasted certificate that begins with "BEGIN TRUSTED CERTIFICATE."

To copy and paste the certificate from a file into the Code window:

    • Select the Copy and Paste (Base 64 PEM) option

    • Copy and then paste the entire certificate from the originating file into the Code window

    • Click [Next]. Proceed to Step #3 below.

  1. One of the following will occur:

    • A pop-up error message notifies you that the certificate could not be added:

      1. Note the error.

      2. Click [OK] to close the dialog box.

      3. Return to the Enter Certificate Info screen. Repeat the addition method as outlined in Step #2.

      4. Click [Next].

      5. If the View Certificate Details screen of the Add Certificate Wizard appears, proceed to Step #4. If the error message appears, continue to fix the error until the Enter Certificate Info screen appears. If necessary, see Technical Support and Contact Information to contact a Technical Support Representative.

    • The View Certificate Details screen of the Add Certificate Wizard appears. Proceed to Step #4.

  1. In the View Certificate Details screen of the Add Certificate Wizard appears, enter an optional name into the Certificate Name field. View certificate details as required.

 NOTE: When creating a new Federated Identity Provider user with that has a client certificate signed by the CA root certificate, the user's "X509 Subject DN" value must match the "Issued to" value of their certificate. See Add and Edit Federated Identity Provider Users and Groups for more information.

  1. Click [Next]. The Specify Certificate Options screen of the Add Certificate Wizard appears. Select one or more usage options for the certificate:

 TIP: Follow the instructions in the appropriate credential source-specific workflow to determine the proper usage options for the certificate. See Identity Bridging Workflow Using  an X.509 Certificate or Identity Bridging Workflow Using SAML.

Outbound SSL Connections

Check the Outbound SSL Connections check box when the imported certificate belongs to an SSL server hosting the protected Web services.

 NOTE: The HTTP(S) Routing assertion in the Web service policy will not be able to connect using SSL if the host server contains self-signed policies or policies that are signed by an untrusted CA (certificate authority) unless the server's certificate (or the CA certificate that was used to sign it) is imported with this certificate usage option. See the Signing Client Certificates usage option below for more information. For more information about the HTTP(S) Routing assertion, see Configure and Edit Message Routing Assertions.

Signing Certificates for Outbound SSL Connections

Check the Signing Certificates for Outbound SSL Connections check box when the imported certificate is the signing certificate of a CA (certificate authority) that signs SSL policies for servers hosting protected Web services.

 NOTE: The HTTP(S) Routing assertion in the Web service policy will not be able to connect using SSL if the host server contains self-signed policies or policies that are signed by an untrusted CA (certificate authority) unless the server's certificate (or the CA certificate that was used to sign it) is imported with this certificate usage option. See the Signing Client Certificates usage option below for more information. For more information about the HTTP(S) Routing assertion, see Configure and Edit Message Routing Assertions.

Signing Client Certificates

Check the Signing Client Certificates check box when the imported certificate is the signing certificate of a CA that signs client policies. The client policies can be used for a variety of purposes, including SSL client authentication and XML signing and encryption (see Configure and Edit XML Security Assertions). Certificates imported with this option enabled can be used in Federated Identity Providers.

Signing SAML Tokens

Check the Signing SAML Tokens check box when the imported certificate is the signing certificate of a SAML issuing authority, such as in the SAML credential source workflow (see Identity Bridging Workflow Using SAML). Certificates imported with this option enabled can be used in Federated Identity Providers.

 TIP: See the Tips Box located near the bottom of the Add Certificate Wizard for additional option-specific information.

  1. Click [Finish]. The Add Certificate Wizard closes and the certificate is added to the Trusted Certificates window in the Manage Certificates form.

 

Edit a Certificate

  1. In the SecureSpan™ Manager, select Tasks > Manage Certificates from the Main Menu. The Manage Certificates form appears.

  2. Select the certificate you want to edit. Click [Properties]. The Certificates Properties form appears. Edit the certificate name and usage options as required. Reference the instructions in Add a New Certificate.

  3. Click [Save]. The modified certificate is updated in the trust store and Manage Certificates form. Click [Close] to close the Manage Certificates form.

 

Delete a Certificate

  1. In the SecureSpan™ Manager, select Tasks > Manage Certificates from the Main Menu. The Manage Certificates form appears.

  2. Select the certificate that you want to delete. Click [Remove].

  3. A confirmation message appears. Click [Remove]. The certificate is removed from the trust store and Manage Certificates form. Click [Close] to close the Manage Certificates form.
