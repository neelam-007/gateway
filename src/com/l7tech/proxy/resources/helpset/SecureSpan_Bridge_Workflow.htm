SecureSpan™ Bridge Workflow

 NOTE: This workflow assumes a fresh installation of the SecureSpan™ Bridge (empty database) in a non-identity bridging-configuration. For information on installing and configuring the SecureSpan Bridge, see the “SecureSpan™ Bridge and SecureSpan™ Manager Installation Manual.” For information on the steps required to configure identity bridging in the SecureSpan Bridge and SecureSpan™ Manager, see Identity Bridging Overview. For assistance in the configuration process, contact a Technical Support Representative as outlined in Technical Support and Contact Information.

The simplest deployment of the SecureSpan™ Bridge utilizes the software as a dynamic policy enforcement point for client applications accessing SecureSpan™ Gateway-protected services. The following workflow provides the general configuration steps for this scenario. For information on configuring the SecureSpan Bridge for additional scenarios, including scenarios that require multiple Gateway Accounts, contact your Professional Services Specialist or a Technical Support Representative as outlined in Technical Support and Contact Information.

General Workflow:

  1. The Create Gateway Account form automatically appears the first time that you launch the SecureSpan™ Bridge. To create the Trusted Gateway Account for the SecureSpan Gateway protecting the target service, see Create Trusted Gateway Accounts.

 NOTE: The SecureSpan™ Bridge automatically sets the first configured Gateway Account as the default account in the Account List. See Manage Gateway Accounts for information on re-assigning or removing the default Gateway Account status.

  1. Configure your client application to send request messages to “http://localhost:7700/”, the service URL of the SecureSpan Bridge. For more information on sending request messages to the SecureSpan Bridge, see Configure Client Applications.

  2. Generate a service request through your client application. The request will be transmitted to the SecureSpan Bridge, which will route it to the target SecureSpan Gateway. The SecureSpan Bridge will route the service response message to the client application. For more information about the processing and routing of service messages in the SecureSpan Bridge, see About Stored Policies and About Message Routing.

 

 NOTE: For information on the steps required to configure identity bridging in the SecureSpan™ Manager and SecureSpan™ Bridge, see Identity Bridging Overview.
