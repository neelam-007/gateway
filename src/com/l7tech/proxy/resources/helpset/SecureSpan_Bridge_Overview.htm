SecureSpan™ Bridge Overview

 NOTE: The term "identity" includes both users and groups, the term "user" can represent an individual human or machine, the term “service” includes both Web services and XML applications, and the term "SecureSpan™ Gateway" includes all of the SecureSpan Gateway products.

The SecureSpan™ Bridge interfaces with client-side applications and automatically negotiates policy-specific security and transaction preferences with the SecureSpan™ Gateway in real-time. Client systems send message requests to the SecureSpan Bridge. The SecureSpan Bridge then functions as a client-side proxy, applying necessary protocols, headers, or transformations to the message as required by the policy in force. Policies modified through the SecureSpan™ Manager are automatically applied in real-time by the SecureSpan Bridge to ensure that all subsequent messages conform to the updated policy. The SecureSpan Bridge ensures rigorous, fine-grained security with automated change control end-to-end across even the simplest service integrations, essentially ensuring loose-coupling between services and their client applications so that service policy changes do not automatically break integrations.

 

The SecureSpan Bridge also enables the bridging of identities that reside in disparate security or identity domains. These domains can be associated with two departments or divisions of the same company or two entirely separate business partners. Configured in conjunction with the SecureSpan Manager, identity bridging is a unique and powerful model that separates authentication and authorization tasks that occur between security domains, delegating authentication to the requesting domain while preserving control over authorization for the provider hosting the Web service. The SecureSpan Bridge facilitates identity bridging by two basic mechanisms. First, the SecureSpan Bridge discovers the policy for a request to determine if a security token will be required by the service provider. Then, if a token is required, the SecureSpan Bridge obtains a security token from a local security token service and binds it into the request. For more information about identity bridging, see Identity Bridging Overview.

 

The Microsoft Windows version of the SecureSpan Bridge offers two modes: 1) a GUI for all configuration and management tasks (see SecureSpan™ Bridge Interface) and 2) a Troubleshooting Mode used by Network Administrators to gather additional information about system errors. A separate Troubleshooting Mode for the Linux version of the SecureSpan Bridge is not available. See SecureSpan™ Bridge Troubleshooting Mode for more information. The SecureSpan Bridge can also be run in daemon mode, or as a non-interactive background process on a Linux or Windows machine. See SecureSpan™ Bridge in Daemon Mode for more information.
