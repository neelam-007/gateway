Manage Policies

NOTE: The term "identity" includes both users and groups, the term "user" can represent an individual human or machine, the term “service” includes both Web services and XML applications, and the term "SecureSpan™ Gateway" includes all of the SecureSpan Gateway products.

The core function of the SecureSpan™ Bridge is its ability to decorate and route service messages to and from a SecureSpan™ Gateway. When the SecureSpan Bridge receives a request message from a client application, it decorates the message according to the security requirements specified in the service’s policy before routing the message to the target SecureSpan Gateway for processing. Upon receiving a response from the SecureSpan Gateway, the SecureSpan Bridge un-decorates and routes the service response message back to the client application. For information about the types of assertions in a SecureSpan Bridge policy, see About Stored Policies. For information about message routing, see Configure Message Routing. For information about sending request messages to the SecureSpan Bridge, see Configure Client Applications.

 

In the SecureSpan Bridge, policies are viewed in the [Service Policies] tab in the Gateway Account Properties form. Policies can be automatically downloaded from the SecureSpan Gateway during the initial consumption of a service, or manually imported from another location and stored as a partial policy. Individual stored policies can be locked, unlocked, configured, imported, exported, and deleted, and all cached policies stored for a Gateway Account can be purged. See Manage Stored Policies and Import/Export Policies for more information.

 

As part of the security requirements for a service, the SecureSpan Bridge may have to establish security session with the target SecureSpan Gateway. For more information about policy-specified security sessions between the SecureSpan Bridge and SecureSpan Gateway, see About SSL/TLS Security Sessions.
