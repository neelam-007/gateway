Change the WSS Assertion Recipient

The default recipient for the following message-level or WS-Security (WSS) assertions can be changed in the SecureSpan™ Manager:

  • The WSS UsernameToken Basic assertion, WSS Signature assertion, and SAML Assertion from the Access Control category of the [Policy Assertions] tab

  • The Sign Request Element, Encrypt Request Element, Sign Response Element, and Encrypt Response Element assertions from the XML Security category of the [Policy Assertions] tab.

 

When a WSS assertion is added to a policy, the associated WSS decoration will be written in the Security header, intended by default for the "next-in-line" recipient (the SecureSpan™ Gateway) in the SOAP message. Changing the recipient of a WSS assertion configures a different downstream recipient, with a unique Actor attribute value, for the WSS decoration, essentially by-passing the SecureSpan Gateway. A policy can contain multiple alternate recipients, each resulting in a separate Security SOAP header with its associated unique Actor attribute.

 

WARNING: If the intended recipient does not accept or recognize Security headers that contain Actor attributes, then you must configure the HTTP(S) Routing assertion (see Configure/Edit Message Routing Assertions) in the policy to instruct the SecureSpan Gateway to promote one of the downstream WSS recipients as the next default WSS header. To do so, select the "Promote other Security header as default before routing" option in the WSS Header Handling section of the HTTP(S) Routing Properties form.

 

Access and configure the Change WSS Recipient form for an individual WSS assertion as follows:

 

NOTE: This section discusses how to access and configure the Change WSS Recipient form that is common for all of the WSS assertions mentioned above. See the category topic pages for configuration instructions for each of the individual WSS assertions. For the WSS UsernameToken Basic, WSS Signature, and SAML Assertion assertions, see Configure and Edit the Access Control Assertions. For the Sign Request Element, Encrypt Request Element, Sign Response Element, and Encrypt Response Element assertions, see Configure and Edit the XML Security Assertions.

 

  1. Right-click a WSS assertion in the Policy Development window. Select WSS Recipient from the drop-down menu.

  2. The Change WSS Recipient form appears. The current SecureSpan Gateway is selected as the default target recipient for the WSS decoration. To change the recipient, select the Specific Recipient option.

  3. Do one of the following:

Choose Existing Recipient

If the intended downstream recipient was already defined for another WSS assertion in the policy, then:

    1. Select the corresponding recipient's Actor attribute value from the Security Header "Actor" Attribute drop-down list.  

    2. The certificate subject information for the previously configured target appears in the Recipient Certificate Subject field.

    3. Click [OK]. The recipient's Actor attribute value appears as an extension of the WSS assertion's name in the Policy Development window.

 

NOTE: To view or edit the recipient for a WSS assertion, right-click the assertion, select WSS Recipient from the drop-down menu, and view or edit the target recipient in the Change WSS Recipient form.

 

Add New Recipient

NOTE: To create a new downstream recipient, you will need access to the recipient's certificate.

Configure a new recipient for use by the WSS assertions as follows:

    1. Click [Add Recipient]. The Add WSS Recipient Wizard appears. Add the certificate for the new recipient by one of the following methods:

NOTE: An error message will appear if you attempt to add an expired or duplicate certificate to the SecureSpan™ Gateway.

Load the Certificate from a URL

The Retrieve via SSL Connection (HTTPS URL) option is selected by default. Do the following:

      • Enter the URL into the field

      • Click [Next]. Proceed to Step #2 below.

Import the Certificate from a File

To import the certificate from a local file:

      • Select the Import from a File option, then either:

        • Enter the file path into the field, or

        • Click [Browse] to navigate to the file location. The Open form appears. Navigate to the location of the certificate. Select the file, and then click [Open].

      • Click [Next]. Proceed to Step #2 below.

Copy and Paste the Certificate

NOTE: You can only cut and paste a certificate that is in Base 64 PEM format. In the Code window, the SecureSpan™ Manager will only add a pasted certificate that begins with "BEGIN TRUSTED CERTIFICATE."

To copy and paste the certificate from a file into the Code window:

      • Select the Copy and Paste (Base 64 PEM) option

      • Copy and then paste the entire certificate from the originating file into the Code window

      • Click [Next]. Proceed to Step #2 below.

    1. One of the following will occur:

      • A pop-up error message notifies you that the certificate could not be added:

        1. Note the error.

        2. Click [OK] to close the dialog box.

        3. Return to the Enter Certificate Info screen. Repeat the addition method as outlined in Step #2.

        4. Click [Next].

        5. If the View Certificate Details screen of the Add Certificate Wizard appears, proceed to Step #3. If the error message appears, continue to fix the error until the Enter Certificate Info screen appears. If necessary, see Technical Support to contact a Technical Support Representative.

      • The View Certificate Details screen of the Add Certificate Wizard appears. Proceed to Step #3.

    2. In the View Certificate Details screen of the Add Certificate Wizard appears, enter an optional name into the Certificate Name field. View certificate details as required.

    3. Click [Next]. The Associate Actor Attribute screen appears. Enter a unique Actor attribute for the recipient certificate into the Actor Attribute Value field.

NOTE: Since a recipient is identified in the Change WSS Recipient form by its associated Actor attribute, ensure that the Actor attribute that you enter is unique. In the SecureSpan™ Manager, the certificate and Actor attribute are a locked combination that can be used multiple times by multiple WSS assertions.

    1. Click [Finish]. The Change WSS Recipient form appears with the Actor attribute and certificate subject information in the respective Security Header "Actor" Attribute drop-down list and Recipient Certificate Subject field.

    2. Click [OK]. The recipient's Actor attribute value appears as an extension of the WSS assertion's name in the Policy Development window.

 

NOTE: To view or edit the recipient for a WSS assertion, right-click the assertion, select WSS Recipient from the drop-down menu, and view or edit the target recipient in the Change WSS Recipient form.
