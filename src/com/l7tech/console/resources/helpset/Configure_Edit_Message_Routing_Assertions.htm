Configure and Edit Message Routing Assertions

NOTE: The JMS Routing assertion in the Message Routing category of the [Assertions] tab is part of the JMS functionality add-on available for the SecureSpan™ Manager. It is disabled in regular versions. Contact a Technical Support Representative as outlined in Technical Support and Contact Information for more information.

In the SecureSpan™ Manager, two assertions are available in the Message Routing category of the [Assertions] tab: A custom JMS Routing assertion that is addressed in the SecureSpan™ Solution Supplementary Guide: Installing, Configuring, and Using JMS PDF, and an HTTP(S) Routing assertion discussed below:

HTTP(S) Routing

The HTTP(S) Routing assertion defines where a Web service message is sent and what authentication credentials it uses. A message routing assertion is an essential security policy element. When you publish a Web service using the Publish Web Service Wizard or Create WSDL Wizard, the SecureSpan™ Manager automatically adds the Web service URL specified during the publication process as an HTTP(S) Routing assertion in the published Web service's initial security policy. For information on publishing a Web service, see Publish Web Services. For information on creating WSDL for a new Web service, see Create Web Service WSDL.

Configure the HTTP(S) Routing Assertion

NOTE: The HTTP(S) Routing assertion should already be present in the Policy Development window. If not, then add the assertion by one of the following methods: 1. Drag and drop the assertion from the [Assertions] tab into the Policy Development window or, 2. Highlight the assertion in the [Assertions] tab and click the  (Move Assertion) button in the Assertion Tool Bar.

  1. In the Policy Development window, right-click the "Route to http" assertion. A drop-down menu appears. Select Routing Properties.

TIP: You can also double-click the assertion in the Policy Development window to open the HTTP Routing Properties form.   

  1. The HTTP Routing Properties form appears. If required, edit the URL path to the Web service in the Service URL field.

TIP: Click [Default] to populate the Service URL field with the URL specified during the Web service publication process.

  1. In the Web Service Authentication section of the form, optionally configure one of the following authentication options for the Web service:

NOTE: No credentials are required for an anonymous Web service.

Identity Authentication

NOTE: For more information on authentication assertions, see Configure and Edit Authentication Assertions.

To require authentication through identity information:

    1. Select the Identity Authentication option.

    2. Enter the user name into the User Name field.

    3. Enter the user password into the Password field.

    4. Enter the authentication realm into the Realm field. The realm can be a secure virtual host or a directory.

    5. Proceed to Step #4.

SAML Security

NOTE: For more information on SAML Security, see Configure and Edit XML Security Assertions.

To require SAML authentication:

    1. Select the SAML Security option.

    2. Optional: Enter or select the time limit for the SAML ticket in the Ticket Expiry (in minutes) field.

    3. Optional: Check the Group Membership Statement check box to require a group membership statement in the SAML ticket.

NOTE: A group membership statement can be used to perform fine grained security decisions based on subject group membership.

    1. Proceed to Step #4.

  1. Click [OK]. The configured assertion is added to the Policy Development window.

Edit the HTTP(S) Routing Assertion

  1. In the Policy Development window, right-click the "Route to http" assertion. A drop-down menu appears. Select Routing Properties.

TIP: You can also double-click the assertion in the Policy Development window to open the HTTP Routing Properties form.   

  1. The HTTP Routing Properties form appears. Edit the properties as outlined in Configure the HTTP(S) Routing Assertion.

  2. Click [OK]. The modified assertion is added to the Policy Development window.
