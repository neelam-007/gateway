Save and Use Policy Templates

The SecureSpan™ Manager allows you to save the assertions for a security policy as a template in the Policy Templates section of the [Assertions] tab. The collection of assertions in a policy template can be imported, or applied, to another security policy to ensure consistency and save configuration time.

Save a Policy as a Template

NOTE: Ensure that the security policy is complete and valid before you save it as a template. See Configure and Edit Security Policies for information.

  1. If necessary, load the target security policy into the Policy Development window by one of the following methods:

    • Right-click the Web service name in the [Web Services] tab. A drop-down menu appears. Select Policy Assertions. Or,

    • Double-click the Web service in the [Web Services] tab.

  1. The Policy Development window appears. If not present, enable the Policy Validation Messages window by selecting [View] > Policy Messages from the Main Menu (check box checked).

NOTE: The Policy Validation Messages window provides you with policy and assertion-level confirmation, warning, and/or error messages that you can use to determine the validity of the security policy. See Configure and Edit Security Policies for more information.

  1. The Policy Validation Messages window appears under the Policy Development window. Note any assertion-level messages. If necessary, edit the security policy as outlined in Configure and Edit Security Policies.

  2. In the Policy Tool Bar, click [Validate]. One of the following will occur:

    • A "Policy validated ok" message indicates that the policy is error free. Proceed to Step #5

    • One or more error messages indicates that the policy contains errors. Do the following:

      • Re-organize the policy assertions so that they conform to the policy and assertion organization rules outlined in Security Policies Overview and Rules

      • Ensure that the individual assertions are configured properly. See the assertion specific instructions outlined in the category pages discussed in Security Policy Assertions Overview.

      • If validation errors still occur, contact a Technical Support Representative as outlined in Technical Support and Contact Information.

NOTE: Do not save a security policy as a template unless it is error free.

  1. In the Policy Tool Bar, click [Save as Template].

  2. A Save File form appears. Enter a descriptive name for the template and click [Save]. The template appears in the Policy Templates section of the [Assertions] tab.

NOTE: By default, the SecureSpan™ Manager saves the policy template as an XML document in a local Manager directory created during installation. Do not manually edit the policy template. See Edit a Policy Template for more information. You can only access a policy template from the machine that it was created on.

Use a Policy Template

NOTE: When you apply a policy template to a security policy, the target policy in the Policy Development window is completely replaced by the policy template. The unique routing assertion(s) for the Web service security policy must be re-configured to reflect the unique Web service URL and other routing data. Ensure that you have the Web service-specific routing information on hand for re-configuring the replaced routing assertion(s).

When a policy is saved as a template in the SecureSpan™ Manager, all of the routing, authentication, and credential assertions are saved as an assertion tree in a local XML file. The template is accessed from the Policy Templates section of the [Assertions] tab and loaded into the Policy Development window to serve as the security policy for the Web service selected in the [Web Services] tab.

  1. If necessary, load the target Policy Development window by one of the following methods:

    • Right-click the Web service in the [Web Services] tab. A drop-down menu appears. Select Policy Assertions. Or,

    • Double-click the Web service in the [Web Services] tab.

  1. The Policy Development window appears. Move the policy template from the Policy Templates section of the [Assertions] tab into the Policy Development window by one of the following methods:

NOTE: The selected policy template will completely overwrite the policy assertions in the Policy Development window, including the Web service-specific routing assertion(s).

    • Drag and drop the selected template from the [Assertions] tab into the Policy Development window, or  

    • Select the template in the [Assertions] tab and click the  (Move Assertion) button in the Assertion Tool Bar.

  1. The policy template assertions appear in the Policy Development window. See Configure and Edit Message Routing Assertions to re-configured any replaced routing assertion(s) with Web service-specific information. See Security Policy Assertions Overview to re-configure other assertions as required.

Edit a Policy Template

NOTE: You cannot directly edit a policy template in the SecureSpan™ Manager.

The SecureSpan™ Manager allows you to rename a policy template that appears in the Policy Templates section of the [Assertions] tab. Although you cannot directly edit the assertions in a policy template, you can overwrite an existing template with a new assertion grouping. See Overwrite a Template below for more information.

Rename a Template

  1. In the Policy Templates section of the [Assertions] tab, right-click the target template name. A drop-down menu appears. Select Rename Template.

  2. The Rename Policy Template form appears. Enter the new template name into the New Name field.

  3. Click [OK]. The template name appears in the Policy Templates section of the [Assertions] tab.

Overwrite a Template

  1. Load the security policy template into the Policy Development window for a new or dummy Web service as outlined in Use a Policy Template.

  2. Edit the policy template assertions and/or organization as outlined in Configure and Edit Security Policies.

  3. Save the policy template as outlined in Save a Policy as a Template. In the Save File form in Step #6, overwrite the existing template file with the new template by entering the same template name.

  4. Click [Save]. The modified template appears in the Policy Templates section of the [Assertions] tab.

NOTE: Overwriting a policy template will not affect the security policies that the previous version has been applied to.

Delete a Policy Template

Delete a policy template from the Policy Templates section of the [Assertions] tab as follows:

  1. In the Policy Templates section of the [Assertions] tab, right-click the target template name. A drop-down menu appears. Select Delete Template.

  2. A confirmation message appears. Click [Yes]. The template is immediately removed from the Policy Templates section of the [Assertions] tab.
