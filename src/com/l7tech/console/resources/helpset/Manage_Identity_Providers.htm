Manage Identity Providers

 NOTE: The term "identity" includes both users and groups, the term "user" can represent an individual human or machine, the term “service” includes both Web services and XML applications, and the term "SecureSpan™ Gateway" includes all of the SecureSpan Gateway products.

Three types of identity providers are used in the SecureSpan™ Manager: LDAP Identity Providers, Federated Identity Providers, and an Internal Identity Provider.

LDAP Identity Providers

Users can configure and manage one or more LDAP Identity Providers (LIPs). An LIP is an LDAP (Lightweight Directory Access Protocol) connector that is used for authentication purposes. For information on configuring an LIP, see Add and Edit LDAP Identity Providers.

 

Federated Identity Providers

 NOTE: The documentation pertaining to Federated Identity Providers is discussed in Manage Federated Identity Providers in the Identity Bridging section of this Help System.

A Federated Identity Provider (FIP) is exclusively used in a SecureSpan™ identity bridging-configuration. Essentially, the FIP allows one security domain to authorize requests containing credentials originating from another security domain. For more information about the role of FIPs in identity bridging, see Manage Federated Identity Providers. For more information about identity bridging, see Identity Bridging.

 

Internal Identity Provider

A single Internal Identity Provider (IIP) is pre-configured as the authentication database inside the SecureSpan™ Gateway. The SecureSpan™ Manager allows you to modify the users and groups in the IIP. For information on adding users and/or groups to the IIP, see Add and Edit Internal Identity Provider Users and Groups.

 

Searching Identity Providers

The Search Identity Providers form permits LDAP Identity Provider, Federated Identity Provider, and Internal Identity Provider searches. See Search Identity Providers for more information.
