Manage Policies

 NOTE: The term "identity" includes both users and groups, the term "user" can represent an individual human or machine, the term “service” includes both Web services and XML applications, and the term "SecureSpan™ Gateway" includes all of the SecureSpan Gateway products.

The core function of the SecureSpan™ Manager is its ability to centrally define, provision, monitor, and audit security and integration policies for Web services and XML applications, collectively referred to as "services."

 

After a service is published using the Publish SOAP Web Service Wizard (see Manage SOAP Web Services), Create WSDL Wizard (see Manage SOAP Web Services), or Publish XML Application Wizard (see Manage XML Applications), the service appears in the [Services] tab and an initial policy is created in the Policy Development window. If a published Web service WSDL document contained at least one HTTP(S) binding, then the initial policy will include an HTTP(S) Routing assertion pre-configured to point to the HTTP(S) binding URL. Before a service is adequately protected, additional policy assertions may need to be defined and configured into a policy that is saved in the SecureSpan™ Gateway.

 

 NOTE: If an authentication method and authorization to specific users and/or groups was defined during the Web service or XML application publication process, then service access will be restricted to those identities in the initial policy. The initial policy will automatically be populated with a Grant Access to Users/Groups authentication assertion for each selected identity. See Configure and Edit Access Control Assertions for more information.

 

A policy defines restrictions for the consumption of a published SecureSpan Gateway-protected service. For more information on policies and assertions, see Policy Organization. For information on the interface areas related to services and policies, see Interfaces. For information on using and configuring the policy assertions in the [Assertions] tab, see About Policy Assertions. For information on configuring a policy in the Policy Development window, see Configure and Edit Policies.
