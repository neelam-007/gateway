Using the TAM Authentication Assertion

 NOTE: The Administrator is responsible for installing and configuring the TAM (Tivoli Access Manager) Custom Assertion package in the SecureSpan™ Gateway. If you encounter authentication errors during the execution of a policy, direct the Administrator to the Troubleshoot TAM Assertion Errors section below.

Installing and configuring the TAM (Tivoli Access Manager) Custom Assertion package in the SecureSpan™ Gateway installs and enables the TAM assertion in the SecureSpan™ Manager. The TAM assertion — called "Tivoli Access Manager Assertion” in the [Assertions] tab and Policy Development window — instructs the SecureSpan Gateway to delegate the authentication and authorization tasks required to gain access to a protected service to the IBM® Tivoli® Access Manager (versions 4.1, 5.0, and 5.1) server.

TAM Assertion Rules

Note the following rules when using the TAM (Tivoli Access Manager) assertion:

  • The TAM assertion cannot be used with:

    • Authentication assertions that encrypt passwords, such as the HTTP Digest and SSL with Client Certificate Authentication assertions from the Access Control category of the [Assertions] tab

    • The Sign Request Elements and Encrypt Request Elements assertions located in the XML Security category of the [Assertions] tab

    • The Grant Access to Users/Groups assertion located in the Access Control category of the [Assertions] tab.

  • The TAM assertion can be used with:

    • The HTTP Basic assertion from the Access Control category of the [Assertions] tab

    • The SSL or TLS Transport assertion from the Transport Layer Security (TLS) category of the [Assertions] tab

    • Any other assertion not listed in the above list.

  • A policy can only contain a single TAM assertion per authentication scheme. More than one TAM assertion may appear in a complex policy that contains more than one authentication scheme

  • In a policy, the TAM assertion must appear before the routing assertion and after the SSL or TLS Transport and authentication method assertions.

 

 NOTE: For information on the general policy and assertion rules used in the SecureSpan™ Manager, see Policy Organization and Rules. For information on configuring, editing, and manipulating a policy in the Policy Development window, see Configure and Edit Policies.

 

Configure and Edit the TAM Assertion

 

Add and Configure the TAM Assertion

  1. Access the Policy Development window for the target service by one of two methods:

 TIP: If not already disabled, it is strongly recommended that you disable a service while constructing its policy to prevent unsecured and unauthorized access. A disabled service contains a red "X" through its left-side icon. See Manage Published Services for information.

    • Right-click the service name in the [Services] tab. A drop-down menu appears. Select Policy Assertions. Or,

    • Double-click the service in the [Services] tab.

  1. The Policy Development window appears. Add the "Tivoli Access Manager Assertion" assertion to the policy by one of two methods:

    • Drag and drop the assertion from the [Assertions] tab into the Policy Development window, or   

    • Highlight the assertion in the [Assertions] tab and click the  (Add Assertion) button in the Assertion Tool Bar.

  1. The assertion is added to the Policy Development window. Right-click the assertion. A drop-down menu appears. Select Custom Assertion Properties.

  1. The Tivoli Access Manager Assertion form appears. Fill in the form as follows:

 NOTE: The action and resource values are determined by the TAM (Tivoli Access Manager) settings used by the SecureSpan™ Gateway. The action value is taken from a list of allowable actions defined in the permission setting of the TAM Access Control List (ACL), and the resource value is the resource specified in the path in the configured TAM object space. Consult your Administrator for information about the action and resource properties.

    • In the Value field of the action row, enter the action (such as “POST” or “GET”) for the protected resource specified in the resource row below

    • In the Value field of the resource row, enter the resource that is being protected by the TAM (Tivoli Access Manager) server.

  1. Click [OK]. The modified assertion is added to the Policy Development window. Proceed to the "Validate a Policy" section in Configure and Edit Policies.

Edit the TAM Assertion

  1. In the Policy Development window, right-click the "Tivoli Access Manager Assertion" assertion. A drop-down menu appears. Select Custom Assertion Properties.

  1. The Tivoli Access Manager Assertion form appears. Modify the assertion properties as outlined in Add and Configure the TAM Assertion.

  2. Click [OK]. The modified assertion is added to the Policy Development window. Proceed to the "Validate a Policy" section in Configure and Edit Policies.

 

Troubleshoot TAM Assertion Errors

 NOTE: Contact your Administrator if you encounter authentication errors.

If configuration errors exist in the IBM® Tivoli Access Manager (TAM) server or the SecureSpan™ Gateway, then one of the following error messages will appear in the SecureSpan™ Manager Gateway Audit Events window when the TAM assertion is used in a policy. For information about the Gateway Audit Events window, see View Gateway Audit Events.

  • "SEVERE: Not init or failed"

This error message appears in the Gateway Audit Events window when:

    • The TAM server is down

    • The TAM process is not running

    • The SecureSpan Gateway is not properly configured to connect to the TAM server.

Verify the SecureSpan Gateway and TAM server connection settings. To contact a Layer 7 Technologies Technical Support Representative, see Technical Support and Contact Information

  • "WARNING: Authorization (access control) failed"

This error message appears in the Gateway Audit Events window when the SecureSpan Gateway connection credentials are not authenticated or authorized by the TAM server. A Log on to Gateway form prompts you to re-enter your user name and/or password. Ensure that the user name and password entered in SecureSpan™ Bridge match those configured in the user database used by the TAM server to authenticate and authorize users. To contact a Layer 7 Technologies Technical Support Representative, see Technical Support and Contact Information.
