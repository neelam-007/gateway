Identity Bridging Requirements

 NOTE: The generic "Trusted Gateway A" and "Federated Gateway B" references are used in the identity bridging examples, workflows, and instructions. The Trusted Gateway A is the certificate authority (CA) that issues and manages security credentials and that is responsible for authentication. The Federated Gateway B is the Web service provider that is responsible for authorization.

The following items are required to configure identity bridging using a SAML or X.509 certificate credential source:

Web Service Requestor-Side Requirements

  • SecureSpan™ Bridge installed and configured

  • When SAML is used as the credential source in an identity bridging configuration, then the "Full Identity Bridging" method must be used to create two proxy entries (trusted and federated) in the SecureSpan™ Bridge. See Identity Bridging Workflow Using SAML for more information

  • When an X.509 certificate is used as the credential source in an identity bridging configuration that consists of two SecureSpan™ Gateways, then the Bridge is configured by one of two methods: Either two proxy entries (trusted and federated) are created using the "Full Identity Bridging" method, or a single proxy entry is created (trusted) and a PKCS#12 keystore certificate is imported using the "Ad-Hoc Identity Bridging" method. See Identity Bridging Workflow Using an X.509 Certificate.

Trusted Gateway A (Authentication Domain) Requirements

  • SecureSpan™ Gateway installed and configured

 NOTE: X.509 credentials do not have to originate from a SecureSpan™ Gateway. Any certificate authority (CA) that issues certificates (such as a system employing the "OpenSSL" toolkit) can sign client certificates. When SAML is used as a credential source in an identity bridging configuration, however, then the Trusted Gateway A must be another SecureSpan™ Gateway. See Identity Bridging Workflow Using SAML for more information.

  • One CA certificate (To be imported into the Federated Gateway B trust store when using an X.509 certificate credential source. For more information, see Identity Bridging Workflow Using an X.509 Certificate)

  • One server certificate (To be imported into the Federated Gateway B trust store when using the SAML credential source. For more information, see Identity Bridging Workflow Using SAML)

  • Individual client certificates signed by the CA certificate (May be imported into the Federated Gateway B trust store for the Federated Identity Provider for additional confidence in authorizing identities by users’ X.509 client certificates. For more information, see Identity Bridging Workflow Using an X.509 Certificate).

Federated Gateway B (Authorization Domain) Requirements

  • SecureSpan™ Gateway installed and configured

  • SecureSpan™ Manager installed and configured

  • CA certificate(s) imported from trusted credential source(s) (Imported from the Trusted Gateway A when configuring an X.509 certificate credential source. For more information, see Identity Bridging Workflow Using an X.509 Certificate)

  • Imported server certificate(s) imported from trusted credential source(s) (Imported from the Trusted Gateway A when configuring a SAML credential source. For more information, see Identity Bridging Workflow Using SAML)

  • Individual client certificates (Optionally imported for the federated users. For more information, see Identity Bridging Workflow Using an X.509 Certificate).
