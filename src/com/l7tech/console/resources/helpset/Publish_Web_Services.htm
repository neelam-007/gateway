Publish Web Services

NOTE: You cannot edit or replace the WSDL for a published Web service. To change the WSDL for a Web service, you must delete the Web service as outlined in Manage Published Web Services and then re-publish the Web service with the Publish Web Service Wizard.

The SecureSpan™ Manager guides you through the Web service publication process. Using the Publish Web Service Wizard, you can easily enter the location, access credentials, and SecureSpan™ Gateway access option for a Web service. Publication adds a Web service to the Manager's [Web Services] tab, establishes the Web service's initial security policy in the Policy Development window, and allows authorized clients to access the Web service through the Gateway. For more information about the interface areas related to Web services and security policies, see SecureSpan™ Manager Interface. For information on disabling/enabling, renaming, or deleting a published Web service, see Manage Published Web Services.

 

In the SecureSpan™ Manager, a Web service is published with the Publish Web Service Wizard. The Wizard contains three screens corresponding to the three main tasks in the configuration process:

  1. The WSDL URL screen locates the WSDL for the Web service.

  2. The Access Control screen allows you to define simple Web service encryption, access control, and authentication rules.

  3. The Credential Transportation screen specifies how the SecureSpan™ Gateway can gain access to the Web service.

 

NOTE: Ensure that you have all required Web service information before you use the Publish Web Service Wizard.

  1. Access the Publish Web Service Wizard by one of three methods:

    • Click the "Publish Web Service" link on the Home Page

NOTE: The Home Page appears upon startup or when the Main Tool Bar's [Home] button is clicked.

    • Click [Tasks] > Publish Web Service from the Main Menu

    • Right-click the "Web services @ https://servername:port" title in the [Web Services] tab. A drop-down menu appears. Select Publish Web Service.

  1. The WSDL URL screen of the Publish Web Service Wizard appears. In the URL field, enter the Web service WSDL URL in the following format: http://services.example.com/warehouse/purchase.wsdl.

  1. Click [Next]. One of the following will occur:

    1. A pop-up error message notifies you that the WSDL URL could not be resolved by the Publish Web Service Wizard. Do the following:

      1. Note the errors.

      2. Click [OK] to close the dialog box.

      3. Re-enter the WSDL URL in the URL field.

      4. Click [Next].

      5. If the Access Control screen of the Publish Web Service Wizard appears, proceed to Step #4. If the error message appears, continue to fix and resolve the URL until the Access Control screen of the Publish Web Service Wizard appears. If necessary, see Technical Support and Contact Information to contact a Technical Support Representative.

    1. A pop-up message appears while the URL is being resolved and then automatically disappears. The Access Control screen of the Publish Web Service Wizard appears. Proceed to Step #4.

  1. In the Access Control screen:

    1. Check the Require SSL/TLS Encryption check box to require that all clients encrypt requests to the Web service during transmission.

    2. Select one of two access control options:

      • Click Allow Anonymous Access to permit users to access the Web service anonymously (without authentication)

      • Click Require Users to Authenticate to require that clients authenticate before gaining access. Define the authentication details for this option as follows:

        1. Select an authentication method from the Authentication Method drop-down list. The authentication method determines what information groups and users are required to provide to gain Web service access.

        2. Select an identity provider that contains the authorized groups and users from the Identity Provider drop-down list.

NOTE: You can give both External Identity Provider and Internal Identity Provider groups and users access to the Web service. For more information on identity providers, see Overview of Identity Providers.

        1. Authorize groups and users to use the Web service by moving them from the No Permission window to the Have Permission window as follows:
          • Click [Add All] to authorize all groups and users in the No Permission window by moving them to the Have Permission window
          • Authorize one or more selected groups or users as follows:

            • To authorize a single group or user, click the group or user name and click [Add]

            • To authorize a continuous range of groups or users, hold down your left mouse button as you move from the first group or user to the last group or user in your target range. Or, left-click the first group or user in the range, hold down the [Shift] button on your keyboard, and then left-click the last group or user in the range. Click [Add]

            • To authorize multiple groups or users placed throughout the list, hold down the [Ctrl] button on your keyboard as you click each group or user name. Click [Add].

NOTE: To de-authorize groups and/or users, move them from the Have Permission window to the No Permission window. Select one or more groups and/or users as outlined above, then click the [Remove] button to remove them from the Have Permission window. To remove all groups and/or users from the Have Permission window, click [Remove All].

        1. Optional: If you need to authorize groups or users from another identity provider, select the new identity provider name from the Identity Provider drop-down list and repeat the instructions in Step #3.

  1. Click [Next]. The Credential Transportation screen appears:

    1. By default, the Web Service URL field is populated with the Web service access information provided in the WSDL document. To change the URL where the SecureSpan™ Gateway can access the Web service:

      1. Click [Change].

      2. Enter a new URL in the Web Service URL field.

      3. Click [Default].

    1.  Select one of two SecureSpan™ Gateway access control options:

      • Click The Gateway can access this Web service anonymously to permit the Gateway to access the protected Web service without authentication

      • Click The Gateway will need to provide credentials to access this Web service to require that the Gateway authenticate to gain Web service access.

  1. Click [Finish]. The Web service is immediately added to the [Web Services] tab.

TIP: Disable the published Web service until its security policy is completed. See Manage Published Web Services for instructions. For information on security policies, see Web Service Security Policies Overview.
