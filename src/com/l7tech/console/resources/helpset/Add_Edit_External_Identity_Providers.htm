Add and Edit External Identity Providers

The SecureSpan™ Manager allows you to configure four types of External Identity Providers (EIPs):

  • Oracle (Oracle Internet Directory)

  • MSAD (Microsoft Active Directory)

  • TivoliLDAP (Tivoli Access Manager)

  • GenericLDAP.

Using the Create Identity Provider Wizard, you can define Group Object Classes and User Object Classes and test the EIP configuration. You can edit or delete an EIP from the [Identity Providers] tab at any time.

Add a New External Identity Provider

A new External Identity Provider (EIP) is added to the SecureSpan™ Gateway with the SecureSpan™ Manager's Create Identity Provider Wizard. The Wizard contains three screens corresponding to the three main tasks in the configuration process:

  1. The Provider Configuration screen defines the EIP connection details.

  2. The Group Object Classes screen provides and defines provider type-specific LDAP group classes.

  3. The User Object Classes screen provides and defines provider type-specific LDAP user classes.

You can view screen-specific information and instructions in the Tips Box located near the bottom of the Create Identity Provider Wizard.

  1. Access the Create Identity Provider Wizard by one of three methods:

    • Click the "Create Identity Provider" link on the Home Page

NOTE: The Home Page appears upon startup or when the Main Tool Bar's [Home] button is clicked.

    • Click [Tasks] > Create Identity Provider from the Main Menu

    • Right-click the "Identity Providers" title in the [Identity Providers] tab. A drop-down menu appears. Select Create Identity Provider.

  1. The Provider Configuration screen of the Create Identity Provider Wizard appears. Select the target EIP type from the Provider Type drop-down list.

  1. The EIP configuration form appears. Fill in the form as follows:

NOTE: The form fields in the Provider Configuration screen and the default classes in the Group Object Classes and User Object Classes screens (see Steps #4 and #5 below, respectively) depend on the EIP type selected from the Provider Type drop-down list.

    • In the Provider Name field, enter a descriptive name for the EIP. For example, enter "My Oracle LDAP"

NOTE: The name entered in the Provider Name field will appear in the [Identity Providers] tab.

    • In the LDAP Host URL field, enter the JNDI (Java Naming and Directory Interface) URL of the LDAP directory service that you want to connect to in the following format: ldap//host:port. For example, enter "ldap://oracle.companyx.com:389"

    • In the Search Base field, enter the search base for groups and users in the LDAP. For example, enter "dc=companyx,dc=com"

    • Optional: In the Bind DN field, enter a binding DN (Distinguished Name) identity for authenticating access to the LDAP directory. For example, enter "cn=manager"

TIP: If you specify an identity in the Bind DN field, then you should also configure a password for the identity in the Bind Password field.

    • Optional: In the Bind Password field, enter a password for the binding DN identity.

  1. Click [Next]. The Group Object Classes screen of the Create Identity Provider Wizard appears. The template for the EIP type selected in the Provider Configuration screen automatically populates the default group object classes. View, edit, and/or add group object classes as follows:

View, Edit, or Delete a Group Object Class

To view the details of a default class, click the class name in the Group Object Classes window and note the attributes in the Attribute Mapping section of the screen. To edit the group object class, click a form element and make changes as required. To delete the group object class, click [Remove].

Add a New Group Object Class

TIP: The Create Identity Provider Wizard is automatically populated with the default group object classes provided by the selected LDAP-type template. Consult your LDAP provider for information about supported group object classes and attribute mapping details before configuring additional group object classes for the External Identity Provider.

Define a new group object class as follows:

    1. In the Group Object Classes screen of the Create Identity Provider Wizard, click [Add]. This action clears the form fields in the Attribute Mapping section of the screen.

    2. Enter the name of the new group object class in the Object Class Name field.

    3. Map the attributes required by the group object class as follows:

NOTE: Mapping group object class attributes associates a SecureSpan™ Gateway attribute with the corresponding attribute in the LDAP schema.

      • In the Group Name field, enter the attribute of the group object class that specifies the group name. For example, a "cn" attribute specifies the name of the group in the LDAP "groupOfUniqueNames" group object class

      • In the Group Member field, enter the attribute of the group object class that specifies the members of the group. For example, a "uniqueMember" attribute specifies the members of the group in the LDAP "groupOfUniqueNames" group object class

      • Select one of the following options from the Member Strategy drop-down list:

Group Member Strategy

Description

Complications

Member is User DN

Every value of the group's member attribute is a fully-formed DN (distinguished name) that is unique within the search base.

When selected, the corresponding attribute in the Group Member field should be something like "cn=user,dc=companyx,dc=com"

Member is User Login

Every value of the group's member attribute is a unique attribute of a user in the directory.

When selected, the corresponding attribute in the Group Member field should be "user"

Member is NV Pair

Every value of the group's member attribute is a name=value pair such as "firstinitiallastname."

When selected, the corresponding attribute in the Group Member field should be "cn=user"

OU Group

If configuring an "organizationalUnit" group object class, then you must associate users to a group.

When an "ou" attribute is entered in the Group Name field, the membership of a user is not described in the group itself but rather by the DN (distinguished name) of the user. For example, the Group Name attribute "ou=group,dc=companyx,dc=com" hints at what the members are, but if you come across a user "cn=user,ou=group,dc=companyx,dc=com," then "user" is a member of that group. When selected, no attribute in the Group Member field is required because the group members are implied by their DN, and if their DN includes the ou, then the members are automatically considered part of the group.

    1. Do one of the following:

      • If required, add another group object class

      • If required, view, edit, or delete an existing group object class as outlined in View, Edit, or Delete a Group Object Class

      • When completed, proceed to Step #5.

  1. Click [Next]. The User Object Classes screen of the Create Identity Provider Wizard appears. The template for the EIP type selected in the Provider Configuration screen automatically populates some user object classes. View, edit, and/or add user object classes as follows:

View, Edit, or Delete a User Object Class

To view the details of a default class, click the class name in the User Object Classes window and note the attributes in the Attribute Mapping section of the screen. To edit the user object class, click a form element and make changes as required. To delete the user object class, click [Remove].

Add a New User Object Class

TIP: The Create Identity Provider Wizard is automatically populated with the default user object classes provided by the selected LDAP-type template. Consult your LDAP provider for information about supported user object classes and attribute mapping details before configuring additional user object classes for the External Identity Provider.

Define a new user object class as follows:

    1. In the User Object Classes screen of the Create Identity Provider Wizard, click [Add]. This action clears the form fields in the Attribute Mapping section of the screen.

    2. Enter the name of the user object class in the User Class Name field.

    3. Map the attributes required by the user object class as follows:

NOTE: Mapping user object class attributes associates a SecureSpan™ Gateway attribute with the corresponding attribute in the LDAP schema.

      • In the User Name field, enter the attribute of the user object class that specifies the user name. For example, the “cn” attribute specifies the name of the user in the LDAP “inetOrgPerson” user object class

      • In the Login Name field, enter the attribute of the user object class that specifies the user login name. For example, the “uid” attribute specifies the login name in the LDAP "inetOrgPerson” user object class

      • In the Password field, enter the attribute of the user object class that specifies the user password. For example, the “userPassword” attribute specifies the user password in the LDAP "inetOrgPerson” user object class

      • In the First Name field, enter the attribute of the user object class that specifies the first name of the user. For example, the "givenName” attribute specifies the first name of the user in the LDAP "inetOrgPerson” user object class

      • In the Last Name field, enter the attribute of the user object class that specifies the last name of the user. For example, the “sn” attribute specifies the last name of the user in the LDAP "inetOrgPerson” user object class

      • In the Email field, enter the attribute of the user object class that specifies the user’s email address. For example, the “mail” attribute specifies the user’s email address in the LDAP “inetOrgPerson” user object class.

    1. Do one of the following:

      • If required, add another user object class

      • If required, view, edit, or delete an existing user object class as outlined in View, Edit, or Delete a User Object Class

      • When completed, proceed to Step #6.

  1. Click [Test] to test the configuration of the new EIP. One of the following will occur:

    • A pop-up error message lists the EIP configuration problems. Do the following:

      1. Note and repair the configuration errors as follows:

        • If an EIP connection error appears, then you must fix the incorrect connection detail(s) in the Provider Configuration screen before you can proceed. Click [OK] to close the dialog box, and then use the [Back] button to return to the Provider Configuration screen. If further errors are not present, proceed to Step #2

        • If a user retrieval error appears, then you must fix the incorrect user object class LDAP mappings in the User Object Classes screen before you can proceed. Click [OK] to close the dialog box, and then use the [Back] button to return to the User Object Classes screen. If further errors are not present, proceed to Step #2

        • If a group member (user) retrieval warning appears, then you can either:

          1. Fix the incorrect group object class LDAP mappings in the Group Object Classes screen. Click [OK] to close the dialog box, and then use the [Back] button to return to the Group Object Classes screen. If further errors are not present, proceed to Step #2.

          2. Leave the incorrect group object class mappings. The SecureSpan™ Gateway will still be able to connect to the EIP. If other errors are not present, proceed to Step #2.

      2. Click [Test]. Continue to fix and test the EIP until no error messages appear. If necessary, see Technical Support and Contact Information to contact a Technical Support Representative. When completed, click [OK] in the Create Identity Provider Wizard. The new EIP is immediately added to the [Identity Providers] tab.

    • A pop-up message validates the configuration of the EIP. Click [OK] to close the dialog box. In the Create Identity Provider Wizard, click [OK]. The new EIP is immediately added to the [Identity Providers] tab.

Edit an External Identity Provider

TIP: Use the Search Identity Provider form to search an External Identity Provider's users and groups. See Search Identity Providers for more information.

An External Identity Provider (EIP) is edited from the Edit Identity Provider Properties Wizard accessed from the [Identity Providers] tab.

 

NOTE: Use the appropriate management tools to modify the individual groups and users that belong to an External Identity Provider.

  1. In the [Identity Providers] tab, access the Edit Identity Provider Properties Wizard for the target EIP by one of two methods:

    • Double-click the EIP name

    • Right-click the EIP name that you want to edit. A drop-down menu appears. Select Properties.

  2. The Edit Identity Provider Properties Wizard appears. Navigate through the Provider Configuration, Group Object Classes, and User Object Classes screens to make changes as required.

TIP: The Edit Identity Provider Properties Wizard is the same as the Create Identity Provider Wizard. Reference the Add a New External Identity Provider section when editing field-level information in the Wizard.

  1. Click [Test] to test the configuration of the modified EIP. One of the following will occur:

    • A pop-up error message lists the EIP configuration problems. Do the following:

      1. Note and repair the configuration errors as follows:

        • If an EIP connection error appears, then you must fix the incorrect connection detail(s) in the Provider Configuration screen before you can proceed. Click [OK] to close the dialog box, and then use the [Back] button to return to the Provider Configuration screen. If further errors are not present, proceed to Step #2

        • If a user retrieval error appears, then you must fix the incorrect user object class LDAP mappings in the User Object Classes screen before you can proceed. Click [OK] to close the dialog box, and then use the [Back] button to return to the User Object Classes screen. If further errors are not present, proceed to Step #2

        • If a group member (user) retrieval warning appears, then you can either:

          1. Fix the incorrect group object class LDAP mappings in the Group Object Classes screen. Click [OK] to close the dialog box, and then use the [Back] button to return to the Group Object Classes screen. If further errors are not present, proceed to Step #2.

          2. Leave the incorrect group object class mappings. The SecureSpan™ Gateway will still be able to connect to the EIP. If other errors are not present, proceed to Step #2.

      2. Click [Test]. Continue to fix and test the EIP until no error messages appear. If necessary, see Technical Support and Contact Information to contact a Technical Support Representative. When completed, click [OK] in the Edit Identity Provider Properties Wizard. EIP changes are implemented immediately in the [Identity Providers] tab.

    • A pop-up message validates the configuration of the EIP. Click [OK] to close the dialog box. In the Edit Identity Provider Properties Wizard, click [OK]. EIP changes are implemented immediately in the [Identity Providers] tab.

Delete an External Identity Provider

NOTE: An error message will appear if you try to delete an External Identity Provider (EIP) whose groups and/or users are attached to a Web service security policy. To delete the EIP, you must first delete the EIP identities from all security policies. See Configure and Edit Security Policies for more information.

  1. In the [Identity Providers] tab, right-click the External Identity Provider name that you want to delete.

  2. A drop-down menu appears. Select Delete.

  3. A confirmation message appears. Click [Yes]. The External Identity Provider is removed from the [Identity Providers] tab.
