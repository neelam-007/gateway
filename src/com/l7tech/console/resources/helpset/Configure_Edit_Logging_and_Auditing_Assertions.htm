Configure and Edit the Logging and Auditing Assertion

In the SecureSpan™ Manager, the following assertion is available in the Logging and Auditing category of the [Assertions] tab:

 

The Logging and Auditing Category of the [Assertions] Tab
 

Audit Assertion

 NOTE: The Audit Assertion is used in conjunction with the Gateway Audit Events form. See View Gateway Audit Events for more information.

The Audit Assertion allows you to control the service-specific audit events that are saved in the SecureSpan™ Gateway database. You can set the severity level of the request audit messages that are collected for analysis in the Gateway Audit Events window. Auditing events are typically message processing summaries or administrative events, such as object changes. See View Gateway Audit Events for more information.

 

 TIP: The Audit Assertion currently only audits events that are at a SecureSpan™ Gateway "warning" level or higher. Contact Layer 7 Technologies to request a lower cutoff severity level, such as "info" or "finer." See Technical Support and Contact Information for contact information.

 

  1. Add the assertion to the Policy Development window by one of two methods:

    • Drag and drop the assertion from the [Assertions] tab into the Policy Development window, or  

    • Highlight the assertion in the [Assertions] tab and click the  (Add Assertion) button in the Assertion Tool Bar.

  1. The "Audit Assertion" assertion is added to the Policy Development window. Right-click the assertion. A drop-down menu appears. Select Audit Assertion Properties.

 TIP: You can also double-click the assertion in the Policy Development window to open the Audit Assertion Properties form.   

  1. The Audit Assertion Properties form appears with a default severity level of Warning that will include all audit alerts that may or may not require follow-up.

 NOTE: The other severity levels in the drop-down list are not enabled in version 3.0 of the SecureSpan™ Manager. Selecting any one of them will not set request auditing. If you want to enable auditing in the target service, then leave the "Warning" setting in the drop-down list.

  1. Click [OK]. The modified assertion is added to the Policy Development window.

 

 NOTE: The Audit Assertion cannot be edited to another severity level (as all except "Warning" are disabled) in version 3.0 of the SecureSpan™ Manager. To remove the auditing of requests, you must delete the Audit Assertion. To do so, right-click the assertion name in the Policy Development window. A drop-down menu appears. Select [Delete Assertion].
