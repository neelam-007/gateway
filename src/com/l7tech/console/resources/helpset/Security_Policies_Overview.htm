Security Policies Overview

 TIP: Become familiar with the policy and assertion information and rules in Security Policy Organization and Rules before configuring a security policy.

The core function of the SecureSpan™ Manager is its ability to centrally define, provision, monitor, and audit security and integration policies for Web services and XML applications, collectively referred to as "services."

 

After a service is published using the Publish Web Service Wizard (see Manage SOAP Web Services), Create WSDL Wizard (see Manage SOAP Web Services), or Publish XML Application Wizard (see Manage XML Applications), the service appears in the [Services] tab and an initial policy is created in the Policy Development window. If a published Web service WSDL document contained at least one HTTP(S) binding, then the initial policy will include an HTTP(S) Routing assertion pre-configured to point to the HTTP(S) binding URL. Before a service is adequately protected, additional security policy assertions may need to be defined and configured into a security policy that is saved in the SecureSpan™ Gateway.

 

 NOTE: If an authentication method and authorization to specific users and/or groups was defined during the Web service or XML application publication process, then service access will be restricted to those identities in the initial security policy. The initial security policy will automatically be populated with an authentication assertion for each Grant Access to Users/Groups assertion corresponding to each selected identity. See Configure and Edit Authentication Assertions for more information.

 

A security policy defines restrictions for the consumption of a published Gateway-protected service. For more information on security policies and assertions, see Security Policy Organization and Rules. For information on the interface areas related to services and security policies, see SecureSpan™ Manager Interfaces. For information on using and configuring the security policy assertions in the [Assertions] tab, see Security Policy Assertions Overview. For information on configuring a security policy in the Policy Development window, see Configure and Edit Security Policies.
