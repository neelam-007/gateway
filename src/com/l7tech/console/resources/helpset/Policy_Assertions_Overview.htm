Policy Assertions Overview

 NOTE: This section discusses configuration instructions for the individual policy assertions in the [Assertions] tab of the SecureSpan™ Manager. For information on the policy and assertion rules used in the SecureSpan Manager, see Policy Organization and Rules. For information on configuring, editing, and manipulating a policy in the Policy Development window, see Configure and Edit Policies.

A policy defines restrictions for the consumption of a published service that is protected by the SecureSpan™ Gateway. Policy assertions are the building blocks for policies in the SecureSpan™ Manager. Located in the [Assertions] tab, the assertions are organized into categories corresponding to the main requirements of a policy:

  • Access Control

The Access Control assertions establish the authorized identities and corresponding credential authentication protocols and requirements in a policy. See Configure and Edit Access Control Assertions for more information

  • Transport Layer Security (TLS)

The Transport Layer Security (TLS) assertion establishes the transport-level encryption requirement for a service. See Configure and Edit the Transport Layer Security (TLS) Assertion for more information

  • XML Security

The XML Security assertions define the message-level encryption and signature requirements for service XML messages and enable protection against replay attacks. See Configure and Edit XML Security Assertions for more information

  • Message Validation/Transformation

The Message Validation/Transformation assertions configure the XML transformations and validation schemas applied to service messages. See Configure and Edit Message Validation/Transformation Assertions for more information

  • Message Routing

The Message Routing assertions define where service messages are sent and what access credentials are required by the back-end service. See Configure and Edit Message Routing Assertions for more information

 NOTE: The Message Routing assertions are not concerned with the credential requirements of the SecureSpan™ Gateway. Use them to define the credential requirements of the target back-end system that will receive SecureSpan Gateway-routed service messages.

  • Service Availability

The Service Availability assertions define the access, availability, and throughput inclusions or restrictions for the service. See Configure and Edit Service Availability Assertions for more information

  • Logging, Auditing, and Alerts

The Logging, Auditing, and Alerts assertions set the threshold for the SecureSpan Gateway audit messages in the Gateway Audit Events window (see View Gateway Audit Events) and configures the properties for SNMP trap alerts. See Configure and Edit the Logging, Auditing, and Alerts Assertions for more information

  • Policy Logic

The Policy Logic assertions organize and define the structure, logic, and processing conditions for the policy. See Configure and Edit the Policy Logic Assertions for more information.

  • Policy Templates

The Policy Templates section is the repository for exported policies. For information on importing or exporting a policy, see Import or Export Policies.

 

 NOTE: The default recipient of the WS-Security (WSS) assertions in the Access Control and XML Security categories of the [Policy Assertions] tab can be changed in the SecureSpan™ Manager. See Change the WSS Assertion Recipient for more information. For information about the four optional SecureSpan™ assertions: the SiteMinder™ authentication assertion, the IBM® Tivoli® Access Manager (TAM) authentication assertion, the Symantec™ Virus Scanning assertion, and the JMS message routing assertion, see Optional Add-On Assertions Overview.

 

Unless specified otherwise, all assertions can be used in a Web service and XML application policy. A policy is constructed by either:

  • Dragging and dropping an assertion from the [Assertions] tab into the Policy Development window, or by

  • Highlighting the target assertion in the [Assertions] tab and clicking the  (Add Assertion) button in the Assertion Tool Bar.

 

 TIP: Double-click an assertion in the Policy Development window to open its corresponding Properties form.

 

Most assertions require configuration either before or after being added to the Policy Development window. See the assertion category-based topic pages for instructions for the individual assertions. See Policy Organization and Rules for general policy and assertion information and rules. See Configure and Edit Policies for information on constructing a policy.
