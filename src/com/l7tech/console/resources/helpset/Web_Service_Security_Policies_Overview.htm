Web Service Security Policies Overview

TIP: Become familiar with the policy and assertion information and rules in Security Policies Overview and Rules before configuring a security policy.

The core function of the SecureSpan™ Manager is its ability to centrally define, provision, monitor, and audit security and integration policies for Web services. After a Web service is published using the Publish Web Service Wizard (see Publish Web Services) or the Create WSDL Wizard (see Create Web Service WSDL), the Web service appears in the [Web Services] tab and a routing assertion is automatically created from the Web service URL specified during the publication process. Before a Web service is considered protected, additional security policy assertions must be defined and configured into a security policy that is saved in the SecureSpan™ Gateway. The security policy defines restrictions for the consumption of a published Web service that is protected by the Gateway. For more information on Web service security policies and assertions, see Security Policies Overview and Rules. For information on the interface areas related to Web services and security policies, see SecureSpan™ Manager Interface. For information on using and configuring the security policy assertions in the [Assertions] tab, see Security Policy Assertions Overview. For information on configuring a Web service security policy in the Policy Development window, see Configure and Edit Security Policies.
