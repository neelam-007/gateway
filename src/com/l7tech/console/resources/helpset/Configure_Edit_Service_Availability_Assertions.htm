Time/Day Availability

The Time/Day Availability assertion allows you to restrict service access by a SecureSpan™ Gateway time and/or day interval. When the SecureSpan Gateway receives a request for the service, it will check the time and/or day restrictions before allowing the message to proceed.

Add and Configure the Time/Day Availability Assertion

  1. Add the assertion to the Policy Development window by one of two methods:

    • Drag and drop the assertion from the [Assertions] tab into the Policy Development window, or  

    • Highlight the assertion in the [Assertions] tab and click the  (Add Assertion) button in the Assertion Tool Bar.

  1. The Time/Day Properties form appears. Specify a time and/or date range in which to restrict access as follows:

NOTE: The time and/or day restrictions are based on the SecureSpan™ Gateway time zone.

Restrict Access by Time

To restrict access between a specific time range:

    1. Check the Restrict Time of Day check box.

    2. In the Between set of time values, select or enter the starting time for the range into the hr, min, and sec fields.

NOTE: The time range must be configured using 24-hour notation from the earliest time (start) to the latest time (end).

    1. In the and set of time values, select or enter the ending time for the range into the hr, min, and sec fields.

NOTE: The entered times are translated into UTC (Coordinated Universal Time) time in the read-only time set under the fields.

    1. Proceed to restrict access by day, or proceed to Step #3.

Restrict Access by Day

To restrict access between a specific day range:

    1. Check the Restrict Day of Week check box.

    2. In the Between drop-down list, select the starting day for the range.

    3. In the and drop-down list, select the ending day for the range.

NOTE: If you select the same day for the range, then access will only be restricted on that single day.   

    1. Proceed to restrict access by time as outlined above, or proceed to Step #3.

 

  1. Click [OK]. The assertion is added to the Policy Development window.

Edit the Time/Day Availability Assertion

  • In the Policy Development window, right-click the "Time/Day Availability" assertion. A drop-down menu appears. Select Time/Day Availability Properties

  • The Time/Day Properties form appears. Edit the assertion properties as outlined in Add and Configure the Time/Day Availability Assertion.

 

IP Address Range

The IP Address Range assertion allows you to restrict or allow service access based on the IP address of the Web service or XML application requestor.

Add and Configure the IP Address Range Assertion

  1. Add the assertion to the Policy Development window by one of two methods:

    • Drag and drop the assertion from the [Assertions] tab into the Policy Development window, or  

    • Highlight the assertion in the [Assertions] tab and click the  (Add Assertion) button in the Assertion Tool Bar.

  1. The Remote IP Range Properties form appears. Select an "Authorize" or "Forbid" option from the drop-down list.

  2. In the The Following IP Range fields, enter the allowable or forbidden IP address and/or "bits" in accordance with the CIDR (Classless Inter-Domain Routing) standard.

  1. Click [OK]. The assertion is added to the Policy Development window.

Edit the IP Address Range Assertion

  • In the Policy Development window, right-click the "IP Address" assertion. A drop-down menu appears. Select IP Address Range Properties

  • The Remote IP Range Properties form appears. Edit the assertion properties as outlined in Add and Configure the IP Address Range Assertion.

 

Throughput Quota

The Throughput Quota assertion allows you to limit the number of service requests permitted within a pre-determined time period. At runtime, the SecureSpan™ Gateway utilizes a counter to enforce a defined quota. The Throughput Quota assertion will fail when the quota limit is reached.

 

Since counters are global to the SecureSpan Gateway, different services can contain Throughput Quota assertions that use the same counter. Similarly, using a single counter across different users or groups facilitates Service Level Agreement (SLA) migration in a policy that must enforce different SLAs for different identities.

 

NOTE: The counters used in the Throughput Quota assertion are synchronized across single or clustered SecureSpan™ Gateways. See Technical Support to contact Layer 7 Technologies for more information.

 

Add and Configure the Throughput Quota Assertion

  1. Add the assertion to the Policy Development window by one of two methods:

    • Drag and drop the assertion from the [Assertions] tab into the Policy Development window, or  

    • Highlight the assertion in the [Assertions] tab and click the  (Add Assertion) button in the Assertion Tool Bar.

  1. The Throughput Quota Assertion form appears. In the Quota section of the form:

    • Enter a request limit into the Max field. The value must be a whole number between 1 - 100,000

    • Select a time unit for the request limit from the drop-down list. The units represent closed time periods. If a quota of 1000 requests per hour is configured, for example, then the SecureSpan™ Gateway will enforce a limit of 1000 requests between the start and end of each hour of the day

    • Select an enforcement option:

      • Select Quota per requestor to apply the assertion to each individual identity, or requestor, in the policy. If a quota of 1000 requests per hour is configured for a policy, for example, then each individual requestor will be permitted 1000 requests per hour

      • Select Global quota to apply the assertion to all identities in the policy. If a quota of 1000 requests per hour is configured for a policy, for example, then the combined requests per hour for all requestors in the policy cannot exceed 1000.

  1. In the Counter section of the Throughput Quota Assertion form, select or configure a counter as follows:

Select an Existing Counter

To select a previously configured counter, select an existing counter name from the Counter ID drop-down list. Since a counter can only be incremented once within a single request context, the same counter can be used for different Throughput Quota assertions within the same policy. In other words, regardless of the number of Throughput Quota assertions in a policy, the SecureSpan Gateway will only count each request once towards the quota when the assertions use the same counter.

Configure a New Counter

To configure a new counter, enter a unique counter name into the Counter ID field.  

  1. Select one of three runtime behavior options for the Throughput Quota assertion counter:

Always Increment

When the Always increment option is selected, the SecureSpan Gateway will always increment the counter (unless the counter was already incremented in this request's context) before determining whether the quota limit has been reached. If the resulting counter value exceeds the quota, the assertion will fail; otherwise, the assertion succeeds.

Increment Only When Still Within Quota

When the Increment only when still within quota option is selected, the SecureSpan Gateway will only increment the counter if the resulting value will not exceed the quota (unless the counter was already incremented in this request's context). If the quota limit has not been reached, then the request is counted and the assertion succeeds. If the quota limit has been reached, then the assertion fails and the initiating request is not counted towards the quota.  

Decrement

When the Decrement option is selected, the SecureSpan Gateway will decrement the counter (if it was already incremented in the request's context) by a single count. Use this option in a policy to ensure that service processing errors or incomplete transactions are not counted towards the quota.

  1. Click [OK]. The assertion is added to the Policy Development window.

Edit the Throughput Quota Assertion

  • In the Policy Development window, right-click the "Max Throughput" assertion. A drop-down menu appears. Select Throughput Quota Properties

  • The Throughput Quota Assertion form appears. Edit the assertion properties as outlined in Add and Configure the Throughput Quota Assertion.
