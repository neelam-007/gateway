SecureSpan™ Manager Interfaces

 NOTE: The Troubleshooting Mode of the SecureSpan™ Manager is launched separately from the main interface. See SecureSpan™ Manager Troubleshooting Mode for more information.

The SecureSpan™ Manager contains two role-based interfaces: a full-featured and privileged Administrator Interface for Administrators and a partial view-only Operator Interface for Operators. The Administrator Interface contains all SecureSpan Manager features and functions, whereas the Operator Interface contains limited read-only features. See SecureSpan™ Manager Operator Documentation for a list of the features available in the Operator Interface. Role-based access for Administrators and Operators is determined by the group assignment of their user credentials in the SecureSpan™ Gateway's Internal Identity Provider. See Add and Edit Internal Identity Provider Users and Groups for more information.

 

 NOTE: This Help System includes all features and functions available in the SecureSpan™ Manager Administrator Interface. For Help System information and navigation instructions, see SecureSpan™ Manager Documentation. For a list of Help System sections relevant to the Operator Interface, see SecureSpan™ Manager Operator Documentation.

General Interface Sections

 NOTE: In the Operator Interface, the Home Page contains a limited set of shortcut links and many features are disabled in the Main Menu, Main Tool Bar, Policy Tool Bar, right-click tab menus, and forms.

Main Menu:

 NOTE: Many Main Menu items can be accessed through keyboard shortcuts where provided. Press [Alt] on your keyboard to enable the shortcuts. The initiating letter for a shortcut is underlined in the Main Menu and in each respective drop-down menu. For example, to open the Preferences form (see Configure SecureSpan™ Manager Preferences) using shortcuts, click [Alt + F] (to open the [File] menu) and then [Alt + P] (to open the Preferences form) on your keyboard.

Contains drop-down menus that include all of the program features, functions, and information.

Main Tool Bar:

Contains shortcut buttons to commonly-used program features.

Home Page:

Contains the Home Page upon startup or when the Main Tool Bar's [Home] button is clicked. The Home Page contains shortcut links to commonly used wizards and forms.

Policy Development Window:

 NOTE: In the Operator Interface, the assertions in the Policy Development window are view-only.

  • Contains the Policy Development window when viewing a service from the [Services] tab. The Policy Development window is the repository for the assertions selected from the [Assertions] tab and configured into a policy for the selected service. The Policy Development window includes the Policy Tool Bar that contains buttons for saving, validating, importing, and exporting a policy. See Policies Overview for more information

 TIP: Double-click an assertion in the Policy Development window to open its corresponding Properties form.

  • Contains the Policy Validation Messages window when [View] > Policy Messages is selected from the Main Menu (check box checked) and when viewing a policy in the Policy Development window. The Policy Validation Messages window contains configuration, warning, and error feedback messages for the policy. To remove the Policy Validation Messages window from the Policy Development window, select [View] > Policy Messages from the Main Menu (check box unchecked). See Configure and Edit Policies for more information

  • Contains the Identity window when [Identity View] is selected from the Policy Tool Bar when viewing a policy in the Policy Development window. To toggle back to the Policy Development window, click [Policy View] on the Policy Tool Bar.

[Identity Providers] Tab:

Contains the Internal Identity Provider (IIP) and any configured LDAP Identity Providers (LIPs) and Federated Identity Providers (FIPs). Right-click the IIP to search, view, and add users and/or groups. Right-click an LIP to search, view, and edit its properties. Right-click an FIP to search, view, edit its properties, and add users and/or groups. For more information, see Identity Providers Overview.

[Services] Tab:

Contains an alphabetical list of published Web services and XML applications, each with an expandable list of messages, port types, bindings, and services. In the tab, Web services are identified by an  icon and XML applications are identified by an  icon. All services are enabled by default when published using the Publish SOAP Web Service Wizard, Create WSDL Wizard, or Publish XML Application Wizard, but can be disabled by right-clicking the service name and selecting [Disable] from the drop-down menu. Disabled services contain a red "X" through the service icon. For more information, see Services Overview. For information about managing the published services in the [Services] tab, see Manage Published Services.

[Assertions] Tab:

  • Contains a categorized list of the policy assertions used to construct a policy in the Policy Development window for a service selected in the [Services] tab. Each assertion category contains an expandable list of assertion options. See Policy Assertions Overview for more information

  • Includes the Assertion Tool Bar that contains shortcut buttons that are used to move and organize the assertions in the Policy Development window. The action commands are also available by right-clicking an assertion in the Policy Development window.

Status Bar:

Contains a user, SecureSpan™ Gateway, and port information message when [View] > Status Bar is selected from the Main Menu (check box checked). To remove the Status Bar, click [View] > Status Bar from the Main Menu (check box unchecked).

 

Wizards

 NOTE: Only the Administrator Interface of the SecureSpan™ Manager permits access to wizards.

The SecureSpan™ Manager employs user-friendly wizards to help users configure complex tasks in several managed and logical steps. Each wizard interface includes three areas:

Steps List:

Presents a numbered list of the steps required by the task. Each step in the Steps list corresponds to a screen in the wizard.

Tips Box:

Contains a text box that describes the principles of or the form elements within the wizard screen.

Form:

Contains the form elements or actions required by the configuration process.
