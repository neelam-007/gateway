Using the SiteMinder™ Authentication Assertion

 NOTE: The Administrator is responsible for installing and configuring the SiteMinder™ Custom Assertion package in the SecureSpan™ Gateway. If you encounter authentication errors during the execution of a policy, direct the Administrator to the Troubleshoot SiteMinder™ Assertion Errors section below.

Installing and configuring the SiteMinder™ Custom Assertion package in the SecureSpan™ Gateway installs and enables the SiteMinder assertion in the SecureSpan™ Manager. The SiteMinder assertion — called "SiteMinder Protected Resource” in the [Assertions] tab and Policy Development window — instructs the SecureSpan Gateway to delegate the authentication and authorization tasks required to gain access to a protected service to the Netegrity SiteMinder Policy Server (versions 5.5 and 6.0).

SiteMinder™ Assertion Rules

Note the following rules when using the SiteMinder™ assertion:

  • The SiteMinder assertion cannot be used with:

    • Authentication assertions that encrypt passwords, such as the HTTP Digest and SSL with Client Certificate Authentication assertions from the Access Control category of the [Assertions] tab

    • The Sign Request Element and Encrypt Request Element assertions located in the XML Security category of the [Assertions] tab

    • The Grant Access to Users/Groups assertion located in the Access Control category of the [Assertions] tab.

  • The SiteMinder assertion can be used with:

    • The HTTP Basic assertion from the Access Control category of the [Assertions] tab

    • The SSL or TLS Transport assertion from the Transport Layer Security (TLS) category of the [Assertions] tab

    • Any other assertion not listed in the above list.

  • A policy can only contain a single SiteMinder assertion per authentication scheme. More than one SiteMinder assertion may appear in a complex policy that contains more than one authentication scheme

  • In a policy, the SiteMinder assertion must appear before the routing assertion and after the SSL or TLS Transport and authentication method assertions.

 

Add and Configure the SiteMinder™ Assertion

  1. Access the Policy Development window for the target service by one of two methods:

 TIP:  Disable a service while constructing its policy to prevent unsecured and unauthorized access. A disabled service contains a red "X" through its left-side icon. See Manage Published Services for information.

    • Right-click the service name in the [Services] tab. A drop-down menu appears. Select Policy Assertions. Or,

    • Double-click the service in the [Services] tab.

  1. The Policy Development window appears. Add the "SiteMinder Protected Resource" assertion to the policy by one of two methods:

    • Drag and drop the assertion from the [Assertions] tab into the Policy Development window, or   

    • Highlight the assertion in the [Assertions] tab and click the  (Add Assertion) button in the Assertion Tool Bar.

  1. The assertion is added to the Policy Development window. Right-click the assertion. A drop-down menu appears. Select Custom Assertion Properties.

  1. The SiteMinder Protected Resource form appears. Fill in the form as follows:

 NOTE: The action and resource values are determined by the settings in the realm that is used by the SecureSpan™ Gateway custom agent in the Netegrity SiteMinder™ Policy Server. Consult your Administrator for information about the action and resource properties.

    • In the Value field of the action row, enter the action (such as “POST” or “GET”) for the protected resource specified in the resource row below

    • In the Value field of the resource row, enter the resource that is being protected by the SiteMinder Policy Server.

  1. Click [OK]. The modified assertion is added to the Policy Development window. Proceed to the "Validate a Policy" section in Configure and Edit Policies.

 

Edit the SiteMinder™ Assertion

  1. In the Policy Development window, right-click the "SiteMinder Protected Resource" assertion. A drop-down menu appears. Select Custom Assertion Properties.

  1. The SiteMinder Protected Resource form appears. Modify the assertion properties as outlined in Add and Configure the SiteMinder™ Assertion.

  2. Click [OK]. The modified assertion is added to the Policy Development window. Proceed to the "Validate a Policy" section in Configure and Edit Policies.

 

Troubleshoot SiteMinder™ Assertion Errors

 NOTE: Contact your Administrator if you encounter authentication errors.

If configuration errors exist in the Netegrity SiteMinder™ Policy Server or the SecureSpan™ Gateway, then one of the following error messages will appear in the SecureSpan™ Manager Gateway Audit Events window when the SiteMinder assertion is used in a policy. For information about the Gateway Audit Events window, see View Audit Events.

  • SEVERE: Unable to connect to the SiteMinder Policy Server

This error message appears in the Gateway Audit Events window when:

    • The SiteMinder Policy Server is down

    • The SecureSpan Gateway is not properly configured to connect to the SiteMinder Policy Server

    • The connection credentials cannot be read properly because the hashed cookie that is presented to the SiteMinder Policy Server cannot be decrypted.

An error message indicating a SiteMinder Agent initialization failure is also displayed. Verify the SecureSpan Gateway and SiteMinder Policy Server connection settings. To contact a Layer 7 Technologies Technical Support Representative, see Technical Support

  • WARNING: Authorization (access control) failed

This error message appears in the Gateway Audit Events window when the SecureSpan Gateway connection credentials are not authenticated or authorized by the SiteMinder Policy Server. A Log on to Gateway form prompts you to re-enter your user name and/or password. Ensure that the user name and password entered in SecureSpan™ Bridge match those configured in the user database used by the SiteMinder Policy Server to authenticate and authorize users. To contact a Layer 7 Technologies Technical Support Representative, see Technical Support.
