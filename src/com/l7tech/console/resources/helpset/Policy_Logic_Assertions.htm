At Least One Assertion Must Evaluate to True

 TIP: See Policy Organization and Rules for more information about using an "At least one assertion must evaluate to true" assertion folder in a policy.

The "At least one assertion must evaluate to true" assertion folder organizes and defines the processing conditions for the assertions that it contains and for the overall policy. When assertions are grouped into an "At least one assertion must evaluate to true" folder in the Policy Development window, each successive child assertion is processed until a single assertion succeeds, yielding a success outcome for the folder. If all child assertions in an "At least one assertion must evaluate to true" folder fail, then the overall folder—and possibly the entire policy—fails. For more information about parent and child assertions in a policy, see Policy Organization and Rules.

 

 TIP: If you do not want the failure of an "At least one assertion must evaluate to true" folder to fail the entire policy, then add a Continue Processing assertion into the assertion folder. The Continuing Processing assertion will always evaluate to true, preventing the failure of a policy due to the failure of a non-essential or conditional assertion in an "At least one assertion must evaluate to true" folder.

 

Add and Configure the At Least One Assertion Must Evaluate to True Folder

  1. Add an "At least one assertion must evaluate to true" assertion folder to the Policy Development window by one of two methods:

Use the Right-Click Menu

Add the assertion folder from the Policy Development window's right-click menu:

    • Highlight an assertion near the desired insert location for the assertion folder

    • Right-click the Policy Development window. A drop-down menu appears

    • Select Add 'One or More' Assertion.

Insert from the [Assertions] Tab

Add the assertion folder from the [Assertions] tab by one of two methods:

    • Drag and drop the assertion from the [Assertions] tab into the Policy Development window, or  

    • Highlight the assertion in the [Assertions] tab and click the  (Add Assertion) button in the Assertion Tool Bar.

  1. The assertion folder is added to the Policy Development window. Drag and drop assertions from the Policy Development window into the assertion folder, or add new assertions directly into the assertion folder as required:

 TIP: To add a new identity into the assertion folder, right-click the assertion folder in the Policy Development window. A drop-down menu appears. Select Add User or Group. Proceed to Configure and Edit Access Control Assertions to configure the new Grant Access to Users/Groups assertion. Note that there should not be more than one first-level Grant Access to Users/Groups assertion in a policy or within an assertion folder. To grant access to more than one user or group in a policy, place each individual Grant Access to Users/Groups assertion in an individual "At least one assertion must evaluate to true" or "All assertions must evaluate to true" folder with an authentication assertion (and other assertions, as required). See Policy Organization and Rules for more information.

    • For assertion configuration instructions, view the category specific instruction pages outlined in Policy Assertions Overview

    • For assertion and assertion folder organization instructions, see Configure and Edit Policies.

  1. Repeat to add and organize additional assertion folders.

Edit an At Least One Assertion Must Evaluate to True Folder

 TIP: See Policy Organization and Rules for more information about using an "At least one assertion must evaluate to true" assertion folder in a policy.

The contents and location of an "At least one assertion must evaluate to true" assertion folder can be modified at any time:

  • New assertions can be added to the folder by dragging and dropping them from the Policy Development window or adding from the [Assertions] tab

  • Assertions can be removed from the folder by dragging and dropping them back into the Policy Development window

  • The location of the assertion folder in a policy, and the order of the assertions within the folder, can be modified as outlined in Configure and Edit Policies.

Delete an At Least One Assertion Must Evaluate to True Folder

 WARNING: Deleting an assertion folder also removes the assertions that it contains from the Policy Development window and the SecureSpan™ Gateway, possibly invalidating the policy.

To delete an "At least one assertion must evaluate to true" assertion folder:

  1. Right-click the folder in the Policy Development window. A drop-down menu appears.

  2. Select Delete Assertion. A confirmation message appears.

  3. Click [Yes]. The assertion folder and its contents are removed from the Policy Development window and the SecureSpan™ Gateway.

 

All Assertions Must Evaluate to True

 TIP: See Policy Organization and Rules for more information about using an "All assertions must evaluate to true" assertion folder in a policy.

The "All assertions must evaluate to true" assertion folder organizes and defines the processing conditions for the assertions that it contains and for the overall policy. When assertions are grouped into an "All assertions must evaluate to true" folder, each successive child assertion is processed until all assertions succeed, yielding a success outcome for the folder. Processing in the "All assertions must evaluate to true" folder will stop when the first child assertion fails, yielding a fail outcome for the folder—and possibly the entire policy. For more information about parent and child assertions in a policy, see Policy Organization and Rules.

Continue Processing

 NOTE: The Continue Processing assertion must be used with an "At least one assertion must evaluate to true" assertion folder.

The Continue Processing assertion is a placeholder assertion that will always yield a successful or true processing result. When used in an "At least one assertion must evaluate to true" assertion folder (see At Least One Assertion Must Evaluate to True), the Continue Processing assertion will ensure that the incidental failure of a non-essential assertion within the same folder will not cause an overall policy failure.

 

  • Add the assertion to the Policy Development window by one of two methods:

    • Drag and drop the assertion from the [Assertions] tab into the Policy Development window, or  

    • Highlight the assertion in the [Assertions] tab and click the  (Add Assertion) button in the Assertion Tool Bar.

  • The assertion is added to the Policy Development window. For information on adding and organizing the Continue Processing assertion into an "At least one assertion must evaluate to true" assertion folder, see At Least One Assertion Must Evaluate to True.

 

 NOTE: You cannot edit the Continue Processing assertion. To remove the assertion, right-click the assertion name in the assertion folder or Policy Development window. A drop-down menu appears. Select [Delete Assertion].
