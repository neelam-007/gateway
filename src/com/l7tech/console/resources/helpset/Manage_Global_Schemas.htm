Manage Global Schemas

 NOTE: The Manage Global Schemas form is used in conjunction with the Validate XML Schema assertion (see Configure and Edit Message Validation/Transformation Assertions).

A global schema is a schema that can be referenced by an import statement in one or more Validate XML Schema assertions (see Configure and Edit Message Validation/Transformation Assertions). During runtime, the SecureSpan™ Gateway or Tarari PCI card resolves a schema referenced by an import statement as part of the validation process. The referenced schema, or global schema, must exist in the Manage Global Schemas form — and hence in the SecureSpan Gateway or Tarari PCI card — in order for validation to succeed. Furthermore, in order for the SecureSpan Gateway to be able to resolve external schemas when running Validate XML Schema assertion, the import statement in the assertion schema must contain a "schemaLocation" attribute value that matches the global schema's "System ID" value.

 

Global schemas are managed in the Manage Global Schemas form, not the Schema Validation Properties form that is associated with the Validate XML Schema assertion. Global schemas can be added, edited, and deleted from the Manage Global Schemas form.

Add a New Global Schema

 TIP: You can add global schemas prior to configuring the Validate XML Schema assertion. Before adding a new global schema, note the value in the "schemaLocation" attribute of the schema import statement that refers to it. This value must match the global schema System ID value configured in Step # 3 below.

Add a new global schema as follows:

  1. In the SecureSpan™ Manager, select Tasks > Manage Global XML Schemas from the Main Menu. The Manage Global Schemas form appears.

 NOTE: When using the Schema Validation Properties form to configure the Validate XML Schema assertion, the Manage Global Schemas form will automatically appear if the added schema contains one or more import statements that reference one or more global schemas that do not exist in the SecureSpan™ Gateway or Tarari PCI card. After adding the required global schema(s) with the Manage Global Schemas form, return to the Schema Validation Properties form (see Configure and Edit Message Validation/Transformation Assertions) to complete the assertion configuration process.

  1. Click [Add]. The Edit Global Schema form appears.

  2. In the System ID field, enter a value corresponding to the "schemaLocation" attribute in the schema import statement that references the global schema. For example, schema A contains an import statement that references global schema B. The value in the "schemaLocation" attribute in schema A is "account.xsd".

<s:import namespace="http://www.acme.com/schemas/account" schemaLocation="account.xsd" />

In order to connect schema A and global schema B in the SecureSpan Gateway or Tarari card, the System ID of global schema B must be "account.xsd."

  1. Use the XML Editor in the Edit Global Schema form window to insert a file into, edit, search, parse, format, or comment the global schema as required:

 TIP: Any change made via the right-click menu can be undone. To undo the last change, select Undo from the drop-down menu. To redo an undone change, select Redo from the drop-down menu.

Insert a File into the Schema

To insert an XML file into the schema:

    1. Place the cursor at the desired insert location in the Edit Global Schema form window.

    2. Right-click inside the Edit Global Schema form window. A drop-down menu appears.

    3. Select Document > Insert File. The Open form appears.

    4. Navigate to and select the target XML file. Click [OK]. The XML content is inserted into the schema.

Edit the Schema

To cut, copy, and/or paste schema content, right-click inside the Edit Global Schema form window. A drop-down menu appears:

    • To cut or copy content, select the target content and click Cut or Copy

    • To paste content, select a target location and click Paste.

Search the Schema Nodes

To search the nodes in the schema:

    1. Right-click inside the Edit Global Schema form window. A drop-down menu appears.

    2. Select Search a Node. The Search form appears containing an expandable list of the nodes within the schema. To view the schema location of a node in the Search form, click the node. The node's corresponding location in the window is highlighted.

    3. Click [OK] to close the Search form.

Parse the Schema

To parse the schema XML:

    1. Right-click inside the Edit Global Schema form window. A drop-down menu appears.

    2. Select XML > Parse. A pop-up status message appears.

    3. Note the errors, if any. Click [OK] to close the message.

Format the Schema

To format the schema XML, right-click inside the Edit Global Schema form window. A drop-down menu appears. Select XML > Format.

Comment the Schema

To insert a comment into the schema:

    1. Place the cursor at the desired insert location in the Edit Global Schema form window.

    2. Right-click inside the window. A drop-down menu appears.

    3. Select XML > Comment. The Comment form appears.

    4. Enter comment text into the window. Multiple lines are allowed.

 NOTE: The text does not require comment characters. The SecureSpan™ Manager automatically formats the text as a comment in the window.

    1. Click [OK]. The comment is inserted into the schema.

 

  1. In the Edit Global Schema form, click [OK]. The new global schema is added to the Manage Global Schemas table.

 

Edit a Global Schema

  1. In the SecureSpan™ Manager, select Tasks > Manage Global XML Schemas from the Main Menu. The Manage Global Schemas form appears.

  1. In the table, select the global schema that you want to edit. Click [Edit].

  2. The Edit Global Schema form appears. Edit the schema as outlined in Add a New Global Schema. You can also directly click and modify schema XML in the Edit Global Schema form window.

 

Delete a Global Schema

 WARNING: If using a Tarari PCI card for schema validation, do not delete the pre-loaded SOAP envelope schema from the Manage Global Schemas form. Doing so will cause a Tarari schema processing error that will cause schema validation to fall back on software validation. Similarly, do not remove a global schema that is referenced by schema import statements in any Validate XML Schema assertions (see Configure and Edit Message Validation/ Transformation Assertions). Doing so will break the resolution process, resulting in schema validation and policy errors.

  1. In the SecureSpan™ Manager, select Tasks > Manage Global XML Schemas from the Main Menu. The Manage Global Schemas form appears.

  1. In the table, select the global schema that you want to delete. Click [Remove].

  2. The global schema is removed from the Manage Global Schemas form and the SecureSpan™ Gateway and/or Tarari PCI card.
