Creating Policies

The primary function of the Manager application is to build security policies for individual Web services. Each policy is made up of several assertions that determine the authentication method, identity credentials, transport method, and/or the routing method. These assertions must follow a specific ordering logic for the policy to be validated and approved. This logic is called the policy model.

The Policy pane on the right-side of the user interface offers several options for editing and building policies. Administrators can drag-and-drop assertions from the Assertions pane to the policy. The policy menu offers functionality to manipulate the new assertion into a logical position. Once the changes are complete, the policy buttons are used to validate and save the revised policy. The Toggle view button is used to flip from the policy assertion view to the identity view.

Policy Model

Policies in the Manager are built using the established policy model. Each line, or assertion, in a policy must follow this model's rules or the policy will not validate as OK. The first level of the policy includes the security assertions in the appropriate order. The third level of the policy includes additional assertion logic folders and their associated users/groups.

Building Policies

When a Web Service is published in the Manager, the basic security policy is created in the Publish Services wizard. Administrators can then edit and change the policy as required.

To edit a policy:

  1. Right-click on the Web Service name and select Service Policy. The policy displays in the right side of the user interface.

  2. Select the desired assertion from the Assertions pane. Drag and drop the assertion to the Policy pane.

  3. Right-click on the new assertion. The assertion menu appears. Move the assertion up or down using the menu options. Use the menu to add any assertion grouping folders, such as "One or more must evaluate to true."

  4. Click the Validate Policy button. View the messages in the Message pane and correct any issues.

Note: For information on interpreting the policy validation messages, see Validating Policies.

  1. Click the Save Policy button.

Policy Menu

The policy menu provides multiple options for viewing assertion properties, moving assertions, adding assertion logic folders, and saving policies. This menu is accessed by right-clicking any assertion in the Policy pane. Each assertion displays menu items that relate only to the assertion's functionality.

Note: Remember to re-validate and save the policy after making any changes via the policy menu options.

The menu options are:

Assertion properties: Displays the Assertion properties dialog box with additional assertion property options.

View identity policy: Displays the Identity policy dialog box with authentication and routing options for the identity. Options are activated when available on the same level as the selected identity.

Add "All" assertion: Adds an assertion logic folder that requires "All assertions must validate to true." This criteria applies to any assertions located below the folder.

Add "One or More" assertion: Adds an assertion logic folder that requires "At least one assertion must validate to true." This criteria applies to any assertions located below the folder.

Delete assertion: Displays the Delete assertion dialog box to verify deletion of the selected assertion. This function is also available in the mid-section buttons.

Move assertion up: Automatically moves the assertion up one level in the policy. This function is also available in the mid-section buttons.

Move assertion down: Automatically moves the assertion down one level in the policy. This function is also available in the mid-section buttons.

Save policy: Automatically saves the most recent changes to the policy.

Validate policy: Automatically assesses the policy's validity and displays validation information in the Messages pane.

Add user or group: Displays the Add identity assertion dialog box with the option to select an identity provider and add (or remove) a user/group to the policy.

Policy Views

The Toggle view button in the Policy pane offers two policy views. Clicking on the button switches between the policy assertion view to the identity view. Administrators can use this view to see what policies are attached to specific users. This perspective is useful for tracking policy decisions and troubleshooting.
