Publishing Services

Once the Manager has been set up, administrators must publish their Web services. Publishing Web services makes the services visible in the Manager user interface and establishes the initial security policy. Administrators can then use the Services pane to disable the service, add to the security policy, and change the basic service properties.

Publish Service Wizard

The Publish Service wizard automates the process of adding a Web Service to the Manager and defining its basic security policy. The wizard guides a user through entering the WSDL, defining the access credentials, and selecting the Gateway access.  Once the service has been published, clients can access the newly protected service via the Gateway.

Launching the Publish Service wizard

There are three different ways to launch the Publish Service wizard:

  • In the Home section, select the Publish protected Web Service link.

  • Under the Action menu, select Publish Service.

  • In the Services section, right-click the Services@https://servername:port and select Publish Service.

Using the Publish Service Wizard

Before publishing a Web Service with the Publish Service wizard, administrators must first define the WSDL and the components of their service policy. The wizard requires the following information:

Protected Service: Provide the URL for the Web Service's WSDL.

Access Control: Determine whether SSL/TLS encryption is used. Select either anonymous access or authenticated access. If authenticated access is used, administrators must define the authentication method, the identity provider, and the users/groups with access.

Service Credentials: Edit and save the Web Service URL, if required. Select either anonymous Gateway access or provide credentials for authenticated Gateway access.

Service Function Menu

Once the service is published, the service name appears in the Service pane. The service function menu is available by right-clicking on the service's name. The menu functions are as follows:

Service policy: Displays the Web Service security policy in the right-side of the user interface. Administrators can use this view to modify and/or validate the service's policy.

Rename: Displays a dialog box to change the Web Service name.

Enable: Enables access to the Web service. Services are enabled by default.

Disable: Disables access to the Web Service. This function is useful for services that are still under development.

Delete: Displays a dialog box that offers the option to delete the Web Service from the Manager.
