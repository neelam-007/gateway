Creating Services

The Manager provides the option of creating a WSDL for Web services. The WSDL Creation Wizard automates the steps involved in creating a WSDL. This wizard guides advanced users through creating the definition, messages, port type, operations, and the bindings.

Warning: The WSDL Creation Wizard is intended for architects and advanced Web services developers. Administrators who use this tool should be well-versed in the WSDL standard, SOAP, XML, and the SOAP protocols.

Launching the WSDL Creation Wizard

There are three different ways to launch the WSDL Creation wizard:

  • In the Home section, select the Create Web Service definition link.

  • Under the Action menu, select Create Service.

  • In the Services section, right-click the Services@https://servername:port and select Create Service.

Using the WSDL Creation Wizard

Before creating a WSDL with the Manager's wizard, administrators must first decide on the key elements. The wizard requires the following information:

Definition: Determine the name, the target namespace, the default namespace, and the namespace details.

Messages: Determine the message names and parts (name and type).

Port Type and Operations: Determine the Port Type name, operation, and messages (input and output).

Bindings: Determine the binding name, style, transport method, and operations.

Service: Determine the service name, port, binding, and routing address.

Note: The service address determines the policy's routing assertion and must be a valid URL.
