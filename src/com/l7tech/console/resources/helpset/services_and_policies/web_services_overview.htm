Web Services Overview

The primary function of the SecureSpan solution is to establish the security layer on top of already existing Web services. Most of the Manager's functionality is, therefore, focused on building and enabling security policies. The key tasks in building Web services security are publishing a service, building a security policy, and enabling the policy.

Publishing Services

Once administrators have set up the Manager (see the Setup Overview), they can proceed to publish protected Web services. All new services are published via the Publish Services wizard. Once published in the Manager, administrators can build customized security policies that include authentication credentials and message routing data. See the Publishing Services and Creating Policies topics for more detailed information.

Creating Services

The Manager provides an additional layer of functionality for advanced users. Architects and experienced Web services developers can use the Create Service wizard to create their own Web Services Description Language (WSDL). A WSDL is an XML-based language used to describe the services a business offers and to provide a way for individuals and other businesses to access those services electronically. See the Creating Services topic for more detailed information.
