Optional Add-On Assertions Overview

 NOTE: For information on purchasing an add-on assertion, contact Layer 7 Technologies as outlined in Technical Support and Contact Information.

Three assertions are available as optional add-ons to the basic SecureSpan™ Solution functionality:

  • SiteMinder™ Authentication

The SiteMinder™ authentication assertion — called “SiteMinder Protected Resource” — appears in the Authentication category of the [Assertions] tab. The assertion instructs the SecureSpan™ Gateway to delegate the authentication and authorization tasks required to gain access to a protected service to the Netegrity SiteMinder™ Policy Server. See Using the SiteMinder™ Authentication Assertion for more information

  • Tivoli Access Manager (TAM) Authentication

The Tivoli Access Manager (TAM) authentication assertion — called “Tivoli Access Manager Assertion” — appears in the Authentication category of the [Assertions] tab. The assertion instructs the Gateway to delegate the authentication and authorization tasks required to gain access to a protected service to the IBM® Tivoli Access Manager solution. See Using the TAM Authentication Assertion for more information

  • JMS Message Routing

The JMS routing assertion — called "JMS Routing" — appears in the Message Routing category of the [Assertions] tab. The JMS assertion instructs the Gateway to route service request messages to a selected outbound queue. See Using the JMS Message Routing Assertion for more information.

 

Each assertion requires special configuration and installation procedures in the Gateway before it is installed and/or enabled for use in the SecureSpan™ Manager. For SiteMinder™ and TAM assertion installation and configuration information and instructions, see the SecureSpan™ Solution Add-On Assertion Installation Guide or contact your Administrator. For JMS assertion installation and configuration information and instructions, contact your Administrator.

 

 NOTE: The Administrator is responsible for installing and configuring the add-on assertions in the SecureSpan™ Gateway. You will require specific configuration-related information from the Administrator to use the individual add-on assertions. Read through the information requirements before using the assertions. If you encounter errors during the execution of a security policy that contains one or more add-on assertions, contact your Administrator.
