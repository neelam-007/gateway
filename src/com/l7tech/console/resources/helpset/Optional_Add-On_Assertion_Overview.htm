Optional Add-On Assertions Overview

 NOTE: For information about purchasing an add-on assertion, contact Layer 7 Technologies as outlined in Technical Support and Contact Information. In the SecureSpan™ Manager, the term "identity" includes both users and groups, the term "user" can represent an individual human or machine, and the term "SecureSpan™ Gateway" includes all of the SecureSpan Gateway products.

Four assertions are available as optional add-ons to the basic SecureSpan™ functionality:

  • SiteMinder™ Authentication

The SiteMinder™ authentication assertion — called “SiteMinder Protected Resource” in the Policy Development window — appears in the Access Control category of the [Assertions] tab. The assertion instructs the SecureSpan™ Gateway to delegate the authentication and authorization tasks required to gain access to a protected service to the Netegrity SiteMinder™ Policy Server. See Using the SiteMinder™ Authentication Assertion for more information

  • Tivoli® Access Manager (TAM) Authentication

The Tivoli® Access Manager (TAM) authentication assertion — called “Tivoli Access Manager Assertion” in the Policy Development window — appears in the Access Control category of the [Assertions] tab. The assertion instructs the SecureSpan Gateway to delegate the authentication and authorization tasks required to gain access to a protected service to the IBM® Tivoli® Access Manager solution. See Using the TAM Authentication Assertion for more information

  • Symantec™ Anti-Virus

The Symantec™ anti-virus assertion — called "Symantec Virus Scanning" in the Policy Development window — appears in the Message Validation/Transformation category of the [Assertions] tab. The assertion instructs the SecureSpan Gateway to send messages and attachments to an existing Symantec™ Virus Scanning engine. See Using the Symantec™ Anti-Virus Assertion for more information

  • JMS Message Routing

The JMS routing assertion — called "JMS Routing" in the Policy Development window — appears in the Message Routing category of the [Assertions] tab. The JMS assertion instructs the SecureSpan Gateway to route service request messages to a selected outbound queue. See Using the JMS Message Routing Assertion for more information.

 

Each assertion requires special configuration and installation procedures in the SecureSpan Gateway before it is installed and/or enabled for use in the SecureSpan™ Manager. For SiteMinder,TAM, and Symantec assertion installation and configuration information and instructions, see the "SecureSpan™ Add-On Assertion Installation Manual" or contact your Administrator. For JMS assertion installation and configuration information and instructions, contact your Administrator.

 

 NOTE: The Administrator is responsible for installing and configuring the add-on assertions in the SecureSpan™ Gateway. You will require specific configuration-related information from the Administrator to use the individual add-on assertions. Read through the information requirements before using the assertions. If you encounter errors during the execution of a policy that contains one or more add-on assertions, contact your Administrator.
