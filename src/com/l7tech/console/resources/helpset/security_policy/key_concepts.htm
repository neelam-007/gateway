Key Concepts

The Manager refers to many technical terms commonly used in the Web services sphere. Since there can be different meanings for the same terms, definitions are provided for the terms listed below. These definitions reflect the meaning of the words in the context of the Manager.

Authentication Method: List of protocols that can be used for credential authentication in a policy.  The protocols included are: HTTP Basic, HTTP Digest, HTTP client certificate, WS Token Basic, and WS Token Digest.

Credentials: Refers to the username and password used for a given role or user-based permission scheme. Username is used by convention and does not imply that only individuals.

Credential source assertion: When a request that requires authentication comes to a published service, the request must include the credentials. There are several different schemes to do this, each with varying levels of security - HTTP Basic, HTTP Digest, SSL Client Certificate, WS Basic, WS Secure.

Gateway: An abbreviated form of the SecureSpan Gateway. This refers to the network appliance that enforces all provider-side security and integration policies.

Grouping assertion: “All” and “One or More” assertions that group multiple assertions to represent more complex business and technical rule logic.

Identity provider: A machine or entity that validates authentication credentials. The two options available are the internal identity provider and the LDAP directory.

Internal groups: Groups of internal users that can be added to a policy.

Internal users: Individual users and their authentication credentials.

LDAP: Lightweight Directory Access Protocol. LDAP is a software protocol enabling anyone to locate organizations, individuals, and other resources such as files and devices in a network.

Message routing: After a decision is made based on the policy, the message must be sent to the protected service that handles the request. The message routing assertion defines where the message is sent and what credentials the service requires.

Policy: is the rule set for accessing a Web service. This rule set is comprised of a series of assertions such as authentication method, users, and message routing.

Policy assertion: Each individual component of a policy.

Protected service: A service that is located inside a corporate security domain and is unavailable to a client application.

Published service: When a policy refers to a protected service, making the service available to users, the service becomes a "published" service.

SOAP: Simple Object Access Protocol. SOAP is a method used by a program running in one operating system to communicate with a program in the same or another kind of an operating system. SOAP uses HTTP and XML as the mechanisms for information exchange.

TLS/SSL assertion: A transport assertion that requires the message to reach the Gateway via the Transport Level Security or Secure Sockets Layer (HTTPS) for the message to evaluate as true.

Web service: Services that are made available from a business's web server and are generally used for application to application communication.

XML security: XML-based encryption and signatures of standard messages. This uses x509 certificates to encrypt a message digest for "sign only" security or exchanges a symmetric key to encrypt the entire message for "sign and encrypt" security.
