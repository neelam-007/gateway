Policy Examples

For administrators that are new to creating policies for Web services, the following scenarios offer simple to complex examples. Brief summaries are provided to explain the logic behind the policy and the assertions used. The scenarios are listed from least complex to most complex.

Note: These examples are for reference purposes only and should not be duplicated.

Scenarios

Scenario 1: One authentication method with no credential authentication (identities)

Scenario 2: One authentication method with credential authentication (identities)

Scenario 3: Multiple authentication methods with credential authentication (identities)

Scenario 4: Multiple authentication methods with credential authentication (identities) and response security

 

Scenario 1: One authentication method with no credential authentication (identities)

This is the simplest security policy. One authentication method is used to access the Web service. No identities (users or groups) have been defined, potentially exposing the service to public access. Requests are routed to the defined target.

This policy includes:

• One authentication method (HTTP BASIC authentication)

• One routing Web service target

 

 

Scenario 2: One authentication method with credential authentication (identities)

This is a slightly more complex policy.  One authentication method is used to access the Web service. Only the users and groups with identities listed have access to the service. Requests are routed to the defined target.

• One authentication method (HTTP BASIC authentication)

• Credential authentication (identities and groups)

• One routing Web service target

 

 

Scenario 3: Multiple authentication methods with credential authentication (identities)

This policy supports multiple authentication methods.  The Web service can be accessed via different authentication methods when the user identities also evaluate to true. The service is only accessible to users and/or groups that are listed in the policy. Requests are routed to the defined target.

• Multiple authentication methods

• Credential authentication (identities and groups), grouped by authentication methods

• One routing Web service target

 

 

Scenario 4: Multiple authentication methods with credential authentication (identities) and response security

This is the most complicated policy example. The Web service can be accessed via different authentication methods when the user identities also evaluate to true. The service is only accessible to users and/or groups that are listed in the policy. The Web service responds to requests with asserted security (sign and encrypt). Requests are routed to the defined target.

• Multiple authentication methods

• Credential authentication (identities and groups), grouped by authentication methods

• One routing Web service target

• Response security assertion (XML response security - sign and encrypt)
