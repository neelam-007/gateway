Manage Certificates

NOTE: The generic "Trusted Gateway A" and "Federated Gateway B" references are used in the identity bridging examples, workflows, and instructions. The Trusted Gateway A is the certificate authority (CA) that issues and manages security credentials and that is responsible for authentication. The Federated Gateway B is the Web service provider that is responsible for authorization.

In an identity bridging configuration, certificates are imported into the Federated Gateway B trust store. The combination and purpose of certificates in the trust store is determined by the chosen credential source and optional configuration elements defined in Workflow Using an X.509 Certificate or Workflow Using SAML. In accordance with the workflow instructions, certificates belonging to the Trusted Gateway A authentication domain will typically be imported into the Federated Gateway B authorization domain using the SecureSpan™ Manager's Add Certificate Wizard (see Add and Edit Certificates).

 

For information on Federated Identity Providers, see Manage Federated Identity Providers. For information on the certificates required in each security domain in an identity bridging configuration, see System Requirements.
