Identity Providers Overview

 NOTE: In the SecureSpan™ Manager, the term "identity" includes both users and groups, and the term "user" can represent an individual human or machine.

Three types of identity providers are used in the SecureSpan™ Manager: External Identity Providers, Federated Identity Providers, and an Internal Identity Provider.

External Identity Providers

Users can configure and manage one or more External Identity Providers (EIPs). An EIP is an LDAP (Lightweight Directory Access Protocol) connector that is used for authentication purposes. For information on configuring an EIP, see Add and Edit External Identity Providers.

Federated Identity Providers

 NOTE: The documentation pertaining to Federated Identity Providers is discussed in Federated Identity Providers Overview in the Identity Bridging section of this Help System.

A Federated Identity Provider (FIP) is exclusively used in a SecureSpan™ Solution identity bridging configuration. Essentially, the FIP allows one security domain to authorize requests containing credentials originating from another security domain. For more information about the role of FIPs in identity bridging, see Federated Identity Providers Overview. For more information about identity bridging, see Identity Bridging Overview.

Internal Identity Provider

A single Internal Identity Provider (IIP) is pre-configured as the authentication database inside the SecureSpan™ Gateway. The SecureSpan™ Manager allows you to modify the users and groups in the IIP. For information on adding users and/or groups to the IIP, see Add and Edit Internal Identity Provider Users and Groups.

Searching Identity Providers

The Search Identity Providers form permits External Identity Provider, Federated Identity Provider, and Internal Identity Provider searches. See Search Identity Providers for more information.
