Configure and Edit Security Policies

NOTE: The type and order of assertions are important when constructing a valid Web service security policy. Become familiar with the policy and assertion information and rules in Security Policies Overview and Rules. For in-depth information on the category and individual assertions used in a security policy, see Security Policy Assertion Overview.

In the SecureSpan™ Manager, a security policy is constructed for a Web service by moving assertions from the [Assertions] tab into the Policy Development window. The policy assertions in the Policy Development window are organized into a logical tree-structure that conforms to the policy and assertions rules defined in Security Policies Overview and Rules. A security policy is configured in four general steps:

  1. Select the Web service.

  2. Add and configure the assertions.

  3. Organize the assertions.

  4. Validate the security policy.

A logical security policy must be constructed for each published Web service in the Manager. See Security Policy Assertion Overview for information about the assertions in the [Assertions] tab.

 

NOTE: For information on the interface areas related to Web services and security policies, see SecureSpan™ Manager Interface.

Configure a Security Policy

TIP: Reference the security policy examples in Security Policy Examples when constructing your own policies. Periodically click the [Save] button in the Policy Tool Bar during the configuration process.

The SecureSpan™ Manager Policy Development window is the repository for the assertions used to develop a security policy. Most assertions require configuration either before or after being added to the Policy Development window. See Security Policy Assertion Overview for individual assertion information and instructions by assertion category. When all or most of the required assertions have been added and configured in the Policy Development window, then they must be organized into the logical structure defined in the individual assertion instructions and/or in Security Policies Overview and Rules. Once organized, proceed to Validate a Security Policy to finalize the security policy.

 

TIP: Before constructing a security policy, ensure that the Policy Validation Messages window is enabled. To enable the window, select [View] > Policy Messages from the Main Menu (check box checked). The Policy Validation Messages window provides you with policy and assertion-level confirmation, warning, or error messages that you can use during the configuration process. See Validate a Security Policy for more information.

  1. Access the Policy Development window for the target Web service by one of two methods:

NOTE: You can only access and configure a security policy for a Web service that is published and enabled. A red "X" appears through a disabled Web service icon in the [Web Services] tab. See Publish Web Services for more information on publishing a Web service. See Manage Published Web Services for information on enabling a disabled Web service.

    • Right-click the Web service name in the [Web Services] tab. A drop-down menu appears. Select Policy Assertions. Or,

    • Double-click the Web service in the [Web Services] tab.

  1. The Policy Development window appears. Add assertions to the security policy by one of two methods:

NOTE: When you publish a Web service using the Publish Web Service Wizard or Create WSDL Wizard, the SecureSpan™ Manager automatically adds the Web service URL specified during the publication process as an HTTP(S) Routing assertion in the Web service's Policy Development window. For information about routing assertions, see Configure and Edit Message Routing Assertions.

    • Drag and drop the assertion from the [Assertions] tab into the Policy Development window, or  

    • Highlight the assertion in the [Assertions] tab and click the  (Add Assertion) button in the Assertion Tool Bar.

NOTE: A configuration form will appear for some assertions before they are added to the Policy Development window. Other assertions will have to be configured after they are added, and some assertions will not require configuration. Before adding a particular assertion to the Policy Development window, see the Security Policy Assertions Overview for the category page that contains information about the assertion.

  1. Repeat to add and configure additional security policy assertions. When completed, organize the assertions using the following tools:

NOTE: To view a read-only presentation of the assertions by each configured identity (Grant Access to Users and Groups assertion), click the [Identity View] button on the Policy Tool Bar. To switch back to the Policy Development window, click the [Policy View] button.

Move an Assertion Up

    • To move an assertion up in the policy, either:

      • Click the  (Move Up) button in the Assertion Tool Bar, or

      • Right-click the target assertion. A drop-down menu appears. Select Move Assertion Up.

    • The assertion moves up one line in the policy.

Move an Assertion Down

    • To move an assertion down in the policy, either:

      • Click the  (Move Down) button in the Assertion Tool Bar, or

      • Right-click the target assertion. A drop-down menu appears. Select Move Assertion Down.

    • The assertion moves down one line in the policy.

Delete an Assertion

    • To delete an assertion from the policy, either:

      • Click the  (Delete) button in the Assertion Tool Bar, or

      • Right-click the target assertion. A drop-down menu appears. Select Delete Assertion.

    • A pop-up confirmation message appears. Click [Yes].

Add an "All" Folder

To organize one or more assertion(s) into an "All must be true" assertion folder:

    • Right-click any assertion. A drop-down menu appears. Select Add 'All' Assertion

    • An "All assertions must evaluate to true" folder is added to the Policy Development window. To apply the requirement to an assertion, drag and drop the assertion from the Policy Development window into the folder. Repeat to add additional assertions.

NOTE: For more information about using assertion folders, see Security Policies Overview and Rules. To remove an assertion from the folder, drag and drop the assertion from the folder back into the Policy Development window.

Add a "One or More" Folder

To organize one or more assertion(s) into an "One or more must be true" assertion folder:

    • Right-click any assertion. A drop-down menu appears. Select Add 'One or More' Assertion

    • An "At least one assertion must evaluate to true" folder is added to the Policy Development window. To apply the requirement to an assertion, drag and drop the assertion from the Policy Development window into the folder. Repeat to add additional assertions.

NOTE: For more information about using assertion folders, see Security Policies Overview and Rules. To remove an assertion from the folder, drag and drop the assertion from the folder back into the Policy Development window.

  1. In the Policy Tool Bar, click [Save] to save the security policy. To perform a final validation check on the policy, proceed to Validate a Security Policy. To save the policy organization and assertions as a template in the Policy Templates section of the [Assertions] tab, see Save and Use Policy Templates.

Validate a Security Policy

The SecureSpan™ Manager provides users with two sources of security policy validation:

  • Automatic feedback messages in the Policy Validation Messages window when [View] > Policy Messages from the Main Menu is enabled (check box checked)

  • Final policy validation using the [Validate] button on the Policy Tool Bar.

Configuration Process Feedback

The Policy Validation Messages window provides users with valuable feedback messages during the security policy configuration process. The window displays three types of messages:

Confirmation Messages

Confirmation messages confirm the addition and proper configuration of an assertion.

Warning Messages

Warning messages warn that an assertion:

  • Has been duplicated

  • Requires the presence of another assertion to be valid

  • Has been improperly configured.

Error Messages

Error messages warn that an assertion:

  • Has not been configured

  • Has been improperly located in the policy

  • Is in conflict with another assertion.

Use the Policy Validation Messages window as a source of guidance during the security policy configuration process. When the policy is completed, perform a final validation check as outlined in Final Policy Validation.

Final Policy Validation

The [Validate] button on the Policy Tool Bar performs the final validation check on a completed security policy. When clicked after the configuration process, one of two types of messages will appear in the Policy Validation Messages window:

  • A "Policy validated ok" message indicates that the policy is error free

  • One or more error messages indicates that the policy contains errors. Do the following:

    • Re-organize the policy assertions so that they conform to the policy and assertion organization, dependency, and combination rules outlined in Security Policies Overview and Rules

    • Ensure that the individual assertions are configured properly. See the assertion specific instructions outlined in the category pages discussed in Security Policy Assertions Overview

    • If validation errors still occur, contact a Technical Support Representative as outlined in Technical Support and Contact Information.

Edit a Security Policy

NOTE: Ensure that policy changes conform to the policy and assertion rules outlined in Security Policies Overview and Rules. When editing a security policy, use the confirmation, warning, and/or error messages in the Policy Validation Messages window as a guide. To enable the Policy Validation Messages window, select [View] > Policy Messages from the Main Menu (check box checked). See Validate a Security Policy for more information.

  1. Access the Policy Development window for the target Web service by one of two methods:

NOTE: You can only access and configure a security policy for a Web service that is published and enabled. A red "X" appears through a disabled Web service icon in the [Web Services] tab. See Publish Web Services for more information on publishing a Web service. See Manage Published Web Services for information on enabling a disabled Web service.

    • Right-click the Web service name in the [Web Services] tab. A drop-down menu appears. Select Policy Assertions. Or,

    • Double-click the Web service in the [Web Services] tab.

  1. The Policy Development window appears:

TIP: Periodically click the [Save] button in the Policy Tool Bar during the editing process.

    • If necessary, re-organize the assertions as outlined in Step #3 in Configure a Security Policy

    • If necessary, modify an editable assertion. See Security Policy Assertions Overview for individual assertion information and instructions by assertion category.

NOTE: The changes that you make to an individual assertion may affect the validity of the security policy. Reference the policy and assertion rules in Security Policies Overview and Rules before you edit the assertion.

  1. In the Assertion Tool Bar, click [Save] to save the security policy. To perform a final validation check on the policy, proceed to Validate a Security Policy. To save the policy organization and assertions as a template in the Policy Templates section of the [Assertions] tab, see Save and Use Policy Templates.
