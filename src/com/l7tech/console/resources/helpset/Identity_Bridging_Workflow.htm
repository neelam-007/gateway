Identity Bridging Workflow Using SAML

 NOTE: You will need the “SecureSpan™ Bridge User Manual” to complete the following instructions. The instructions are not complete to the referenced forms. To fully utilize the guidance that this workflow provides, reference this topic when configuring each requirement in the SecureSpan™ Manager and SecureSpan™ Bridge.

Configure identity bridging using the SAML credential source as follows:

 

 NOTE: The generic "Trusted Gateway A" and "Federated Gateway B" references are used in the identity bridging examples, workflows, and instructions. The Trusted Gateway A is the certificate authority (CA) that issues and manages security credentials and that is responsible for authentication. The Federated Gateway B is the Web service provider that is responsible for authorization. In order to configure the SecureSpan™ Bridge in an identity bridging configuration using the SAML credential source, the Trusted Gateway A and the Federated Gateway B must be SecureSpan™ Gateways residing in the respective authentication and authorization domains.

 

  1. Confirm that the system requirements outlined in Identity Bridging Requirements are met for configuring identity bridging with SAML.

  2. In the SecureSpan™ Manager (Federated Gateway B domain), connect to the Trusted Gateway A as outlined in Connect to the SecureSpan™ Gateway.

  3. Add a new Internal Identity Provider user in the Trusted Gateway A as outlined in Add and Edit Internal Identity Provider Users and Groups. For example, "user A."

  4. Disconnect from the Trusted Gateway A, then connect to the Federated Gateway B.

  5. Import the Trusted Gateway A server certificate (issued by the Trusted Gateway A's CA) into the Federated Gateway B trust store as outlined in Add and Edit Certificates. In the Specify Certificate Options screen of the Add Certificate Wizard, select the Signing SAML Tokens check box.

  6. Create a new Federated Identity Provider (FIP) as outlined in Add and Edit Federated Identity Providers. Configure the new FIP for the SAML credential source as follows:

    • In the Enter Provider Information screen, check the SAML Token check box for the Credential Source Type Allowed item

    • In the Select the Trusted Certificates screen, click [Add] to attach the Trusted Gateway A server certificate imported in Step #5 above

    • Complete the configuration process as outlined in Add and Edit Federated Identity Providers.

  1. Create a new federated user as outlined in Add and Edit Federated Identity Provider Users and Groups:

    • One new federated user must have a subject DN in the format “CN=<login>,” where <login> is the login ID of the Trusted Gateway A Internal Identity Provider (IIP) user (for example, "user A") configured in Step #3 above. In the Create Federated User form:

      1. Enter the subject DN for the Trusted Gateway A IIP user, such as "CN=user A," into the X509 Subject DN field.

 NOTE: In the Create Federated User form, the X509 Subject DN must match the "Issued to" value in an imported certificate.

      1. Complete the configuration process as outlined in Add and Edit Federated Identity Providers.

    • Add additional required federated users, federated groups, and/or virtual groups to the FIP as outlined in Add and Edit Federated Identity Provider Users and Groups.

  1. Configure a policy with the SAML Assertion for the shared Web service as outlined in Configure SAML Policies.

  1. In the SecureSpan™ Bridge, use the Full Identity Bridging method to add two SecureSpan™ Gateway Accounts as outlined in Configure SecureSpan™ Gateway Accounts.

 TIP: For in-depth information about SecureSpan™ Gateway Accounts in the SecureSpan™ Bridge, see Manage SecureSpan™ Gateway Accounts in the "SecureSpan™ Bridge User Manual."

  1. When completed, consume the shared Web service as outlined in Consume Shared Web Services.
