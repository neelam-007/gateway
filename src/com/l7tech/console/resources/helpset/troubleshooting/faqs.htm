FAQ Answers

Why does my service policy look different from the last time?

The Toggle view button in the Policy pane provides two different policy views: policy and identities.  When the policy view is active, the button appears flat (unpressed) and all policy assertions are visible.  When the identities view is active, the button appears sunken (pressed) and the identity's assertions are visible.  You can toggle these views by clicking onto the Toggle view button.

 

I do not like the look and feel of the user interface, can I customize the appearance to suit my needs?

The Preferences button launches a dialog box where the look and feel of the user interface can be changed. Disconnect from the Manager, change the Preferences, and then reconnect.

 

How do I view the service policy details?

You can view the service policy details by right-clicking on the service name. Select Service policy from the pop-up menu. The service policy details will display in the Policy pane on the right-hand side of the user interface.

 

Can I cancel out after revoking a certificate?

No, you cannot cancel from a user certificate revoking action.  This action takes place immediately after you choose to revoke a user certificate.

 

Why does nothing happen after I modify a service policy?

If you have multiple services defined, ensure that you are modifying the correct service. The service's Bindings folder may help you to identify the actual Web service. The name of the Service policy and the Service URL (in the routing properties) may be misleading, especially if they were incorrectly defined.

 

Why did the identities under the LDAP directory disappear?

The LDAP may have been re-configured by the administrator.  Disconnect from the Manager, then connect again.

 

When performing a search on identity providers, what wild cards can I use?

The available search wild cards are: “?” for a single character, and “*” for a substring.

 

After revoking a user’s certificate, the Gateway does not authorize the password.  Why?

When revoking a user certificate, the user’s password will be reset. The administrator must change the user's password after revoking the certificate.

 

Why can't I see the logs and statistics tabs after I select the Gateway Log and Statistics items in the View menu?

This may be a result of the sizing of your application window. Maximize the application window to display the Logs and Statistics tabs in the bottom of the user interface.

 

Why does the inactivity timeout not take effect?

The Inactivity Timeout option is not active when Gateway log and/or Statistics are displayed and the auto-refresh option is selected. These features stop background activities (such as the inactivity timeout) in order to continuously refresh information.

 

Why does the user certificate not revoke after I have changed the user password?

Ensure that the new password is different from the old password.  The certificate only revokes when the new password is different from the previous password.

 

Why doesn't the sliding scale Manager Log work?

The sliding scale change is to control the future logs. The scale does not filter the past logs.

 

I have deleted a user identity from a service policy.  Why can the user still access the service?

Duplicate entries of the identity may exist in the service policy and only one has been deleted. If a service policy has a long list of identities, you may want to create a group in order to minimize security issues caused by human error.

 

I added a user/group to a service policy, but the user could not be authenticated.  Why?

Verify that the service policy has been validated and saved.

 

After entering the correct credentials for a protected Web service (requiring Identity, Password, and Realm), the Web service cannot be accessed. Why?

Verify that the service policy was saved after updating the policy with the Web service authentication information.

 

What parameters should I use when creating a new external identity provider such as LDAP or MS Active Directory?

LDAP Identity Provider

Most LDAP providers can support anonymous binding and do not require search credentials. As a result, you can omit values for the Bind DN and Bind password.

For example:

Identity Provider: myLDAP
Provider type: LDAP
LDAP host url:  ldap://ldapserver:389
search base: dc=layer7-tech, dc=com

MS Active Directory Identity Provider

Some LDAP providers such as MS Active Directory do not support anonymous binding. The Manager needs to capture the search credentials. The search credentials are usually the username/password for an account that has search permission (not to be confused  with the LDAP admin account).

For example:

Identity Provider: myMSAD
Provider type: MS Active Directory
LDAP host url: ldap://mail.abc.com:3268
search base: ou=ABC Company Users,dc=ABC,dc=LOCAL
Bind DN: mybrowse
Bind Password: mypassword

 

Why are my Agent users receiving an “Error in Assertion Processing” message when the Manager validates the policy as OK?

When validating a service policy, the routing and Web service location are not checked or validated. Only the composition of the policy is validated. Ensure that the Web service policy is sending requests to the correct URL. If the routing URL is incorrect, amend the service policy by right-clicking on the routing assertion, selecting Routing properties, and changing the Service URL.

 

I can see the user identity in the service policy, but the user cannot authenticate.  Why?

Check to see if the user has been removed from the identity provider list. The user may have been removed from the identity provider after being added to a service policy.

 

Why can't I view the users in my identity provider?

When a user attempts to configure an identity provider with incorrect information (incorrect URL, search base, or bind DN/password).  The Gateway is not able to access the identity provider to retrieve information and returns the error below.

 

Severity: SEVERE

Message: [LDAP: error code 1 - 00000000: LdapErr: DSID-0C0905FF, comment: In order to perform this operation a successful bind must be completed on the connection., data 0, vece

Class: com.l7tech.identity.ldap.AbstractLdapUserManagerServer

Method: findAllHeaders

 

Followed by:

Severity: SEVERE

Message: Listing users yielded no results.

Class: com.l7tech.identity.IdProvConfManagerServer

Method: test

 

When do I use Web service credentials?

Some Web services are anonymous, while other Web services are protected with authentication. Web service credentials (identity, password, realm) are required if the Web service to be consumed is protected with authentication. The SecureSpan Manager administrator can configure a service policy to use Web service credentials on the routing assertion. Right-click the routing assertion and select Routing properties to enter the identity, password, and realm. Be sure to save the policy for the changes to take affect.

 

How do I set up a Web service that is protected with authentication on Internet Information Services (IIS)?

To set up a Web service on IIS:

  1. Open up Internet Information Services (IIS) on the web server.

  2. Select your web service, then go to Directory Security to turn off the anonymous credential requirement.

  3. Select Basic authentication to turn on credential requirement. Sample credentials are -  identity: myName, password: myPassword, realm: myCompany.

Note: The identity must be a valid user on the server.

  1. Apply the changes.

  2. Use the credentials (e.g. identity: myName, password: myPassword, realm: myCompany) to consume the web service.

 

Error Message Answers

There was an error updating Group SSGAdmin

This is a valid error. The administrator is forbidden to remove all members from the “SSGAdmin” group. One user must remain as an administrator to be able to log in to the SecureSpan™ Manager.

 

Error saving service and policy

If multiple Manager applications attempt to modify the same policy simultaneously, conflicts can occur. The administrator should either disconnect the additional Managers or refresh the view to incorporate other changes. Once done, the administrator can try saving again.

 

Cannot connect to the service https://<hostname>:8443.  Please check your connection settings and application log for details.  If problem persist, contact your network administrator.

Ensure that the URL you have entered for the Gateway in the Manager is correct. If you are using a hostname (not an IP address) for the Gateway URL, ensure that the hostname is reachable. To do this, ping the Gateway using the hostname ("ping [hostname]").

If the settings are correct and the administrator is unable to ping the server, there may be another problem with resolving the hostname. Enter the hostname information into the Manager server's hosts file. The hosts file can be located at:

  • Windows systems: C:\[windows directory]\system32\drivers\etc\hosts

  • Linux systems:  /etc/hosts

Ensure that the following entry appears in that file: IPAddress MyGateway MyGateway.MyCompany.com.

"IPAddress" is the IP address of the server, "MyGateway" is the local host name, and "MyGateway.MyCompany.com" is the full server name. An example would be: 10.10.10.10 servername servername.company.com.

 

Admin service not available at host [gateway hostname]

Try to ping the Gateway hostname from the machine running the Manager. If the ping does not receive a response, check to see if the Gateway's IP address has been modified or if the Gateway resides behind a firewall.

 

Access control specified without authentication scheme (id assertion without prior cred assertion)

This is an error that appears when validating a policy. The identity assertion has been placed before the authentication method. In order for the policy to be validated as okay, the authentication method must be position before the identities.

 

Duplicate identity (two id assertions in same path)

This is an error that appears when validating a policy. Two identical identities have been used in the same assertion folder. Delete one of the duplicate identities.

 

A credential assertion requires client certs. Only internal identities support client certs.

This is an error that appears when validating a policy. The credential assertion being used requires the HTTP client certificate authentication method. When using client certificates, only identities from the internal identity provider can be used.

 

A credential assertion requires digest authentication. Only internal identities support digest authentication.

This is an error that appears when validating a policy. The credential assertion being used requires the HTTP Digest authentication method. When using client certificates, only identities from the internal identity provider can be used.

 

XML Response Security must occur after routing.

This is an error that appears when validating a policy. The XML Response Security assertion must be placed after the routing assertion.

 

HttpClientCert requires to have SSL transport.

This is an error that appears when validating a policy. The HTTP client certificates authentication method requires SSL Transport to be used in the policy (as either optional or required).

 

HttpClientCert requires to have SSL transport (not forbidden).

This is an error that appears when validating a policy. The HTTP client certificates authentication method requires SSL Transport to be used in the policy (as either optional or required).

 

The SecureSpan Manager version does not match with the SecureSpan Gateway version! Contact your system administrator for the correct version. Current Manager version is n.m build xxxx.

This error message appears when you attempt to log on to the SecureSpan Gateway with an incompatible version of the SecureSpan Manager. The Manager and the Gateway must be the same version number for the log on to be successful.

 

System.NullReferenceException: Object reference not set to an instance of an object

Verify if the routing properties' service URL is correct. It must point to the URL of the Web service.

 

Error 500 Internal Server Error

When all SOAP requests return this HTTP error message, check the Gateway port numbers defined for the Web service port and the SSL port.  By default, the SecureSpan Agent is configured to send outgoing requests to the Gateway to standard ports - Web Service port 8080 and SSL port 8443.  If the Gateway is listening to non-standard port numbers, you can configure the SecureSpan Agent to use these ports.

To configure the Agent:

  1. In the Agent GUI, select File > Properties. The Gateway Properties dialog box appears.

  2. Select the Network tab.

  3. Under Outgoing requests to the Gateway, select Gateway requires custom ports. Type the custom port numbers into the Web Service port and the SLL port fields.

  4. Click OK.

 

Server did not recognize the value of HTTP Header SOAPAction: <URL>

This error message appears when the published service's WSDL is inconsistent with the service URL where the request is routing. Check with your SecureSpan Gateway administrator.

In the Manager: Check whether the routing URL (right-click on the routing assertion and select Routing properties) is compatible with the published service policy WSDL (right-click on the service and select View Service Wsdl).

If they are not compatible or you want the routing URL to use the original one, reset the routing assertion service URL. To do this, right-click onto the policy's routing assertion and select Routing properties from the pop-up menu. Click on the Default button beside the Service URL, then click OK.
