FAQ Answers

Why does my service policy look different from the last time?

The Toggle view button in the Policy pane provides two different policy views: policy and identities.  When the policy view is active, the button appears flat (unpressed) and all policy assertions are visible.  When the identities view is active, the button appears sunken (pressed) and the identity's assertions are visible.  You can toggle these views by clicking onto the Toggle view button.

 

I do not like the look and feel of the user interface, can I customize the appearance to suit my needs?

The Preferences button launches a dialog box where the look and feel of the user interface can be changed. Disconnect from the Manager, change the Preferences, and then reconnect.

 

How do I view the service policy details?

You can view the service policy details by right-clicking on the service name. Select Service policy from the pop-up menu. The service policy details will display in the Policy pane on the right-hand side of the user interface.

 

Can I cancel out after revoking a certificate?

No, you cannot cancel from a user certificate revoking action.  This action takes place immediately after you choose to revoke a user certificate.

 

Why does nothing happen after I modify a service policy?

If you have multiple services defined, ensure that you are modifying the correct service. The service's Bindings folder may help you to identify the actual Web service. The name of the Service policy and the Service URL (in the routing properties) may be misleading, especially if they were incorrectly defined.

 

Why did the identities under the LDAP directory disappear?

The LDAP may have been re-configured by the administrator.  Disconnect from the Manager, then connect again.

 

When performing a search on identity providers, what wild cards can I use?

The available search wild cards are: “?” for a single character, and “*” for a substring.

 

After revoking a user’s certificate, the Gateway does not authorize the password.  Why?

When revoking a user certificate, the user’s password will be reset. The administrator must change the user's password after revoking the certificate.

 

Why can't I see the logs and statistics tabs after I select the Gateway Log and Statistics items in the View menu?

This may be a result of the sizing of your application window. Maximize the application window to display the Logs and Statistics tabs in the bottom of the user interface.

 

Why does the inactivity timeout not take effect?

The Inactivity Timeout option is not active when Gateway log and/or Statistics are displayed and the auto-refresh option is selected. These features stop background activities (such as the inactivity timeout) in order to continuously refresh information.

 

Why does the user certificate not revoke after I have changed the user password?

Ensure that the new password is different from the old password.  The certificate only revokes when the new password is different from the previous password.

 

Why doesn't the sliding scale Manager Log work.  Why?

The sliding scale change is to control the future logs. The scale does not filter the past logs.

 

I have deleted a user identity from a service policy.  Why can the user still access the service?

Duplicate entries of the identity may exist in the service policy and only one has been deleted. If a service policy has a long list of identities, you may want to create a group in order to minimize security issues caused by human error.

 

I added a user/group to a service policy, but the user could not be authenticated.  Why?

Verify that the service policy has been validated and saved.

 

After entering the correct credentials for a protected Web service (requiring Identity, Password, and Realm), the Web service cannot be accessed. Why?

Verify that the service policy was saved after updating the policy with the Web service authentication information.

 

What parameters should the I use when creating a new external identity provider such as LDAP or MS Active Directory?

LDAP Identity Provider

Most LDAP providers can support anonymous binding and do not require search credentials. As a result, you can omit values for the Bind DN and Bind password.

For example:

Identity Provider: myLDAP
Provider type: LDAP
LDAP host url:  ldap://ldapserver:389
search base: dc=layer7-tech, dc=com

MS Active Directory Identity Provider

Some LDAP providers such as MS Active Directory do not support anonymous binding. The Manager needs to capture the search credentials. The search credentials are usually the username/password for an account that has search permission (not to be confused  with the LDAP admin account).

For example:

Identity Provider: myMSAD
Provider type: MS Active Directory
LDAP host url: ldap://mail.l7tech.com:3268
search base: ou=Layer 7 Users,dc=L7TECH,dc=LOCAL
Bind DN: browse
Bind Password: password

 

Error Message Answers

There was an error updating Group SSGAdmin

This is a valid error.  The administrator is forbidden to remove all members from the “SSGAdmin” group. One user must remain as an administrator to be able to log in to the SecureSpan™ Manager.

 

Error saving service and policy

If multiple Manager applications attempt to modify the same policy simultaneously, conflicts occur.  Close the error message, refresh the service policy information, then continue.

 

Cannot connect to the service https://<hostname>:8443.  Please check your connection settings and application log for details.  If problem persist, contact your network administrator.

Ensure that the IP address and the host name lookup table contains an entry (IP hostname hostname.domain) of the Gateway server you are connecting to.  

For example, if the operating system running the Manager is UNIX, the Gateway IP address is “192.169.12.123”, the hostname is “mygateway”, the domain is “mycompany.com”. The “/etc/hosts” file should contain an entry like: 192.169.12.123 mygateway mygateway.mycompany.com.

 

Admin service not available at host <gateway hostname>

Try to ping the Gateway hostname from the machine running the Manager. If the ping does not receive a response, check to see if the Gateway's IP address has been modified or if the Gateway resides behind a firewall.
