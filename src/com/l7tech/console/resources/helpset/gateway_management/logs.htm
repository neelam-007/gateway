Logs Overview

The Manager offers two set of logs: one that displays the Gateway messages and one that displays the Manager messages. The Gateway log is the key tool for administrative purposes. The Manager log primarily exists as a troubleshooting tool for Layer 7 Technologies.

Gateway Log

The function of the Gateway log is to provide the administrator with a view into the messages that are sent to and from the Gateway. The log provides a tracking tool for administrators to troubleshoot any issues with connectivity and communication.

Administrators can use the sliding scale to filter the messages according to their severity. When the "all" option is selected, the log displays every message from "finest" to "severe." The Details option provides additional message details in the right-hand pane such as the message class and method.

Note: The HTTP Digest authentication method requires two messages for every "attempted request." The first message represents a challenge request before the actual client request message. Other authentication methods count only one message per request.

To view the Gateway log

  1. Select View > Gateway log. The Log tab appears at the bottom of the user interface.

  2. Select the level of message display on the sliding scale.

  3. Select Auto-refresh to receive the most recent log data.

  4. Select Details to view a separate pane with the complete message details.

Message Severity Levels

The messages are sorted in the Gateway log according to their severity. The message severity levels and their meanings are:

Severe: Indicates a serious problem. Administrators should take actions to rectify the problem immediately. e.g. "Attempt to authenticate using unsupported method."

Warning: Provides an alert. Administrators may determine whether a follow-up action is required.  e.g. "No authorization header found."

Info: Provides general information about an event. e.g. "revoking cert and password for user francis."

Fine: Provides information about the progress of an event. e.g. "Request routed successfully."

Finer: Provides additional information about the progress of an event.  e.g. "cert extracted from digital signature for user francis."

Finest: Provides the full details about an event. e.g. "Request already authenticated."

Manager Log

The Manager log displays the messages that are sent between the Manager and the Gateway. Like the Gateway log, this log provides a sliding scale to filter messages according to their severity. This tool is used primarily by the Layer7 troubleshooting staff and is not intended for external use. Access to the Manager log is found under the Help menu.
