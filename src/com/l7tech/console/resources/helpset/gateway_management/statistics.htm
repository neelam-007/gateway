Statistics

The Statistics tab offers basic statistics on the Web service activity that is monitored by the Manager. Administrators can use this data to track the Gateway performance and troubleshoot potential issues with individual services.

The statistics are organized by service name and these names are listed in alphabetical order. The statistics include attempted requests, authorized requests, completed requests, and the rate of completed requests. If the auto-refresh check box is selected, the statistics are continually updated as the data changes.

Note: The HTTP Digest authentication method counts two requests for each "attempted request." The first message represents a challenge request before the actual client request message. Other authentication methods count only one message per request.

To view the Gateway statistics:

  1. Select View > Statistics. The Statistics tab appears at the bottom of the user interface.

  2. Select Auto-refresh to receive the most recent statistics data.

Server Uptime and Average Load

The statistics tab also offers basic information about the server uptime and the average load per minute. The server uptime is counted from the last restart of the Gateway. This feature provides perspective on the number of requests processed within the total server uptime.

The average load per minute tracks the number of work processes completed in the Unix system over the past minute. This number should not exceed 4.0. If the number reaches 4.0, the Gateway is under heavy load and the overall server performance will be slow.
