Product Architecture Model

The Gateway consists of a high performance server executing core messaging, security and management software. Typically residing inside the DMZ, the Gateway provides a high availability solution and supports an active standby configuration with database sync capability.

Services can be centrally managed and audited through a management application - the Manager - linked to the Gateway. The Manager resides outside of the DMZ to augment the security of the Gateway. The Manager allows administrators to de-couple policy control from a service’s programmed business logic so that no manual configuration of the service itself is required.  

The client-side Agent streamlines the process of coordinating security between Web services and their clients. Residing non-invasively on any Linux, Solaris or .NET application platform, the Agent automates the negotiation of both transport and message-level security. Security policies defined through the Gateway are retrieved by the Agent. Messages from the Agent are then automatically formatted with all of the security parameters required to be trusted and processed by the Gateway.
