Consume Shared Web Service

NOTE: The generic "Trusted Gateway A" and "Federated Gateway B" references are used in the identity bridging examples, workflows, and instructions. The Trusted Gateway A is the certificate authority (CA) that issues and manages security credentials and that is responsible for authentication. The Federated Gateway B is the Web service provider that is responsible for authorization. Before consuming the Web service, ensure that it is published and enabled with a completed policy. See Manage SOAP Web Services, Manage Published Services, and Configure and Edit Policies for more information.

The final step in the identity bridging-configuration process is the initial consumption of the shared Web service.

 

The consumption of the shared Web service takes place through one of the SecureSpan™ Bridge Gateway Accounts configured in Configure SecureSpan™ Gateway Accounts. If the Full Identity Bridging method was used to configure the SecureSpan Bridge, then the shared Web service is consumed through the Federated Gateway Account, or Federated Gateway B URL. If the Ad-Hoc Identity Bridging method was used to configure the SecureSpan Bridge, then the shared Web service is consumed through the Trusted Gateway Account, or Trusted Gateway A URL.

 

The initial consumption of a shared Web service tests the configuration between the Trusted Gateway A and the Federated Gateway B, or the authentication and authorization domains. A confirmation message will appear during the first connection to the shared Web service asking if you want to trust the SecureSpan™ Gateway using the server certificate as displayed:

  • Click [Trust] to trust the certificate. The confirmation message will not appear during subsequent connections

  • Click [Cancel] to distrust and reject the certificate and cancel the Web service consumption process.
