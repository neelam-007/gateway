Add and Edit Federated Identity Provider Users and Groups

In an identity bridging configuration, the federated users, groups, and virtual groups added to the Federated Identity Provider (FIP) serve to authorize corresponding users, groups, or credential patterns in other trust domains. Each FIP can contain zero or more federated users, groups and/or virtual groups:

Federated Users

A federated user contains a number of attributes relating to users in other trust domains, including:

  • A DN (distinguished name). The subject of the signed certificate in an incoming request must exactly match the DN of the federated user

  • A login that may be found in the NameIdentifier of an incoming SAML NameIdentifier with the “windowsDomain” format. (SAML credential source only. See Workflow Using SAML)

  • An email address, if applicable, that may be found in the NameIdentifier of an incoming SAML assertion with the “emailAddress” format. (SAML credential source only. See Workflow Using SAML)

  • An X.509 certificate that may be found in an incoming WS-Security X.509 BinarySecurityToken or HTTPS client certificate. (Self-signed or otherwise explicitly trusted. See Workflow Using an X.509 Certificate).

 

Only request credentials that exactly match the federated user DN and other information will pass the corresponding user assertion that is required to gain Web service access.

Federated Groups

Federated groups allow administrators to organize federated users into groups that have local relevance.

Federated Virtual Groups

A federated virtual group is a pattern that incoming request credentials must match. The pattern may include:

  • The subject of a signed certificate

  • A regular expression describing a pattern that the NameIdentifier value in incoming SAML tokens (with the corresponding NameIdentifier format) must match (SAML credential source only. See Workflow Using SAML)

  • A set of attribute names and values that must be present in incoming SAML tokens that have an AttributeStatement. The allowable attribute names must have been previously registered with the FIP. (SAML credential source only. See Workflow Using SAML).

 

In order to authorize users in a virtual group, a Federated Identity Provider (FIP) must contain the CA root certificate of the issuer of the certificates belonging to the identities in the virtual group. See Add and Edit Certificates for information on adding a certificate to the trust store, and see Add and Edit Federated Identity Providers for information on adding a certificate to an FIP.

Add a New Federated User

 TIP: View the properties of the signed certificates attached to the Federated Identity Provider before configuring the federated user. Note the values, such as the "Issued to" value, that must be configured in the Create Federated User form. See Add and Edit Certificates for instructions.

In the SecureSpan™ Manager, a new user is added to the Federated Identity Provider (FIP) with the Create Federated User form (basic information) and the optional User Properties form.

 

  1. Right-click the FIP name in the [Identity Providers] tab. A drop-down menu appears. Select Create User.

  2. The Create Federated User form appears. In the X509 Subject DN field, enter the DN value that incoming certificate-based credentials must match in order to be authorized as this federated user. For example: CN=user A.

  3. Optional: To allow this user to be authorized based on incoming SAML tokens with “windowsDomain” NameIdentifier formats, enter the value into the Login field.

  4. Optional: To allow this user to be authorized based on incoming SAML tokens with the “emailAddress” NameIdentifier format, enter the value into the Email field.

  5. Optional: If desired, replace the user name that was derived from the Subject DN and placed in the User Name field with a name unique to the federated user.

 NOTE: The name in the User Name field will appear in the Search Identity Provider form when searching and/or adding federated users to a policy. The User Name is a human readable value that does not impact the usage or validity of the federated user in an identity bridging configuration.

  1. Do one of the following:

    • If you do not want to specify additional user information, then click [Create] without checking the Define Additional Properties check box. The user is added to the FIP. See Search, Edit, or Delete a Federated User, Group, or Virtual Group to view, edit, and/or delete the user at any time

    • To specify the full name, define group membership, and/or import the user's client certificate, proceed to Configure Federated User Properties.

Configure Federated User Properties

 

  1. After you complete the fields in Create Federated User form, check the Define Additional Properties check box. Click [Create].

  2. The User Properties form appears. In the [General] tab, optionally:

    • Enter the user's first name into the First Name field

    • Enter the user's last name into the Last Name field.

 NOTE: The X509 Subject Name field is addressed in the Search, Edit, or Delete a Federated User, Group, or Virtual Group section.

  1. Click the [Membership] tab. The blank User Groups window appears. To optionally add the user to one or more groups, click [Add].

  1. The Add Users to the Group form appears. Add the user to one or more FIP groups as follows:

Add to a Single Group

To add the user to a single group, select the group's name and click [Add]. The Add Users to the Group form closes and the group is added to the User Groups window in the User Properties form.

Add to Multiple Groups

    1. Add the user to multiple groups by one of two methods:

      • To select a consecutive range of groups, press [Shift] on your keyboard, left-click the first group in the range, and then left-click the last group in the range

      • To select groups placed throughout the list, press [Ctrl] on your keyboard as you click each group name.

    1. Click [Add]. The Add Users to the Group form closes and the groups are added to the User Groups window in the User Properties form.

  1. Click the [Certificate] tab. The blank Certificate Status window appears. To optionally import a client certificate for the user, click [Import]. The Add Certificate Wizard appears.

 NOTE: It is only necessary to import client certificates for federated users when the Federated Identity Provider (FIP) is configured with no trusted CA certificates. When the FIP is configured with one or more CA certificates, then federated users can be successfully authorized based only on the attributes entered in the Create Federated User form, as long as the certificate presented along with their request was signed by one of the CAs whose certificates are trusted by the FIP. For more information on the different X.509 certificate credential source scenarios, see Workflow Using an X.509 Certificate. The [Remove] button in the [Certificate] tab is addressed in the Search, Edit, or Delete a Federated User, Group, or Virtual Group section.

  1. In the Enter Certificate Info screen of the Add Certificate Wizard, do one of the following:

 NOTE: The client certificate must be signed by the CA root certificate.

Import the Certificate from a File

To import the certificate from a local file:

    • Select the Import from a File option, then either:

      • Enter the file path into the field, or

      • Click [Browse] to navigate to the file location. The Open form appears. Navigate to the location of the certificate. Select the file, and then click [Open].

    • Click [Next]. Proceed to Step #6 below.

Copy and Paste the Certificate

 NOTE: You can only cut and paste a certificate that is in Base 64 PEM format. In the Code window, the SecureSpan™ Manager will only add a pasted certificate that begins with "BEGIN TRUSTED CERTIFICATE."

To copy and paste the certificate from a file into the Code window:

    • Select the Copy and Paste option

    • Copy and then paste the entire certificate from the originating file into the Code window

    • Click [Next]. Proceed to Step #6 below.

  1. One of the following will occur:

    • A pop-up error message notifies you that the certificate could not added:

      1. Note the error.

      2. Click [OK] to close the dialog box.

      3. Return to the Enter Certificate Info screen. Repeat the addition method as outlined in Step #5 above.

      4. Click [Next].

      5. If the View Certificate Details screen of the Add Certificate Wizard appears, proceed to Step #7 below. If the error message appears, continue to fix the error until the Enter Certificate Info screen appears. If necessary, see Technical Support to contact a Technical Support Representative.

    • The View Certificate Details screen of the Add Certificate Wizard appears. Proceed to Step #7 below.

  1. In the View Certificate Details screen of the Add Certificate Wizard, revise the optional name in the Certificate Name field, if required, and view certificate details.

  2. In the Add Certificate Wizard, click [Finish]. One of the following will occur:

    1. If the "Issued to" DN value in the imported certificate does not match the existing X509 Subject DN field value for the federated user, then a message will appear asking you if you want to replace the original DN with the certificate DN. Either:

      • Click [Replace] to proceed with the replacement. The certificate appears in the Certificate Status window in the [Certificate] tab in the User Properties form. Proceed to Step #9 below. Or,

      • Click [Cancel] to cancel the certificate. The User Properties form appears. Repeat Steps #4-8 to add another certificate.

    2. The certificate appears in the Certificate Status window in the [Certificate] tab in the User Properties form. Proceed to Step #9 below.

  1. In the User Properties form, click [OK]. The form closes and the federated user is added to the FIP.

 

Add a New Federated Group

 WARNING: In the SecureSpan™ Manager, a created group is an arbitrary organizational and time saving tool that is useful for granting Web service and XML application access to an organized block of users. Only the default "Gateway Administrators" and "Gateway Operators" access groups actually assign SecureSpan™ Gateway system rights to the users that they contain. Since you cannot add a group to a group in the SecureSpan Manager, you cannot directly give a created group access rights. If you want to give users in a "System Administrators" group Administrator Interface access, for example, then you must add each individual user in that group to the "Gateway Administrators" access group.

In the SecureSpan™ Manager, a new federated group is added to the Federated Identity Provider (FIP) with the Create Federated Group form (basic information) and the optional Group Properties form.

  2. In the Create Federated Group form, enter a descriptive group name into the Group Name field, then do one of the following:

 

 NOTE: The name in the Group Name field will appear in the Search Identity Provider form when searching and/or adding federated groups to a policy. The name is a human readable value that does not impact the validity of the federated group in an identity bridging configuration.

 

    • If you do not want to specify additional group information, then click [Create] without checking the Define Additional Properties check box. The group is added to the FIP. See Search, Edit, or Delete a Federated User, Group, or Virtual Group to view, edit, and/or delete the group at any time

    • To specify a group description and/or add federated users to the group, proceed to Configure Federated Group Properties.

 NOTE: Use the instructions in Search, Edit, or Delete a Federated User, Group, or Virtual Group to modify group membership.

Add Multiple Users

        • To select a consecutive range of users, press [Shift] on your keyboard, left-click the first user in the range, and then left-click the last user in the range

        • To add users placed throughout the list, press [Ctrl] on your keyboard as you click each user name that you want to add.

      • Click [Add]. The Add Users to the Group form closes and the users are added to the Group Membership window in the Group Properties form.

  1. In the Group Properties form, click [OK]. The form closes and the group is added to the FIP.

 

Add a New Federated Virtual Group

 NOTE: You cannot create a virtual group for the Federated Identity Provider (FIP) unless it contains a trusted certificate. In the X.509 certificate credential source workflow (see Workflow Using an X.509 Certificate), the trusted certificate is a CA root certificate attached to the FIP with the Create Federated Identity Provider Wizard. In the SAML credential source workflow, the trusted certificate is a SSL server certificate attached to the FIP with the Create Federated Identity Provider Wizard. Since the imported certificate will have been used to sign client certificates, the "Signing Client Certificates" check box in the Specify Certificate Options screen of the Add Certificate Wizard must also be selected. See Add and Edit Certificates for more information. A virtual group cannot be created in the "Client Certificate Only" X.509 certificate credential source scenario since a trusted certificate does not exist in the FIP.

In the SecureSpan™ Manager, a new federated virtual group is added to the Federated Identity Provider (FIP) with the Create Virtual Group form.

 

  1. Right-click the FIP name in the [Identity Providers] tab. A drop-down menu appears. Select Create Virtual Group.

  2. In the Create Virtual Group form, enter a descriptive group name into the Group Name field.

 NOTE: The name in the Group Name field will appear in the Search Identity Provider form when searching and/or adding virtual groups to a policy. The name is a human readable value that does not impact the validity of the virtual group in an identity bridging configuration.

  1. Enter an optional descriptive statement about the group into the Description field.

  2. In X509 Subject DN field, enter the pattern that the subject DN values of signed client certificates must match in order to be authorized as a member of this virtual group.

 NOTE: The X509 subject DN must at least partially match the "Issued to:" value in the CA root certificate attached to the Federated Identity Provider. Use the asterisk truncation operator (*) to retrieve DNs with a common initial spelling. The (*) substitutes a string of zero or more characters in a subject DN. For example, a virtual group could contain the partial DN "O=ACME Inc., OU=Anvils, CN=*." Request messages in which the Subject DN was "O=ACME Inc., OU=Anvils, CN=Name" or "O=ACME Inc., OU=Anvils, CN=Name2" would both pass the corresponding group assertion that is required to gain Web service access. However, request messages carrying the Subject DN “O=ACME Inc., CN=Name” would not pass. Every attribute specified in the subject DN pattern of a virtual group must be present in incoming certificates in order to be authorized.  

  1. Optional: To authorize incoming requests with incoming SAML tokens using the “emailAddress” NameIdentifier format with email addresses matching a regular expression pattern, enter the pattern into the Email field.

  1. Click [Create]. The form closes and the virtual group is added to the FIP.

 

Search, Edit, or Delete a Federated User, Group, or Virtual Group

In the SecureSpan™ Manager, Federated Identity Provider (FIP) users, groups, and virtual groups are edited in two steps:

  • The Search Identity Provider form is used to find, and if necessary, delete the target user, group, or virtual group

  • The User Properties, Group Properties, or Group Properties forms are used to edit the properties of the respective user, group, or virtual group.

 

  1. Access the Search Identity Provider form by one of two methods:

    • Click the "Search Identity Provider" link on the Home Page

    • Click [Tasks] > Search Identity Provider from the Main Menu

    • Right-click the FIP name in the [Identity Providers] tab. A drop-down menu appears. Select Search Identity Provider.

  1. In the Search Identity Provider form:

    • If not already selected, select the FIP name from the Search drop-down list

    • Select the search target — users, groups, or both — from the Type drop-down list

    • If necessary, specify any search criteria using the Name drop-down list and field

 TIP: The Search Identity Provider form supports the following wild card characters in the Name field:

  • The asterisk truncation operator (*) retrieves terms with a common initial spelling. It substitutes a string of zero or more characters when searching Federated Identity Providers.

    • Click [Search].

  1. Search results appear in the Search Results window. Do any of the following:

Delete a Federated User, Group, or Virtual Group

 WARNING: An error message will appear if you try to delete a federated user, group, or virtual group that is being used by a policy. To delete the federated user, group, or virtual group, you must first delete the identity from all of the policies that it is attached to. See Configure and Edit Policies for more information.

To delete a federated user, group, or virtual group:

 

    1. Select the target federated user, group, or virtual group name in the Search Results window.

    2. Click [Delete]. A pop-up confirmation message appears. Click [OK].

    3. The federated user, group, or virtual group is deleted from the FIP. Do one of the following:

      • Make further changes as necessary, or

      • If further changes are not required, click [Close] to close the Search Identity Provider form.

View or Edit Federated User, Group, or Virtual Group Properties

View or Modify a Federated User

To view or modify a federated user in the FIP:

 

    1. In the Search Results window in the Search Identity Provider form, double-click the target federated user name or select the user name and click [Select].

    2. The User Properties form appears. In the [General] tab, modify federated user details as required.

 TIP: For field-level form information, see Configure Federated User Properties.

    1. In the [Membership] tab, remove the federated user from one or more federated groups if required as follows:

      • To remove the user from a single group, select the group name in the User Groups window and click [Remove]

      • To remove the user from a consecutive range of groups, press [Shift] on your keyboard, left-click the first group in the range, and then left-click the last group in the range. Click [Remove]

      • To remove the user from multiple groups placed throughout the list, press [Ctrl] on your keyboard as you click each group name that you want to remove the user from. Click [Remove].

    2. The group(s) is/are removed from the User Groups window. To add the federated user to one or more groups, follow the instructions in Configure Federated User Properties.

    3. In the [Certificate] tab, click [Remove] to remove the current client certificate if required. To add another client certificate, follow the instructions in Configure Federated User Properties.

    4. If further user changes are not required, then click [Close] to close the User Properties form.

    5. If further FIP changes are not required, then click [Close] to close the Search Identity Provider form.

View or Modify a Federated Group

To view or modify a federated group in the FIP:

 

    1. In the Search Results window in the Search Identity Provider form, double-click the target group name or select the group name and click [Select].

    2. The Group Properties form appears:

      • If required, modify the statement in the Description field

      • If required, remove one or more federated users from the group:

        1. Click the [Membership] tab.

        2. The users that belong to the group are displayed in the Group Membership window:

          • To remove a single user, select the user name that you want to remove. Click [Remove]

          • To remove a consecutive range of users, press [Shift] on your keyboard, left-click the first user in the range, and then left-click the last user in the range. Click [Remove]

          • To remove users placed throughout the list, press [Ctrl] on your keyboard as you click each user name that you want to remove. Click [Remove].

        3. The federated user(s) is/are removed from the Group Membership window.

    3. If further group changes are not required, then click [Close] to close the Group Properties form.

    4. If further FIP changes are not required, then click [Close] to close the Search Identity Provider form.

View or Modify a Virtual Group

To view or modify a virtual group in the FIP:

 

    1. In the Search Results window in the Search Identity Provider form, double-click the target virtual group name or select the virtual group name and click [Select].

    2. The Group Properties form appears:

      • If required, modify the statement in the Description field

      • If required, modify the value in the X509 Subject DN field.

 NOTE: The X509 subject DN must at least partially match the "Issued to" value in the CA root certificate attached to the Federated Identity Provider. Use the asterisk truncation operator (*) to retrieve DNs with a common initial spelling. The (*) substitutes a string of zero or more characters in a subject DN. For example, a virtual group could contain the partial DN "O=ACME Inc., OU=Anvils, CN=*." Request messages in which the DN was "O=ACME Inc., OU=Anvils, CN=Name" or "O=ACME Inc., OU=Anvils, CN=Name2" would both pass the corresponding group assertion that is required to gain Web service access.

    1. If further group changes are not required, then click [Close] to close the Group Properties form.

    2. If further FIP changes are not required, then click [Close] to close the Search Identity Provider form.
