Security Policy Examples

NOTE: The type and order of assertions are important when constructing a valid Web service security policy. Become familiar with the policy and assertion information and rules in Security Policies Overview and Rules. For in-depth information on the category and individual assertions used in a security policy, see Security Policy Assertion Overview. For information on configuring a security policy, see Configure and Edit Security Policies.

The following security policy scenarios illustrate the policy and assertion organization, dependency, and combination rules described in Security Policies Overview and Rules. Use the scenarios as reference-only examples when learning how to construct your own security policy.

 

NOTE: The following examples are for reference purposes only. An "identity" is a group or user configured with the Grant Access to Users and Groups authentication assertion. See Configure and Edit Authentication Assertions for more information. By default, all security policies contain a message routing assertion. See Configure and Edit Message Routing Assertions for more information.

Scenario One: One Authentication Method/One or More Identities

In this security policy, a single authentication method is used, and only the listed groups and users have authorized access to the Web service. Access requests are routed to the defined HTTP(S) Web service URL.

Scenario Two: Multiple Authentication Methods/One or More Identities

In this security policy, multiple authentication methods are used to gain Web service access when the authorized identities evaluate to "true." Access requests are routed to the defined HTTP(S) Web service URL.

Scenario Three: Multiple Authentication Methods/One or More Identities/Response Security

In this security policy, multiple authentication methods are also used to gain Web service access when the authorized identities evaluate to "true." The Web service signs and encrypts the XML response messages. Access requests are routed to the defined HTTP(S) Web service URL.
