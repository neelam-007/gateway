Identity Providers

The Manager displays the content of two kinds of identity providers: internal and external. The internal identity provider refers to the authentication database inside the Gateway. External identity providers are the LDAP authentication sources that reside outside of the Gateway, but inside an organization's firewall. Identity searches can be performed on all providers.

Internal Identity Provider

The Gateway provides an internal identity provider that can be used for small-scale implementations. Administrators use the Manager to create multiple users and groups with full authentication credentials. This data resides directly on the Gateway.

The internal identity provider is best suited for tasks such as acceptance and performance testing or for implementations that have a limited number of users (e.g. external partners). The Manager acts as the credential coordinator for the internal users. Administrators can add, remove, and customize user credentials. For more information on how to add and customize users and groups, see Creating Users and Groups.

Note: The XML Request Security authentication methods can only be used with identities from the internal identity provider.

External Identity Providers

External identity providers must be added to the Manager configuration. Administrators can easily connect to their corporate LDAP or Microsoft Active Directory for access to their established database of users. The Manager offers a simple search view of the external LDAP or Active Directory. This search feature allows administrators to find select groups of users as required.

Note: Identities from external identity providers cannot be combined with the XML Request Security authentication method.

To create a new identity provider:

  1. Go to the Action menu.

  2. Select New provider. The Edit Identity Provider dialog box appears.

  3. Type an appropriate name in the Identity Provider field and select the provider type.

Note: LDAP and MS Active Directory are currently the only external identity provider types.

  1. Enter the relevant data in the following fields: LDAP host url, Search base, Bind DN, and Bind password.

  1. Select Save. The new provider displays under the Identity Provider folder.

Identity Searches

When identity providers have numerous entries, finding a specific identity is a critical task. The Manager offers a search tool to search for and confirm that an identity exists in a specific provider.

To find an identity:

  1. Click the Find button on the Navigation bar. The Search for identities dialog box appears.

  2. Select an identity provider, an identity type, and the search criteria from the drop-down lists and fields provided.

  3. Click the Search button. The search results are displayed in an expanded dialog box.

Note: Double-clicking the identity name displays the identity properties.

  1. Click the Close button to finish.
