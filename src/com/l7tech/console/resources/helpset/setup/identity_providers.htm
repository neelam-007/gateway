Identity Providers

The Manager displays the content of two kinds of identity providers: internal and external. The internal identity provider refers to the authentication database inside the Gateway. External identity providers are the LDAP authentication sources that reside outside of the Gateway, but inside an organization's firewall.

Internal Identity Provider

The Gateway provides an internal identity provider that can be used for small-scale implementations. Administrators use the Manager to create multiple users and groups. The internal identity provider is best suited for tasks such as acceptance and performance testing or for implementations that have a limited number of users (e.g. external partners).

The Manager acts as the credential coordinator for the internal users. Administrators can add, remove, and customize user credentials. For more information on how to add and customize users and groups, see Creating Users and Groups.

External Identity Providers

External identity providers must be added to the Manager configuration. Administrators can easily connect to their corporate LDAP for access to their established database of users. The Manager offers a simple search view of the external LDAP. This search feature allows administrators to find select groups of users as required.

To create a new identity provider:

  1. Go to the Action menu.

  2. Select New provider. The Edit Identity Provider dialog box appears.

  3. Type an appropriate name in the Identity Provider field and select the provider type.

  4. Select Save. The new provider displays under the Identity Provider folder.

Note: LDAP is currently the only identity provider type available. Future releases will offer more provider type options.
