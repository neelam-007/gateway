Users and Groups

Administrators can use the Manager to add new users and groups to the internal identity provider. Users are defined based on their name, authentication credentials, group membership status, and certificate usage. Groups are defined by their description and their user membership.

Creating Users and Groups

The internal identity provider comes with one default user (admin) and one default group (gateway administrators). Administrators can build additional users and groups that suit their policy strategy. Duplicating user credentials that exist on an external LDAP is not advised.

To create a new user:

  1. Go to the Action menu and select Create new user.

  2. Type a user name in the User field and the password in both the Password and the Confirm fields. Select Define Additional Properties if additional credentials, such as email and group membership are required. Click Create.

  3. If additional properties are required, the Properties dialog box appears. Define the additional properties under the General (identification), Membership (groups), and Certificate (added security) tabs. Click OK. The new user appears under the Internal Identity Provider > Users directory.

To create a new group:

  1. Go to the Action menu and select Create new group. The New Group dialog box appears.

  2. Type a name in the Group field. Select Define Additional Properties if additional credentials, such as email and group membership are required. Click Create.

  3. If additional properties are required, the Properties dialog box appears. Define the additional properties under the General (description) and the Membership (add users) tabs. Click OK. The new group appears under the Internal Identity Provider > Users directory.

User/Group Properties

The user properties can be accessed by right-clicking on the user's or the group's name and selecting User Properties or Group Properties. The User Properties dialog box includes three tabs: General, Membership, and Certificate. These tabs support credential changes, group membership, and adding certificates to the user. The Group Properties dialog box includes two tabs: General and Membership. These tabs support description and group membership data.

Changing Group Membership

These tasks refer to changing user and group information in the internal identity provider. Users and groups in the external identity providers (LDAP or Active Directory) should be modified in their appropriate management tools.

To add a user to a group:

  1. Go to the Internal Identity Provider > Group directory.

  2. Right-click on the appropriate group name and select Group Properties from the popup menu. The Properties dialog box appears.

  3. Select the Membership tab. Click the Add button. The Add Users to the Group dialog box appears.

  4. Select the user(s) to add to the group. Hold the CTRL down to select several users. Click the Add button.

  5. Verify the new users in the group and click OK.

To remove a user from a group:

  1. Go to the Identity Provider > Group directory.

  2. Right-click on the appropriate group name and select Group Properties from the pop-up menu. The Properties dialog box appears.

  3. Select the Membership tab. Select the user(s) to remove from the group. Hold the CTRL down to select several users. Click the Remove button.

  4. Verify the final list of users and click OK.

Finding Users

External identity providers may contain hundreds, if not thousands, of identities. The Manger offers a simple search tool to find an identity in any of their identity providers. Administrators can use this tool to verify that a user or group resides in the identity provider and the exact naming convention used.

To find a user/group:

  1. Go to the navigation bar and click the Find button. The Search for identities dialog box appears.

  2. From the drop-down lists provided, select the following information:
    Search provider     - select an identity provider
    Type     - select an identity type (i.e. user, group, or all)
    Name     - type the complete name or the first letters of the name

  3. Click the Search button. The Search results field appears with the name and description (if provided) for the identity. If no results are displayed, there is no match for the search criteria.

  4. Click the New Search button to attempt a new search. Repeat steps 2 and 3.

  5. Click Close to exit.

Deleting Users/Groups

Deleting a user or a group in the Manager is a very simple task. The user/group is immediately removed from both the Manager interface and the Gateway's internal identity provider. As a result, it's important to be sure you want to remove the user or group permanently before following the deletion procedure.

To delete a user/group:

  1. Go to the Internal Identity Provider > User (or Group) directory.

  2. Right-click on the user/group name and select Delete from the pop-up menu. The Delete User/Group dialog box appears.

  3. Verify that you are deleting the correct user/group and click Yes. The user/group is removed from the list.
