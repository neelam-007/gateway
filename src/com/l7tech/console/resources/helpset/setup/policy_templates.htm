Policy Templates

Policies that contain standard, frequently used assertions can be saved as a template. Administrators can then leverage their templates to ensure consistency and to save time when creating multiple services.

Administrators must save policy templates in the appropriate directory to be able to access them via the Manager. This directory appears as the default selection in the policy creation process (policy.templates). Templates saved to this location can be found in the .l7tech\<home> directory and are saved in the XML format.

Caution: Always use the Manager to edit template contents. Do not edit the template manually in the XML format.

Using Policy templates

Administrators must create their own templates according to their own security strategy. Once the templates exist, applying them is a simple drag-and-drop process that eliminates the time-consuming and detailed task of creating individual policies.

To create a policy template

  1. Create a set of policy assertions that will serve as the template.

  2. Click the Validate policy button to ensure that the policy functions cleanly.

  3. Click the Save policy template button. The Save as dialog box appears.

  4. Type a template name in the File Name field. Use the default directory (policy.templates) listed in the Save In field. Saving to this directory supports retrieving the template in the Manager.

  5. Click Save. The new template appears in the Policy templates folder in the Assertions pane.

To apply a policy template

  1. Publish a new service.

  2. Display the service policy in the user interface.

  3. Select the policy template in the Policy templates folder. Either drag-and-drop the template or click the move button to apply the template contents to the service policy on the right side of the screen.  

  4. The policy template overwrites the current policy contents. Click the Save policy button to save the changes.
