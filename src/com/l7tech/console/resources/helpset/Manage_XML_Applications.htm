Manage XML Applications

 NOTE: The SecureSpan™ Manager differentiates between SOAP Web services and XML or non-SOAP applications. To publish, edit, or view a SOAP Web service, see Manage SOAP Web Services. For information on disabling/enabling, renaming, or deleting a published XML application, see Manage Published Services. XML application policies do not support the message-level security assertions found in the XML Security category in the [Assertions] tab (see Configure and Edit XML Security Assertions).

In the SecureSpan™ Manager, XML or non-SOAP applications are published using the Publish XML Application Wizard. The Wizard guides you through the publication process, easily allowing you to enter connection and routing information and access credentials for the XML application. Publication adds an XML application to the SecureSpan Manager's [Services] tab, establishes the XML application's initial policy in the Policy Development window, and allows authorized clients to access the XML application through the SecureSpan™ Gateway. After publication, the SecureSpan Manager allows you to edit the SecureSpan Gateway URL that receives requests for the XML application.

Publish an XML Application

The Publish XML Application Wizard contains two screens corresponding to the two main tasks in the configuration process:

  1. The XML Service Information screen specifies the connection and routing information for the XML application.

  2. The Access Control screen allows you to define access control and authentication rules for the XML application.

 

 TIP:  View screen-specific information and instructions in the Tips Box located near the bottom of the Publish XML Application Wizard.

 

  1. Access the Publish XML Application Wizard by one of three methods:

    • Click the "Publish XML Application" link on the Home Page

    • Click [Tasks] > Publish XML Application from the Main Menu

    • Right-click the "Services @ https://servername:port" title at the top of the [Services] tab. A drop-down menu appears. Select Publish XML Application.

  1. The XML Service Information screen of the Publish XML Application Wizard appears. Do the following:

    • In the Service Name field, enter a name for the XML application

 NOTE: Upon publication, the name entered in the Service Name field will appear in the [Services] tab.

    • In the Target URL field, enter the HTTP URL of the XML application. The SecureSpan™ Gateway will route service requests to the target URL

 NOTE: Upon publication, the URL in the Target URL field will appear as an HTTP(S) Routing assertion in the XML application's initial policy. See Manage Policies and Configure and Edit Message Routing Assertions for more information.  

    • In the Gateway URL field, complete the SecureSpan Gateway URL provided by the SecureSpan™ Manager with a unique URI that corresponds to the unique address that will receive requests for the XML application. Only enter a URI that completes the embedded SecureSpan Gateway URL into the field. If you are connected to SecureSpan Gateway "machinename.domain.com/xml/," for example, then enter the URI for the XML application, such as "Warehouse," into the Gateway URL field. In this example, the final XML application-specific URL that will receive requests would be "machinename.domain.com/xml/Warehouse."

 WARNING: Do not duplicate Gateway URL and URI combinations. Each final URL that receives requests for each individual XML application must be unique. After publication, you can view or edit the Gateway URL from the [Services] tab (see Edit the Gateway URL).

  1. Click [Next]. One of the following will occur:

    1. A pop-up error message notifies you that information is missing or improperly configured:

      1. Note the errors.

      2. Click [OK] to close the dialog box.

      3. Enter or re-enter the information as required.

      4. Click [Next].

      5. If the Access Control screen of the Publish XML Application Wizard appears, proceed to Step #4. If the error message appears, continue to fix and resolve the information until the Access Control screen of the Publish XML Application Wizard appears. If necessary, see Technical Support to contact a Technical Support Representative.

    1. The Access Control screen of the Publish XML Application Wizard appears. Proceed to Step #4.

  1. In the Access Control screen:

    1. Optionally check the Require SSL/TLS Encryption check box if you want to enforce that all requestors consume the XML application through the SSL entry point.

    2. Select one of two access control options:

      • Click Allow Anonymous Access to permit requestors to access the XML application anonymously (without credentials)

      • Click Require Users to Authenticate to require that requestors provide credentials to gain XML application access. Define the authentication details for this option as follows:

 NOTE: If you specify an authentication method and one or more authorized identities in the Publish XML Application Wizard, then XML application access will be restricted to those identities. The initial policy will automatically be populated with an authentication assertion for each Grant Access to Users/Groups assertion corresponding to each selected identity. See Manage Policies and Configure and Edit Access Control Assertions for more information.

        1. Select an authentication method from the Authentication Method drop-down list. The authentication method determines what information users and groups are required to provide to gain XML application access.

        2. Select an identity provider that contains the authorized users and groups from the Identity Provider drop-down list.

 NOTE: You can give LDAP Identity Provider, Federated Identity Provider, and Internal Identity Provider (see Manage Identity Providers) users and groups access to the XML application.

        1. Authorize users and groups to use the XML application by moving them from the No Permission window to the Have Permission window as follows:

          • Click [Add All] to authorize all users and groups in the No Permission window by moving them to the Have Permission window

          • Authorize one or more selected users and groups as follows:

            • To authorize a single user or group, click the user or group name and click [Add]

            • To authorize a continuous range of users and/or groups, hold down your left mouse button as you move from the first user or group to the last user or group in your target range. Or, left-click the first user or group in the range, hold down the [Shift] button on your keyboard, and then left-click the last user or group in the range. Click [Add]

            • To authorize multiple users or groups placed throughout the list, hold down the [Ctrl] button on your keyboard as you click each user or group name. Click [Add].

 NOTE: To de-authorize users and/or groups, move them from the Have Permission window to the No Permission window. Select one or more users and/or groups as outlined above, then click the [Remove] button to remove them from the Have Permission window. To remove all users and/or groups from the Have Permission window, click [Remove All].

        1. Optional: If you need to authorize users or groups from another identity provider, select the new identity provider name from the Identity Provider drop-down list and repeat the instructions in Step #3.

  1. Click [Finish]. The XML application is immediately added to the [Services] tab. Disable the published XML application until its policy is completed. See Manage Published Services for instructions. For information on policies, see Manage Policies.

 

Edit the Gateway URL

The SecureSpan™ Manager allows you to edit the unique SecureSpan™ Gateway URL plus URI combination that receives requests for an XML application.

 

  1. Right-click the XML application name in the [Services] tab. A drop-down menu appears.

  1. Select View Gateway URL. The View Gateway URL form appears.

  2. Complete the Gateway URL by entering a new URI into the field.

 NOTE: In the Gateway URL field, complete the SecureSpan™ Gateway URL provided by the SecureSpan™ Manager with a unique URI that corresponds to the unique address that will receive requests for the XML application. Only enter a URI that completes the embedded Gateway URL into the field. Do not duplicate Gateway URL and URI combinations. Each final URL that receives requests for each individual XML applications must be unique.

  1. Click [OK]. The XML application is updated with the new URL.
