Security Policy Assertions Overview

NOTE: This section discusses configuration instructions for the individual security policy assertions in the [Assertions] tab of the SecureSpan™ Manager. For information on the policy and assertion rules used in the Manager, see Security Policies Overview and Rules. For information on configuring, editing, and manipulating a security policy in the Policy Development window, see Configure and Edit Security Policies.

A security policy defines restrictions for the consumption of a published Web service that is protected by the SecureSpan™ Gateway. Policy assertions are the building blocks for security policies in the SecureSpan™ Manager. Located in the [Assertions] tab, the assertions are organized into categories corresponding to the main requirements of a Web service security policy:

  • Authentication

The Authentication assertions establish the protocols used for credential authentication in a security policy and the authentication requirements for accessing a Web service. See Configure and Edit Authentication Assertions for more information

  • Transport Layer Security (TLS)

The Transport Layer Security (TLS) assertion establishes the encryption requirement for a Web service. See Configure and Edit Transport Layer Security (TLS) Assertions for more information

  • XML Security

The XML Security assertions define the encryption and signature requirements for Web service XML messages. See Configure and Edit XML Security Assertions for more information

  • XML Transformation/Validation

The XML Transformation/Validation assertions configure the XML transformations and validation schemas applied to Web service messages. See Configure and Edit XML Transformation/Validation Assertions for more information

  • Message Routing

The Message Routing assertions define where Web service messages are sent and what access credentials are required by the Web service. See Configure and Edit Message Routing Assertions for more information

  • Web Service Availability

The Web Service Availability assertions define access and availability inclusions or restrictions for the Web service. See Configure and Edit Web Service Availability Assertions for more information

  • Policy Templates

The Policy Templates section is a repository for previously configured security policies that are saved as templates for future use in other Web service policies. For  information on saving and using security policy templates, see Save and Use Security Policy Templates.

 

A Web service security policy is constructed by either:

  • Dragging and dropping an assertion from the [Assertions] tab into the Policy Development window, or by

  • Highlighting the target assertion in the [Assertions] tab and clicking the  (Move Assertion) button in the Assertion Tool Bar.

Most assertions require configuration either before or after being added to the Policy Development window. See the assertion category-based topic pages for instructions for the individual assertions. See Security Policies Overview and Rules for general policy and assertion information and rules. See Configure and Edit Security Policies for information on constructing a security policy.
