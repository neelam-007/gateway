Security Policy Assertions Overview

 NOTE: This section discusses configuration instructions for the individual security policy assertions in the [Assertions] tab of the SecureSpan™ Manager. For information on the policy and assertion rules used in the Manager, see Security Policy Organization and Rules. For information on configuring, editing, and manipulating a security policy in the Policy Development window, see Configure and Edit Security Policies.

A security policy defines restrictions for the consumption of a published service that is protected by the SecureSpan™ Gateway. Policy assertions are the building blocks for security policies in the SecureSpan™ Manager. Located in the [Assertions] tab, the assertions are organized into categories corresponding to the main requirements of a security policy:

  • Authentication

The Authentication assertions establish the authorized identities and corresponding credential authentication protocols and requirements in a security policy. See Configure and Edit Authentication Assertions for more information

  • Transport Layer Security (TLS)

The Transport Layer Security (TLS) assertion establishes the transport-level encryption requirement for a service. See Configure and Edit the Transport Layer Security (TLS) Assertion for more information

  • XML Security

The XML Security assertions define the message-level encryption and signature requirements for service XML messages and enable protection against replayed request messages. See Configure and Edit XML Security Assertions for more information

  • Message Validation/Transformation

The Message Validation/Transformation assertions configure the XML transformations and validation schemas applied to service messages. See Configure and Edit Message Validation/Transformation Assertions for more information

  • Message Routing

The Message Routing assertions define where service messages are sent and what access credentials are required by the back-end service. See Configure and Edit Message Routing Assertions for more information

 NOTE: The Message Routing assertions are not concerned with the credential requirements of the SecureSpan™ Gateway. Use them to define the credential requirements of the target back-end system that will receive Gateway-routed service messages.

  • Service Availability

The Service Availability assertions define access and availability inclusions or restrictions for the service. See Configure and Edit Service Availability Assertions for more information

  • Logging and Auditing

The Logging and Auditing assertion sets the audit criteria for the SecureSpan™ Gateway service activity messages audited through the Gateway Audit Events window. See Configure and Edit the Logging and Auditing Assertion for more information

  • Policy Templates

The Policy Templates section is the repository for exported security policies. For information on importing or exporting a security policy, see Import or Export Security Policies.

 

 NOTE: See Optional Add-On Assertions Overview for information about the three optional SecureSpan™ Solution assertions: the SiteMinder™ authentication assertion, the Tivoli Access Manager (TAM) authentication assertion, and the JMS message routing assertion.

 

Unless specified otherwise, all assertions can be used in a Web service and XML application security policy. A security policy is constructed by either:

  • Dragging and dropping an assertion from the [Assertions] tab into the Policy Development window, or by

  • Highlighting the target assertion in the [Assertions] tab and clicking the  (Add Assertion) button in the Assertion Tool Bar.

Most assertions require configuration either before or after being added to the Policy Development window. See the assertion category-based topic pages for instructions for the individual assertions. See Security Policy Organization and Rules for general policy and assertion information and rules. See Configure and Edit Security Policies for information on constructing a security policy.
