About Version 3.3 of the SecureSpan™ Manager

Version 3.3 of the SecureSpan™ Manager includes the following additions and enhancements:

  • Ability to load policy schemas on a Tarari PCI card (installed in the SecureSpan™ Gateway appliance or on a server with the SecureSpan Gateway software) for accelerated schema processing and validation

 WARNING: If using a Tarari PCI card for schema validation, then note the limitations in Validate XML Schema in Configure and Edit Message Validation/Transformation Assertions prior to configuring a service policy.

  • Using the new Manage Global Schemas form (see Manage Global Schemas), the ability to add and manage the global XML schemas that are referenced by import statements in the schemas added with the Validate XML Schema assertion (see Configure and Edit Message Validation/Transformation Assertions)

  • Ability to add comments to a policy with the new Comment assertion (see Configure and Edit Policy Logic Assertions)

  • Ability to configure a SAML attesting entity (client-side) certificate (see Manage Certificates) for validation by the server-side Federated Identity Provider (see Manage Federated Identity Providers) and SAML Assertion (see Configure and Edit Access Control Assertions)

  • New KeyInfo references in the SAML Assertion that allow the storage of SAML assertions as HTTP cookies (see Configure and Edit Access Control Assertions)

  • SSL/TLS client authentication support through the HTTP(S) Routing assertion (see Configure and Edit Message Routing Assertions)

  • Context Variable support through the enhanced XPath assertions (see Configure and Edit Message Validation/Transformation Assertions) and the new Equality assertion (see Configure and Edit Policy Logic Assertions) that will compare two specific policy values for equality during runtime

  • Safenet Luna SP appliance support, allowing the SecureSpan™ Gateway to use Luna as the CA keystore, as a JCE provider, and for certificate signing

  • A new Stealth Fault assertion that instructs the SecureSpan Gateway to drop a client connection instead of generating a SOAP fault or other error when policy requirements are not met or when a service cannot be resolved at runtime (see Configure and Edit Transport Layer Security (TLS) Assertions).

 

Version 3.3 of the SecureSpan Manager also includes improvements to many 3.2 features.
