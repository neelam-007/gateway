SSL or TLS Transport

The SSL or TLS Transport assertion uses the Transport Layer Security (TLS) protocol to ensure transport-level confidentiality and integrity. The SecureSpan™ Manager allows you to set one of three SSL or TLS requirements for a policy:

Required

Select the Required option to require that incoming requests arrive over an SSL-secured connection. This is the default setting for the assertion. When selected, the client certificate authentication option is enabled. To require client certificate authentication of request messages in the policy, check the Require Client Certificate Authentication check box.

Optional

Select the Optional option to configure the SecureSpan™ Gateway to match the behavior of the incoming request. Requests are not required to arrive over an SSL-secured connection, but will be processed anyway if they happen to do so.

Forbidden

Select the Forbidden option to disallow requests that arrive over an SSL-secured connection.

Add and Configure the SSL or TLS Transport  Assertion

  1. Add the assertion to the Policy Development window by one of two methods:

    • Drag and drop the assertion from the [Assertions] tab into the Policy Development window, or  

    • Highlight the assertion in the [Assertions] tab and click the  (Add Assertion) button in the Assertion Tool Bar.

  1. The assertion appears in the Policy Development window with a default setting of "required." To optionally change the assertion setting, right-click the "Require SSL or TLS Transport" assertion. A drop-down menu appears. Select SSL Properties.

  1. The SSL or TLS Options form appears. Select one of the SSL or TLS requirement options.

TIP: If you select the Required option, then you can also opt to require the client certificate authentication of request messages in the policy. To do so, check the Require Client Certificate Authentication check box.

  1. To require a client certificate signed by the SecureSpan™ Gateway certificate authority as part of the SSL-TLS handshake, check the Require Client Certificate Authentication checkbox. The client certificate is used to authenticate the service requestor.

NOTE: When the Require Client Certificate Authentication checkbox is checked, the SSL or TLS Transport assertion is the same as the SSL with Client Certificate Authentication assertion from the Access Control category in the [Assertions] tab (see Configure and Edit Access Control Assertions).

  1. Click [OK]. Assertion changes take effect immediately.

Edit the SSL or TLS Transport Assertion

  • In the Policy Development window, right-click the "SSL or TLS Transport" assertion. A drop-down menu appears. Select SSL Properties

  • The SSL or TLS Options form appears. Edit the assertion properties as outlined in Add and Configure the SSL or TLS Transport Assertion.

 

Stealth Fault

The Stealth Fault assertion instructs the SecureSpan™ Gateway to drop a client connection instead of generating a SOAP fault or other error when the policy requirements for a particular service are not met. When this assertion is encountered in a policy path, the policy execution context is flagged to not return any errors. If the policy execution does result in an error, then this error is not sent back to the requestor; the connection to the client, or requestor, is dropped instead.

 

TIP: The Stealth Fault assertion permits the policy-based suppression of SOAP faults. You can configure the global suppression of SOAP faults for service resolution errors by directly modifying the SecureSpan™ Gateway properties file. By default, the SecureSpan Gateway generates a "service not found" SOAP fault when a service cannot be resolved at runtime. By directly adjusting the SecureSpan Gateway properties file, you can configure the SecureSpan Gateway to drop the client connection whenever requests do not resolve a service instead. When a service is resolved when global suppression is configured in the SecureSpan Gateway, then the presence of the Stealth Fault assertion will determine the policy-based suppression of SOAP faults. See the "SecureSpan™ Installation and Maintenance Manual" for instructions and more information.

 

  • Add the assertion to the Policy Development window by one of two methods:

    • Drag and drop the assertion from the [Assertions] tab into the Policy Development window, or  

    • Highlight the assertion in the [Assertions] tab and click the  (Add Assertion) button in the Assertion Tool Bar.

  • The assertion is added to the Policy Development window.

 

NOTE: To remove the suppression of SOAP faults, remove the Stealth Fault assertion from the Policy Development window. To do so, right-click the assertion name in the Policy Development window. A drop-down menu appears. Select [Delete Assertion].
