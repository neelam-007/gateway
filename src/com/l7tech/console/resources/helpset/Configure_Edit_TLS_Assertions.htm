Configure and Edit Transport Layer Security (TLS) Assertions

In the SecureSpan™ Manager, the following single assertion is available in the Transport Layer Security (TLS) category of the [Assertions] tab:

SSL Security

The SSL Security assertion uses the Transport Layer Security (TLS) protocol to ensure message confidentiality and security. In the Policy Development window, you can enable or disable the assertion as required.

Add and Configure the SSL Security Assertion

  1. Add the assertion to the Policy Development window by one of two methods:

    • Drag and drop the assertion from the [Assertions] tab into the Policy Development window, or  

    • Highlight the assertion in the [Assertions] tab and click the  (Move Assertion) button in the Assertion Tool Bar.

  1. The assertion appears in the Policy Development window with a default setting of "required." Optional: To change the assertion setting, right-click the "Require SSL Transport" assertion. A drop-down menu appears. Select SSL Properties.

TIP: You can also double-click the assertion in the Policy Development window to open the SSL Transport Properties form.  

  1. The SSL Transport Properties form appears. Select a setting option from the drop-down list. Click [OK].

  1. The modified assertion is added to the Policy Development window.

Edit the SSL Security Assertion

  1. In the Policy Development window, right-click the "Require SSL Transport" assertion. A drop-down menu appears. Select SSL Properties

TIP: You can also double-click the assertion in the Policy Development window to open the SSL Transport Properties form.   

  1. The SSL Transport Properties form appears. Change the assertion setting in the drop-down list as required.

  2. Click [OK]. The modified assertion is added to the Policy Development window.
