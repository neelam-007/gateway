Configure and Edit the Transport Layer Security (TLS) Assertion

In the SecureSpan™ Manager, the following assertion is available in the Transport Layer Security (TLS) category of the [Assertions] tab:

 

The Transport Layer Security (TLS) Category of the [Assertions] Tab

SSL Transport

The SSL Transport assertion uses the Transport Layer Security (TLS) protocol to ensure message confidentiality and security. SSL is an industry-standardized method of encryption that relies on PKI (public key infrastructure) to create a secure communication pipe between the service requestor and the SecureSpan™ Gateway. The SecureSpan™ Manager allows you to set one of three SSL security requirements for a security policy:

Optional

The optional setting of the SSL Transport assertion will match the behavior of the incoming request. Requests are not required to arrive over an SSL-secured connection, but will be processed anyway if they happen to do so.

Required

The required setting of the SSL Transport assertion disallows requests that do not arrive over an SSL-secured connection. This is the default setting for the SSL Transport assertion.

Disallowed

The disallowed setting of the SSL Transport assertion disallows requests that arrive over an SSL-secured connection. The assertion setting could be used to discourage users of a free service from consuming server SSL resources without paying for an upgraded account.

Add and Configure the SSL Security Assertion

  1. Add the assertion to the Policy Development window by one of two methods:

    • Drag and drop the assertion from the [Assertions] tab into the Policy Development window, or  

    • Highlight the assertion in the [Assertions] tab and click the  (Add Assertion) button in the Assertion Tool Bar.

  1. The assertion appears in the Policy Development window with a default setting of "required." To optionally change the assertion setting, right-click the "Require SSL Transport" assertion. A drop-down menu appears. Select SSL Properties.

 TIP: You can also double-click the assertion in the Policy Development window to open the SSL Transport Properties form.  

  1. The SSL Transport Properties form appears. Select a setting option from the drop-down list. Click [OK].

  2. The modified assertion is added to the Policy Development window.

Edit the SSL Security Assertion

  1. In the Policy Development window, right-click the "SSL Transport" assertion. A drop-down menu appears. Select SSL Properties.

 TIP: You can also double-click the assertion in the Policy Development window to open the SSL Transport Properties form.   

  1. The SSL Transport Properties form appears. Change the assertion setting in the drop-down list as required.

  2. Click [OK]. The modified assertion is added to the Policy Development window.
