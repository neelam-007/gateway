Performance test framework based on Japex.

This framework generates 3 types of reports: Japex Test Reports, Trend Report and Regression Report.

The normal usage pattern is:

  1. Run {@link com.sun.japex.Japex}.
    • Input: a Japex configuration file that uses a {@link com.l7tech.test.performance.JUnitDriver} to time individual test methods in a JUnit {@link junit.framework.TestCase} (see javadoc in {@link com.l7tech.test.performance.JUnitDriver} for an example configuration file)
    • Output: a Japex test report (XML) with timing results
    Repeat for different snapshot builds to build up a series of Japex test reports.
    Hand-edit the test report to rename report or add notes if neccessary.
    Saves test reports of official release to source control as appropriate.
  2. Run {@link com.l7tech.test.performance.TrendReport}.
    • Input: Japex test reports that carries out the same tests on different snapshot builds.
    • Output: a trend report (HTML) that summarizes the Japex test reports using tables and charts
  3. Run {@link com.l7tech.test.performance.RegressionReport}.
    • Input: Japex test reports that carries out the same tests on different snapshot builds.
    • Output: a regression report (HTML) and e-mail notifications that flags tests that have reached a predefined threshold.

The recommended directory hierarchy to saves these report is:

web server subfolder/
→reports/
→trend/
→regression/

@see Japex Manual
